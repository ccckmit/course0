# 資訊理論

**標籤**: #資訊理論 #熵 #交叉熵 #KL散度 #互資訊 #霍夫曼編碼 #夏農 #香農極限
**日期**: 2024-01-15

資訊理論是由克勞德·夏農 (Claude Shannon) 在 1948 年創立的一門數學學科，主要研究資訊的量化、儲存、傳輸和處理。其核心目標是研究如何有效率且準確地在通訊系統中傳遞訊息。

## 熵 (Entropy)

熵是資訊理論中最核心的概念，用來衡量隨機變數或資訊源的不確定性。

### 定義

對於離散隨機變數 $X$，其取值為 $x$ 的機率為 $p(x)$，則熵 $H(X)$ 定義為：

$$H(X) = - \sum_{x \in X} p(x) \log_2 p(x)$$

這裡使用以 2 為底的對數，單位是位元 (bit)。

### 熵的直覺意義

1. **不確定性**：事件越不確定、發生機率越低，熵越高。
2. **資訊量**：熵越高，該訊息包含的資訊量越大。
3. **編碼長度**：熵代表無損壓縮時每個符號所需的平均最小編碼位元數。

### 二元熵函數

對於二元隨機變數，設 $p(1) = p$，$p(0) = 1-p$，則熵為：

$$H(p) = -p \log_2 p - (1-p) \log_2(1-p)$$

此函數在 $p = 0.5$ 時達到最大值 $1$。

## 交叉熵 (Cross-Entropy)

交叉熵用來衡量兩個機率分佈之間的差異。

### 定義

$$H(p, q) = - \sum_{x \in X} p(x) \log_2 q(x)$$

其中 $p(x)$ 是真實分佈，$q(x)$ 是預測分佈。

### 與熵的關係

$$H(p, q) = H(p) + D_{KL}(p \| q)$$

## KL 散度 (Kullback-Leibler Divergence)

KL 散度又稱相對熵，用來衡量用一個分佈近似另一個分佈所付出的代價。

### 定義

$$D_{KL}(p \| q) = \sum_{x \in X} p(x) \log_2 \frac{p(x)}{q(x)}$$

### 特性

- 當 $p = q$ 時，$D_{KL}(p \| q) = 0$
- KL 散度是**非對稱**的：$D_{KL}(p \| q) \neq D_{KL}(q \| p)$

## 互資訊 (Mutual Information)

互資訊用來衡量兩個隨機變數之間的相互依賴程度。

### 定義

$$I(X; Y) = \sum_{x \in X} \sum_{y \in Y} p(x, y) \log_2 \frac{p(x, y)}{p(x)p(y)}$$

也可寫成：

$$I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)$$

## 霍夫曼編碼 (Huffman Coding)

霍夫曼編碼是一種無損資料壓縮演算法，其核心思想是給予高頻率的字元較短的編碼，給予低頻率的字元較長的編碼。

### 與熵的關係

- 熵是理論上的最小平均編碼長度
- 當所有符號的機率都是 $2$ 的負整數次方時，霍夫曼編碼的平均編碼長度完全等於熵

## 信道編碼定理

### 信道容量

對於連續信道，信道容量由夏農-哈特利定理給出：

$$C = B \log_2\left(1 + \frac{S}{N}\right)$$

其中：
- $B$ 是信道頻寬
- $S/N$ 是訊號雜訊比

### 定理內容

- 若傳輸速率 $R < C$，則存在一種編碼方式可使錯誤機率任意小
- 若 $R > C$，則無法實現可靠傳輸

這就是著名的**香農極限**。

## 應用領域

1. **資料壓縮**：ZIP、Huffman 編碼
2. **通訊系統**：5G、衛星通訊
3. **機器學習**：損失函數、變分推斷
4. **密碼學**：金鑰長度設計

## 舉例說明

### 舉例：計算二元熵

硬幣擲出正面機率 p = 0.7，反面機率 q = 0.3：

$$H = -0.7 \log_2(0.7) - 0.3 \log_2(0.3)$$
$$= -0.7 \times (-0.515) - 0.3 \times (-1.737)$$
$$= 0.361 + 0.521 = 0.882 \text{ bits}$$

若硬幣公平 (p = q = 0.5)：
$$H = -0.5 \log_2(0.5) - 0.5 \log_2(0.5) = 1 \text{ bit}$$

這說明：越不公平的硬幣，熵越低（不確定性越低）。

### 舉例：KL 散度的非對稱性

真實分佈 p = {A: 0.5, B: 0.5}，兩個近似分佈：
- q₁ = {A: 0.5, B: 0.5}
- q₂ = {A: 0.9, B: 0.1}

$$D_{KL}(p || q₁) = 0 \text{（完全相同）}$$

$$D_{KL}(p || q₂) = 0.5 \log_2(0.5/0.9) + 0.5 \log_2(0.5/0.1)$$
$$= 0.5 \times (-0.847) + 0.5 \times 2.322 = 0.737 \text{ bits}$$

但若反過來：
$$D_{KL}(q₂ || p) = 0.9 \log_2(0.9/0.5) + 0.1 \log_2(0.1/0.5)$$
$$= 0.9 \times 0.847 + 0.1 \times (-2.322) = 0.437 \text{ bits}$$

兩者不相等，這就是 KL 散度的非對稱性。

### 舉例：互資訊的計算

天氣與是否帶傘的關係：
- P(帶傘) = 0.3，P(不带傘) = 0.7
- P(下雨) = 0.2，P(不下雨) = 0.8
- P(帶傘且下雨) = 0.18，P(帶傘且不下雨) = 0.12

計算 H(天氣) = -(0.2 log₂0.2 + 0.8 log₂0.8) = 0.722 bits
計算 H(天氣|帶傘) 會發現知道帶與否會減少天氣的不確定性
I(天氣;帶傘) > 0，表示兩者有相關性。

### 舉例：霍夫曼編碼的實際應用

字元頻率：A: 45, B: 13, C: 12, D: 16, E: 9, F: 5

建樹過程：
1. 將每個字元視為一個節點
2. 合併頻率最低的兩個節點
3. 重複直到只剩一個根節點

編碼結果（可能之一）：
- A: 0
- B: 101
- C: 100
- D: 111
- E: 1101
- F: 1100

平均編碼長度：
(45×1 + 13×3 + 12×3 + 16×3 + 9×4 + 5×4) / 100 = 2.32 bits

理論最小值（熵）：
H = -(0.45 log₂0.45 + 0.13 log₂0.13 + 0.12 log₂0.12 + 0.16 log₂0.16 + 0.09 log₂0.09 + 0.05 log₂0.05) = 2.21 bits

接近熵值，壓縮效果好。

### 舉例：香農極限的計算

衛星通訊系統：
- 頻寬 B = 10 MHz
- 訊號雜訊比 S/N = 100 (20 dB)

$$C = 10 \times 10^6 \times \log_2(1 + 100)$$
$$= 10^7 \times \log_2(101)$$
$$= 10^7 \times 6.658$$
$$\approx 66.58 \text{ Mbps}$$

這是理論最大值，實際系統會略低於此極限。
