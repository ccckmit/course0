# 微服務架構：服務拆分與協作

## 概述

微服務架構是一種將應用程式拆分為一組小型、自治服務的設計方式。每個服務運行在自己的進程中，通過輕量級協議（通常是 HTTP REST 或消息隊列）進行通訊。這種架構支持服務的獨立部署和擴展，但也帶來了分散式系統的挑戰。

## 服務設計原則

```python
# 服務設計示例：電子商務系統

微服務拆分：
- 用戶服務 (User Service)

- 通知服務 (Notification Service)
- 物流服務 (Shipping Service)
"""

# API 設計
class UserService:

    endpoints = {

        'PUT /users/{id}': '更新用戶信息',
        'DELETE /users/{id}': '刪除用戶',
        'POST /users/{id}/addresses': '添加地址',
    }
    
    def create_user(self, user_data):
        # 驗證數據
        # 保存到數據庫
        # 發布用戶創建事件
        pass

# Saga 模式 - 分散式事務
class OrderSaga:
    """訂單處理的 Saga 協調"""
    
    def execute_create_order(self, order_request):
        steps = [
            ('reserve_inventory', self.reserve_inventory),
            ('process_payment', self.process_payment),
            ('create_order', self.create_order),
            ('notify_customer', self.notify_customer),
        ]
        
        completed = []
        for step_name, step_func in steps:
            try:
                result = step_func(order_request)
                completed.append((step_name, result))
            except Exception as e:
                # 補償已完成的步驟
                self.compensate(completed)
                raise OrderCreationFailed(str(e))
        
        return completed[-1][1]
    
    def compensate(self, completed):
        for step_name, _ in reversed(completed):
            compensation = self.get_compensation(step_name)
            compensation()
```

## 總結

微服務架構提供了靈活性但增加了複雜性。成功的微服務需要良好的服務邊界定義、API 版本管理、服務發現和分散式追蹤。
