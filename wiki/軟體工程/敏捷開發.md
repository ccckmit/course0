# 敏捷開發

敏捷開發是一種以人為本、迭代式的軟體開發方法論，它強調快速交付可運作軟體、持續回饋和適應變化。敏捷宣言於 2001 年由 17 位軟體專家在美國猶他州滑雪勝地共同發布，徹底改變了軟體產業的開發方式。與傳統的瀑布模型相比，敏捷開發更加靈活，能夠更好地應對不確定的需求和快速的技術變化。

## 敏捷宣言

```
┌─────────────────────────────────────────────────────────────┐
│                  敏捷宣言四個價值觀                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 個人與互動  >  流程與工具                               │
│      - 重視團隊成員的能力和協作                              │
│      - 工具只是手段，不是目的                               │
│                                                             │
│   2. 可運作軟體  >  詳盡文件                                 │
│      - 最終交付的是可用的軟體                              │
│      - 文件是支持工具，但不是成功標準                       │
│                                                             │
│   3. 客戶合作  >  合約談判                                   │
│      - 與客戶持續合作，共同定義需求                         │
│      - 成功的專案需要雙方的投入和信任                        │
│                                                             │
│   4. 回應變化  >  遵循計畫                                   │
│      - 變化是機會，不是問題                                │
│      - 計畫應該有彈性，歡迎改變                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 十二個原則

1. 最高優先級是透過早期和持續交付有價值的軟體來滿足客戶
2. 歡迎改變需求，即使在開發後期也歡迎
3. 頻繁交付可運作軟體（數週而非數月）
4. 業務人員和開發人員必須每天在一起工作
5. 專案是由動機充足的人構建的
6. 面對面交談是最有效的溝通方式
7. 可運作軟體是進度的首要衡量標準
8. 可持續的開發速度
9. 持續關注技術卓越和良好設計
10. 簡約——最大化未完成工作量的藝術
11. 自組織團隊產生最好的架構和設計
12. 定期反思並調整行為

#### Sprint Planning

```python
# Sprint Planning 議程
def sprint_planning(product_backlog, sprint_goal, team_capacity):
    """
    Sprint 規劃會議
    """
    # 輸入
    print(f"目標: {sprint_goal}")
    print(f"團隊容量: {team_capacity} story points")
    
    # 選擇待辦事項
    selected_items = []
    remaining_capacity = team_capacity
    
    for item in sorted(product_backlog, key=lambda x: x.priority):
        if item.points <= remaining_capacity:
            selected_items.append(item)
            remaining_capacity -= item.points
        else:
            break
    
    # 承諾
    sprint_backlog = {
        "goal": sprint_goal,
        "items": selected_items,
        "capacity": team_capacity,
        "committed_points": sum(item.points for item in selected_items)
    }
    
    return sprint_backlog
```

### 2. Kanban

Kanban 是一種視覺化的工作流管理方法，起源於豐田的生產系統。

```
┌─────────────────────────────────────────────────────────────┐
│                    Kanban 看板                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   To Do   │ In Progress │  Code Review  │   Done   │
│   ┌───┐   │   ┌───┐     │    ┌───┐     │  ┌───┐   │
│   │ T1 │   │   │ T3 │     │    │ T5 │     │  ┌───┐   │
│   ├───┤   │   ├───┤     │    ├───┤     │  ├───┤   │
│   │ T2 │   │   │ T4 │     │    │ T6 │     │  ├───┤   │
│   └───┘   │   └───┘     │    └───┘     │  └───┘   │
│           │              │              │              │
│   WIP: 2  │   WIP: 2     │   WIP: 2     │              │
│   Limit   │   Limit      │   Limit      │              │
│                                                             │
│   規則：                                                   │
│   - 限制每個階段的在製品數量（WIP Limit）                   │
│   - 看板必須流動才能拉取新工作                             │
│   - 可視化瓶頸和問題                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. 速率（Velocity）計算

```python
# 團隊速率計算
velocity_data = {
    "sprint_1": {"committed": 25, "completed": 22},
    "sprint_2": {"committed": 27, "completed": 25},
    "sprint_3": {"committed": 23, "completed": 18},
    "sprint_4": {"committed": 30, "completed": 27},
    "sprint_5": {"committed": 25, "completed": 21}
}

# 平均速率
completed_points = [22, 25, 18, 27, 21]
avg_velocity = sum(completed_points) / len(completed_points)
print(f"平均速率: {avg_velocity} story points/sprint")

# 用於預測
def predict_completion(story_points, avg_velocity):
    sprints_needed = story_points / avg_velocity
    return f"預估需要 {sprints_needed:.1f} 個 Sprint"
```
