# 程式碼品質 (Code Quality)

## 概述

程式碼品質指程式碼的可讀性、可維護性、效率與可靠程度。高品質程式碼能降低錯誤率、減少維護成本、加速新功能開發。

## 品質維度

### 可讀性 (Readability)
- 一致的命名規範
- 適當的註解與文件
- 清晰的程式結構

### 可維護性 (Maintainability)
- 低耦合、高內聚
- 遵循 DRY 原則（Don't Repeat Yourself）
- 避免長函數與過度巢狀

### 效率 (Efficiency)
- 演算法複雜度合理
- 資源使用優化
- 避免不必要的計算

### 可靠度 (Reliability)
- 錯誤處理完善
- 邊界條件考慮周全
- 單元測試覆蓋率高

## 程式碼標準

### 命名規範
```python
# 變數：小寫 + 底線
user_name = "Alice"
is_active = True

# 類別：駝峯式
class UserAccount:
    pass

# 常數：全大寫
MAX_RETRY_COUNT = 3

# 函數：動詞或動詞片語
def get_user_info(user_id):
    pass

def validate_email(email):
    pass
```

### 程式碼結構
```python
# 好的範例：清晰的分段
class OrderProcessor:
    def __init__(self, payment_gateway, inventory_service):
        self.payment = payment_gateway
        self.inventory = inventory_service

    def process_order(self, order):
        # 1. 驗證訂單
        self._validate_order(order)
        
        # 2. 檢查庫存
        self._check_inventory(order)
        
        # 3. 處理付款
        self._process_payment(order)
        
        # 4. 發送通知
        self._notify_customer(order)

    def _validate_order(self, order):
        if not order.items:
            raise ValueError("Order must have items")
```

## 常見反模式

| 反模式 | 說明 | 後果 |
|--------|------|------|
| God Object | 單一類別承擔過多功能 | 難以維護、測試 |
| Shotgun Surgery | 小改變需要跨多檔案修改 | 高耦合 |
| Magic Numbers | 使用未命名的數值常數 | 可讀性差 |
| Spaghetti Code | 複雜的跳轉邏輯 | 難以理解 |

## 程式碼審查 (Code Review)

### 審查清單
- [ ] 程式碼是否可讀？
- [ ] 是否有測試覆蓋新功能？
- [ ] 是否遵循團隊程式碼標準？
- [ ] 錯誤處理是否完善？
- [ ] 是否有安全漏洞？

### 審查技巧
1. **小型增量**：每次審查少於 400 行
2. **自動化先行**：先跑靜態分析工具
3. **建設性回饋**：建議而非批評

## 靜態分析工具

| 語言 | 工具 |
|------|------|
| Python | Pylint, Flake8, Black |
| JavaScript | ESLint, Prettier |
| Java | Checkstyle, SpotBugs |
| 多語言 | SonarQube |
