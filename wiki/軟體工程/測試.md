# 測試

測試是軟體開發過程中確保軟體品質的關鍵活動。透過系統化的測試，我們可以驗證軟體是否正確實現了預期功能，並在軟體變更時及時發現可能引入的錯誤。良好的測試策略不僅能提高軟體品質，還能增強開發團隊的信心，使他們能夠安心地進行重構和持續改進。

## 測試的價值與目的

### 為什麼需要測試？

- **發現缺陷**：在軟體上線前發現錯誤
- **防止回歸**：確保新變更不會破壞現有功能
- **文件化行為**：測試案例作為軟體行為的文件
- **重構信心**：有充分測試覆蓋才能安全重構
- **持續交付基礎**：自動化測試是 CI/CD 的基石

### 測試金字塔

測試金字塔是指導測試策略的重要原則，它建議在不同層次的測試中分配不同的資源：

```
                    ▲
                   /│\        E2E 測試（少）
                  / │ \
                 /──│──\      整合測試（中）
                /   │   \
               /────│────\   單元測試（多）
              ╱     │     ╲
             ╱──────│──────╲
            ╱       │       ╲
           ▼────────│────────▼

     ┌─────────────────────────────────────┐
     │  單元測試 → 整合測試 → E2E 測試    │
     │  (快速、便宜)   (中等)    (慢、貴) │
     └─────────────────────────────────────┘
```

**金字塔原則**：
- **底部（單元測試）**：數量多、執行快、成本低
- **中部（整合測試）**：數量適中、驗證組件協作
- **頂部（E2E測試）**：數量少、執行慢、接近真實使用

## 測試的層次

### 1. 單元測試（Unit Test）

單元測試是對軟體中最小可測試單元的測試，通常是單個函數或類別。

#### 單元測試的特點

- **隔離性**：每個測試獨立運行，不依賴其他測試
- **快速**：執行時間通常在毫秒級
- **確定性**：相同的輸入總是產生相同的輸出
- **可維護性**：測試代碼應該與被測試代碼一樣整潔

#### 單元測試範例

```python
# 測試對象：簡單的計算器函數
def add(a, b):
    """返回兩個數字的和"""
    return a + b

def subtract(a, b):
    """返回兩個數字的差"""
    return a - b

def multiply(a, b):
    """返回兩個數字的積"""
    return a * b

def divide(a, b):
    """返回兩個數字的商，b 不能為 0"""
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b

# 使用 pytest 編寫單元測試
import pytest

        assert add(2, 3) == 5
    
    def test_add_negative_numbers(self):
        assert add(-1, -1) == -2
    
    def test_add_with_zero(self):
        assert add(5, 0) == 5
    
    def test_subtract(self):
        assert subtract(10, 5) == 5
    
    def test_multiply(self):
        assert multiply(3, 4) == 12
    
    def test_divide(self):
        assert divide(10, 2) == 5
    
    def test_divide_by_zero(self):
        with pytest.raises(ValueError):
            divide(10, 0)

# 參數化測試 - 用不同參數測試同一邏輯
@pytest.mark.parametrize("a,b,expected", [
    (2, 3, 5),
    (-1, 1, 0),
    (0, 0, 0),
    (-5, -5, -10),
])
def test_add_parametrized(a, b, expected):
    assert add(a, b) == expected
```

### 2. 整合測試（Integration Test）

整合測試驗證多個組件之間的協作，確保它們正確地協同工作。

```python
# 整合測試範例：測試資料庫操作
import pytest
from app.database import Database
from app.models import User

@pytest.fixture
def db():
    """提供測試用的資料庫連接"""
    database = Database(":memory:")  # 使用記憶體資料庫
    database.initialize()
    yield database
    database.close()

class TestUserRepository:
    def test_create_user(self, db):
        # 測試創建用戶
        user = User(name="Alice", email="alice@example.com")
        result = db.users.create(user)
        
        assert result.id is not None
        assert result.name == "Alice"
    
    def test_find_user_by_email(self, db):
        # 先創建用戶
        user = User(name="Bob", email="bob@example.com")
        db.users.create(user)
        
        # 測試查找
        found = db.users.find_by_email("bob@example.com")
        
        assert found is not None
        assert found.name == "Bob"
    
    def test_update_user(self, db):
        # 創建用戶
        user = User(name="Charlie", email="charlie@example.com")
        created = db.users.create(user)
        
        # 更新用戶
        created.name = "Charles"
        updated = db.users.update(created)
        
        assert updated.name == "Charles"
    
    def test_delete_user(self, db):
        # 創建用戶
        user = User(name="David", email="david@example.com")
        created = db.users.create(user)
        
        # 刪除用戶
        db.users.delete(created.id)
        
        # 驗證已刪除
        found = db.users.find_by_id(created.id)
        assert found is None
```

## 測試驅動開發（TDD）

TDD 是一種軟體開發方法論，強調在編寫功能代碼之前先編寫測試。

### TDD 循環：紅-綠-重構

```
┌─────────────────────────────────────────────┐
│                  TDD 循環                    │
├─────────────────────────────────────────────┤
│                                             │
│   ┌─────────┐                                │
│   │  紅    │  編寫一個失敗的測試              │
│   │ (Red)  │  → 定義期望的行為               │
│   └────┬────┘                                │
│        │                                     │
│        ▼                                     │
│   ┌─────────┐                                │
│   │  綠    │  編寫最少的代碼讓測試通過       │
│   │ (Green)│  → 實現功能                     │
│   └────┬────┘                                │
│        │                                     │
│        ▼                                     │
│   ┌─────────┐                                │
│   │  重構   │  改善代碼結構而不破壞功能      │
│   │(Refactor)│ → 提升代碼品質                │
│   └────┬────┘                                │
│        │                                     │
│        └─────────────┬───────────────────────│
│                      ▼                       │
│              重複循環                         │
└─────────────────────────────────────────────┘
```

### TDD 範例

```python
# 情境：實現一個簡單的 Stack 資料結構

# 第一步：紅（編寫失敗的測試）
class TestStack:
    def test_push(self):
        stack = Stack()
        stack.push(1)
        assert stack.peek() == 1  # 會失敗，Stack 還沒實現
    
    def test_pop(self):
        stack = Stack()
        stack.push(1)
        assert stack.pop() == 1  # 會失敗
    
    def test_empty_pop(self):
        stack = Stack()
        with pytest.raises(Exception):
            stack.pop()  # 會失敗

# 第二步：綠（實現最少的代碼讓測試通過）
class Stack:
    def __init__(self):
        self.items = []
    
    def push(self, item):
        self.items.append(item)
    
    def pop(self):
        if not self.items:
            raise Exception("Stack is empty")
        return self.items.pop()
    
    def peek(self):
        if not self.items:
            raise Exception("Stack is empty")
        return self.items[-1]

# 第三步：重構（改善代碼）
class Stack:
    def __init__(self):
        self._items = []
    
    def push(self, item: int) -> None:
        self._items.append(item)
    
    def pop(self) -> int:
        if self.is_empty():
            raise EmptyStackException("Stack is empty")
        return self._items.pop()
    
    def peek(self) -> int:
        if self.is_empty():
            raise EmptyStackException("Stack is empty")
        return self._items[-1]
    
    def is_empty(self) -> bool:
        return len(self._items) == 0
    
    def size(self) -> int:
        return len(self._items)
```

## 行為驅動開發（BDD）

BDD 是 TDD 的延伸，強調使用自然語言描述系統行為，促進開發者、測試者和業務人員的溝通。

### Gherkin 語法

BDD 通常使用 Gherkin 語法編寫測試案例：

```gherkin
# feature/login.feature
Feature: User Login
  As a registered user
  I want to log in to the system
  So that I can access my account

  Background:
    Given the user "John" exists with password "secret123"

  Scenario: Successful login
    Given I am on the login page
    When I enter username "John"
    And I enter password "secret123"
    And I click the "Login" button
    Then I should be redirected to the dashboard
    And I should see "Welcome, John"

  Scenario: Invalid password
    Given I am on the login page
    When I enter username "John"
    And I enter password "wrongpassword"
    And I click the "Login" button
    Then I should see an error message "Invalid credentials"
    And I should remain on the login page

  Scenario: Empty username
    Given I am on the login page
    When I enter password "secret123"
    And I click the "Login" button
    Then I should see validation error "Username is required"
```

```python
# 使用 behave 實現 BDD 測試
# steps/login_steps.py
from behave import *

@given('the user "{name}" exists with password "{password}"')
def step_create_user(context, name, password):
    # 創建用戶
    User.objects.create(username=name, password=password)

@given('I am on the login page')
def step_visit_login(context):
    context.browser.visit('/login')

@when('I enter username "{username}"')
def step_enter_username(context, username):
    context.browser.fill('username', username)

@when('I enter password "{password}"')
def step_enter_password(context, password):
    context.browser.fill('password', password)

@when('I click the "{button}" button')
def step_click_button(context, button):
    context.browser.click_link_by_text(button)

@then('I should be redirected to the dashboard')
def step_check_dashboard(context):
    assert context.browser.url.endswith('/dashboard')

@then('I should see "{message}"')
def step_see_message(context, message):
    assert message in context.browser.html
```

## 測試覆蓋率

測試覆蓋率衡量測試對程式碼的覆蓋程度。

### 覆蓋率類型

- **行覆蓋率（Line Coverage）**：執行到的程式碼行數
- **分支覆蓋率（Branch Coverage）**：執行的分支（if/else）數量
- **函數覆蓋率（Function Coverage）**：執行的函數數量
- **路徑覆蓋率（Path Coverage）**：執行的程式路徑數量

```python
# pytest-cov 配置
# .coveragerc
[run]
source = src
omit = 
    */tests/*
    */migrations/*
    */__pycache__/*

[report]
exclude_lines =
    pragma: no cover
    def __repr__
    raise AssertionError
    raise NotImplementedError
    if __name__ == .__main__.:

[html]
directory = htmlcov
```

```bash
# 執行測試並生成覆蓋率報告
```

## .Mock 與測試替身

測試時常需要隔離被測試的單元，這時需要使用測試替身（Test Doubles）。

```python
import pytest
from unittest.mock import Mock, patch, MagicMock

# Mock 範例
def test_order_processing():
    # 創建 Mock 對象
    payment_gateway = Mock()
    
    # 配置 Mock 的行為
    payment_gateway.process_payment.return_value = True
    
    # 使用 Mock 測試
    order = Order(total=100)
    result = order.process(payment_gateway)
    
    assert result is True
    payment_gateway.process_payment.assert_called_once_with(100)

# Patch 範例
@patch('app.services.email_service.send_email')
def test_user_registration(mock_send_email):
    mock_send_email.return_value = True
    
    user_service = UserService()
    user = user_service.register('test@example.com', 'password123')
    
    assert user.email == 'test@example.com'
    mock_send_email.assert_called_once()

# MagicMock 用於複雜對象
def test_cache_operations():
    cache = MagicMock()
    
    cache.get.return_value = None
    cache.set.return_value = True
    
    # 使用 cache
    result = cache.get('key')
    if result is None:
        cache.set('key', 'value')
    
    cache.get.assert_called()
    cache.set.assert_called_once_with('key', 'value')
```

### AAA 模式（Arrange-Act-Assert）

```python
def test_transfer_funds_between_accounts(self):
    # Arrange: 準備測試資料
    source = Account(balance=1000)
    target = Account(balance=500)
    amount = 300
    
    # Act: 執行要測試的操作
    source.transfer_to(target, amount)
    
    # Assert: 驗證結果
    assert source.balance == 700
    assert target.balance == 800
```

### 測試隔離原則

```python
# 每個測試應該是獨立的，不依賴其他測試的執行順序
def test_database_insert():
    # 使用 fixture 確保每個測試有乾淨的資料庫
    db = create_test_database()
    
    # 測試邏輯
    
    # 測試結束後自動清理
    db.cleanup()

@pytest.fixture
def create_test_database():
    db = Database(':memory:')
    yield db
    db.close()
```
