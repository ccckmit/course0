# 設計模式

設計模式是軟體開發中經過驗證的解決方案，用於解決常見的設計問題。它不是可以直接複製的程式碼，而是描述在特定情況下如何解決問題的抽象指導方針。設計模式由四位作者 Erich Gamma、Richard Helm、Ralph Johnson 和 John Vlissides 在 1994 年的經典著作《Design Patterns: Elements of Reusable Object-Oriented Software》中首次系統化整理，因此被稱為「四人幫」（Gang of Four，GoF）設計模式。

## 設計模式的價值


## 創建型模式

創建型模式專注於物件的建立過程，幫助系統獨立於物件的創建、組合和表示方式。

### 1. Singleton（單例模式）

確保一個類別只有一個實例，並提供全域訪問點：

```python
class Singleton:
    """執行緒安全的 Singleton 實現"""
    _instance = None
    _lock = __import__('threading').Lock()
    
    def __new__(cls):
        if cls._instance is None:
            with cls._lock:
                if cls._instance is None:
                    cls._instance = super().__new__(cls)
        return cls._instance
    
    def __init__(self):
        if not hasattr(self, '_initialized'):
            self._data = {}
            self._initialized = True
    
    def set(self, key, value):
        self._data[key] = value
    
    def get(self, key):
        return self._data.get(key)

# 使用
s1 = Singleton()
s2 = Singleton()
print(s1 is s2)  # True

s1.set("name", "Alice")
print(s2.get("name"))  # Alice
```

```python
# Python 模組實現 Singleton
# my_singleton.py
class _Singleton:
    def method(self):
        pass

singleton = _Singleton()

# 使用時
# from my_singleton import singleton
```

### 2. Factory Method（工廠方法）

定義創建物件的介面，但讓子類決定要實例化的類別：

```python
from abc import ABC, abstractmethod

class Product(ABC):
    """產品介面"""
    @abstractmethod
    def operation(self) -> str:
        pass

class ConcreteProductA(Product):
    def operation(self) -> str:
        return "Product A"

class ConcreteProductB(Product):
    def operation(self) -> str:
        return "Product B"

class Creator(ABC):
    """創建者抽象類別"""
    @abstractmethod
    def factory_method(self) -> Product:
        pass
    
    def some_operation(self) -> str:
        product = self.factory_method()
        return f"Creator: {product.operation()}"

class ConcreteCreatorA(Creator):
    def factory_method(self) -> Product:
        return ConcreteProductA()

class ConcreteCreatorB(Creator):
    def factory_method(self) -> Product:
        return ConcreteProductB()

# 使用
creator = ConcreteCreatorA()
print(creator.some_operation())  # Creator: Product A
```

### 3. Abstract Factory（抽象工廠）

提供一個創建一系列相關物件的介面，無需指定具體類別：

```python
from abc import ABC, abstractmethod

# 抽象產品
class Button(ABC):
    @abstractmethod
    def render(self) -> str:
        pass

class Checkbox(ABC):
    @abstractmethod
    def render(self) -> str:
        pass

# 具體產品
class WindowsButton(Button):
    def render(self) -> str:
        return "<Windows Button>"

class MacButton(Button):
    def render(self) -> str:
        return "[Mac Button]"

class WindowsCheckbox(Checkbox):
    def render(self) -> str:
        return "<Windows Checkbox>"

class MacCheckbox(Checkbox):
    def render(self) -> str:
        return "[Mac Checkbox]"

# 抽象工廠
class UIFactory(ABC):
    @abstractmethod
    def create_button(self) -> Button:
        pass
    
    @abstractmethod
    def create_checkbox(self) -> Checkbox:
        pass

# 具體工廠
class WindowsFactory(UIFactory):
    def create_button(self) -> Button:
        return WindowsButton()
    
    def create_checkbox(self) -> Checkbox:
        return WindowsCheckbox()

class MacFactory(UIFactory):
    def create_button(self) -> Button:
        return MacButton()
    
    def create_checkbox(self) -> Checkbox:
        return MacCheckbox()

# 使用
def create_ui(factory: UIFactory):
    button = factory.create_button()
    checkbox = factory.create_checkbox()
    return button.render(), checkbox.render()

print(create_ui(WindowsFactory()))  # ('<Windows Button>', '<Windows Checkbox>')
print(create_ui(MacFactory()))      # ('[Mac Button]', '[Mac Checkbox]')
```

### 4. Builder（建造者）

將複雜物件的構建與其表示分離：

```python
class Pizza:
    def __init__(self):
        self.dough = None
        self.sauce = None
        self.topping = None
    
    def __str__(self):
        return f"Pizza(dough={self.dough}, sauce={self.sauce}, topping={self.topping})"

class PizzaBuilder:
    def __init__(self):
        self.pizza = Pizza()
    
    def set_dough(self, dough):
        self.pizza.dough = dough
        return self
    
    def set_sauce(self, sauce):
        self.pizza.sauce = sauce
        return self
    
    def set_topping(self, topping):
        self.pizza.topping = topping
        return self
    
    def build(self):
        return self.pizza

class Director:
    def __init__(self, builder: PizzaBuilder):
        self._builder = builder
    
    def make_margherita(self):
        return (self._builder
                .set_dough("thin")
                .set_sauce("tomato")
                .set_topping("mozzarella")
                .build())
    
    def make_pepperoni(self):
        return (self._builder
                .set_dough("thick")
                .set_sauce("spicy")
                .set_topping("pepperoni")
                .build())

# 使用
builder = PizzaBuilder()
director = Director(builder)
print(director.make_margherita())  # Pizza(dough=thin, sauce=tomato, topping=mozzarella)
print(director.make_pepperoni())  # Pizza(dough=thick, sauce=spicy, topping=pepperoni)
```

### 5. Prototype（原型）

通過複製現有物件來創建新物件：

```python
import copy

class Prototype(ABC):
    @abstractmethod
    def clone(self):
        pass

class Document(Prototype):
    def __init__(self, title: str, content: str, metadata: dict = None):
        self.title = title
        self.content = content
        self.metadata = metadata or {}
    
    def clone(self):
        # 深拷貝
        return Document(
            self.title,
            self.content,
            copy.deepcopy(self.metadata)
        )
    
    def __str__(self):
        return f"Document({self.title}, {self.content[:20]}...)"

# 使用
doc = Document("Report", "This is the content...", {"author": "John", "version": 1})
print(doc)

cloned = doc.clone()
cloned.title = "Report (Copy)"
cloned.metadata["version"] = 2

print(doc)       # Document(Report, This is the content...)
print(cloned)    # Document(Report (Copy), This is the content...)
print(doc.metadata)      # {'author': 'John', 'version': 1}
print(cloned.metadata)   # {'author': 'John', 'version': 2}
```

## 結構型模式

結構型模式專注於物件的組合，以形成更大的結構。

### 1. Adapter（轉接器）

將類別的介面轉換為客戶端期望的另一個介面：

```python
# 舊系統
class OldPaymentSystem:
    def process_payment(self, amount: float, currency: str) -> str:
        return f"Old: Processed {amount} {currency}"

# 新系統
class NewPaymentAPI:
    def pay(self, amount: int, currency: str) -> dict:
        return {"status": "success", "amount": amount, "currency": currency}

# Adapter
class PaymentAdapter:
    def __init__(self, new_api: NewPaymentAPI):
        self._api = new_api
    
    def process_payment(self, amount: float, currency: str) -> str:
        # 轉換舊格式到新格式
        amount_cents = int(amount * 100)
        result = self._api.pay(amount_cents, currency.upper())
        return f"Processed: {amount} {currency} (status: {result['status']})"

# 使用
old = OldPaymentSystem()
new_api = NewPaymentAPI()
adapter = PaymentAdapter(new_api)

print(old.process_payment(99.99, "USD"))  # Old: Processed 99.99 USD
print(adapter.process_payment(99.99, "USD"))  # Processed: 99.99 USD (status: success)
```

### 2. Decorator（裝飾器）

動態地為物件添加額外職責：

```python
class Coffee:
    def cost(self) -> float:
        return 5.0
    
    def description(self) -> str:
        return "Coffee"

class CoffeeDecorator(Coffee):
    def __init__(self, coffee: Coffee):
        self._coffee = coffee
    
    def cost(self) -> float:
        return self._coffee.cost()
    
    def description(self) -> str:
        return self._coffee.description()

class Milk(CoffeeDecorator):
    def cost(self) -> float:
        return self._coffee.cost() + 1.0
    
    def description(self) -> str:
        return self._coffee.description() + ", Milk"

class Sugar(CoffeeDecorator):
    def cost(self) -> float:
        return self._coffee.cost() + 0.5
    
    def description(self) -> str:
        return self._coffee.description() + ", Sugar"

class WhippedCream(CoffeeDecorator):
    def cost(self) -> float:
        return self._coffee.cost() + 2.0
    
    def description(self) -> str:
        return self._coffee.description() + ", Whipped Cream"

# 使用
coffee = Coffee()
print(f"{coffee.description()}: ${coffee.cost()}")  # Coffee: $5.0

coffee_with_milk = Milk(coffee)
print(f"{coffee_with_milk.description()}: ${coffee_with_milk.cost()}")  # Coffee, Milk: $6.0

```

### 3. Facade（外觀）

提供統一的高級介面，簡化複雜子系統的使用：

```python
class CPU:
    def freeze(self): print("CPU: Freezing")
    def jump(self, position): print(f"CPU: Jumping to {position}")
    def execute(self): print("CPU: Executing")

class Memory:
    def load(self, position, data): print(f"Memory: Loading {data} at {position}")

class HardDrive:
    def read(self, sector, size): 
        print(f"HardDrive: Reading {size} bytes from sector {sector}")
        return "boot data"

class ComputerFacade:
    def __init__(self):
        self.cpu = CPU()
        self.memory = Memory()
        self.hard_drive = HardDrive()
    
    def start(self):
        print("Starting computer...")
        self.cpu.freeze()
        boot_data = self.hard_drive.read(0, 1024)
        self.memory.load(0, boot_data)
        self.cpu.jump(0)
        self.cpu.execute()
        print("Computer started!")

# 使用
computer = ComputerFacade()
computer.start()
# Starting computer...
# CPU: Freezing
# HardDrive: Reading 1024 bytes from sector 0
# Memory: Loading boot data at 0
# CPU: Jumping to 0
# CPU: Executing
# Computer started!
```

### 4. Proxy（代理）

為其他物件提供一個代理以控制對它的訪問：

```python
from abc import ABC, abstractmethod
import time

class Subject(ABC):
    @abstractmethod
    def request(self):
        pass

class RealSubject(Subject):
    def request(self):
        print("RealSubject: Handling request...")
        time.sleep(1)
        return "Result"

class Proxy(Subject):
    def __init__(self, real_subject: RealSubject):
        self._real_subject = real_subject
        self._cache = None
    
    def request(self) -> str:
        if self._cache is None:
            print("Proxy: Checking cache... miss")
            self._cache = self._real_subject.request()
        else:
            print("Proxy: Checking cache... hit")
        return self._cache

# 使用
real = RealSubject()
proxy = RealSubject()

print("First call (cache miss):")
start = time.time()
proxy.request()
print(f"Time: {time.time() - start:.2f}s")

print("\nSecond call (cache hit):")
start = time.time()
proxy.request()
print(f"Time: {time.time() - start:.2f}s")
```

## 行為型模式

行為型模式專注於物件之間的通訊和職責分配。

### 1. Observer（觀察者）

定義物件間的一對多依賴關係，當物件狀態改變時，所有依賴它的物件都會被通知：

```python
from abc import ABC, abstractmethod

class Observer(ABC):
    @abstractmethod
    def update(self, message: str):
        pass

class Subject:
    def __init__(self):
        self._observers: list[Observer] = []
    
    def attach(self, observer: Observer):
        self._observers.append(observer)
    
    def detach(self, observer: Observer):
        self._observers.remove(observer)
    
    def notify(self, message: str):
        for observer in self._observers:
            observer.update(message)

class NewsAgency(Subject):
    def __init__(self):
        super().__init__()
        self._latest_news = None
    
    def set_news(self, news: str):
        self._latest_news = news
        self.notify(news)

class NewsChannel(Observer):
    def __init__(self, name: str):
        self.name = name
    
    def update(self, message: str):
        print(f"{self.name} received: {message}")

# 使用
agency = NewsAgency()

channel1 = NewsChannel("CNN")
channel2 = NewsChannel("BBC")

agency.attach(channel1)
agency.attach(channel2)

agency.set_news("Breaking: AI model achieves human-level performance!")
# CNN received: Breaking: AI model achieves human-level performance!
# BBC received: Breaking: AI model achieves human-level performance!

agency.detach(channel1)
agency.set_news("Update: Markets respond to AI news")
# BBC received: Update: Markets respond to AI news
```

### 2. Strategy（策略）

定義一系列演算法，封裝每一個，使它們可以相互替換：

```python
from abc import ABC, abstractmethod

class SortStrategy(ABC):
    @abstractmethod
    def sort(self, data: list) -> list:
        pass

class BubbleSort(SortStrategy):
    def sort(self, data: list) -> list:
        arr = data.copy()
        n = len(arr)
        for i in range(n):
            for j in range(0, n-i-1):
                if arr[j] > arr[j+1]:
                    arr[j], arr[j+1] = arr[j+1], arr[j]
        return arr

class QuickSort(SortStrategy):
    def sort(self, data: list) -> list:
        if len(data) <= 1:
            return data
        pivot = data[len(data) // 2]
        left = [x for x in data if x < pivot]
        middle = [x for x in data if x == pivot]
        right = [x for x in data if x > pivot]
        return self.sort(left) + middle + self.sort(right)

class Sorter:
    def __init__(self, strategy: SortStrategy):
        self._strategy = strategy
    
    def set_strategy(self, strategy: SortStrategy):
        self._strategy = strategy
    
    def sort(self, data: list) -> list:
        return self._strategy.sort(data)

# 使用
data = [64, 34, 25, 12, 22, 11, 90]

sorter = Sorter(BubbleSort())
print("Bubble Sort:", sorter.sort(data))

sorter.set_strategy(QuickSort())
print("Quick Sort:", sorter.sort(data))
```

### 3. Command（命令）

將請求封裝為物件，從而允許參數化、排隊或記錄請求：

```python
from abc import ABC, abstractmethod
from datetime import datetime

class Command(ABC):
    @abstractmethod
    def execute(self):
        pass
    
    @abstractmethod
    def undo(self):
        pass

class Light:
    def on(self): print("Light is ON")
    def off(self): print("Light is OFF")

class LightOnCommand(Command):
    def __init__(self, light: Light):
        self._light = light
    
    def execute(self):
        self._light.on()
    
    def undo(self):
        self._light.off()

class LightOffCommand(Command):
    def __init__(self, light: Light):
        self._light = light
    
    def execute(self):
        self._light.off()
    
    def undo(self):
        self._light.on()

class RemoteControl:
    def __init__(self):
        self._history = []
    
    def execute_command(self, command: Command):
        command.execute()
        self._history.append(command)
    
    def undo_last(self):
        if self._history:
            command = self._history.pop()
            command.undo()

# 使用
light = Light()
remote = RemoteControl()

remote.execute_command(LightOnCommand(light))  # Light is ON
remote.execute_command(LightOffCommand(light))  # Light is OFF

print("Undo...")
remote.undo_last()  # Light is ON
remote.undo_last()  # Light is OFF
```

### 4. Template Method（模板方法）

定義演算法的骨架，將一些步驟延遲到子類：

```python
from abc import ABC, abstractmethod

class DataMiner(ABC):
    """模板方法：數據開採流程"""
    
    def mine(self, path: str) -> str:
        file = self.open_file(path)
        raw_data = self.extract_data(file)
        data = self.parse_data(raw_data)
        analysis = self.analyze_data(data)
        self.send_report(analysis)
        return analysis
    
    def open_file(self, path: str):
        print(f"Opening file: {path}")
        return path
    
    @abstractmethod
    def extract_data(self, file) -> str:
        pass
    
    @abstractmethod
    def parse_data(self, raw_data: str) -> list:
        pass
    
    def analyze_data(self, data: list) -> str:
        return f"Analyzed {len(data)} records"
    
    def send_report(self, analysis: str):
        print(f"Report: {analysis}")

class PDFDataMiner(DataMiner):
    def extract_data(self, file) -> str:
        return "PDF raw data"
    
    def parse_data(self, raw_data: str) -> list:
        print("Parsing PDF...")
        return ["record1", "record2"]

class CSVDataMiner(DataMiner):
    def extract_data(self, file) -> str:
        return "CSV raw data"
    
    def parse_data(self, raw_data: str) -> list:
        print("Parsing CSV...")
        return ["row1", "row2", "row3"]

# 使用
print("=== PDF Mining ===")
pdf_miner = PDFDataMiner()
result = pdf_miner.mine("report.pdf")

print("\n=== CSV Mining ===")
csv_miner = CSVDataMiner()
result = csv_miner.mine("data.csv")
```

### 5. State（狀態）

允許物件在內部狀態改變時改變其行為：

```python
from abc import ABC, abstractmethod

class State(ABC):
    @abstractmethod
    def insert_coin(self, machine):
        pass
    
    @abstractmethod
    def eject_coin(self, machine):
        pass
    
    @abstractmethod
    def turn_crank(self, machine):
        pass

class VendingMachine:
    def __init__(self):
        self._state = NoCoinState()
    
    def set_state(self, state: State):
        self._state = state
    
    def insert_coin(self):
        self._state.insert_coin(self)
    
    def eject_coin(self):
        self._state.eject_coin(self)
    
    def turn_crank(self):
        self._state.turn_crank(self)
    
    def dispense(self):
        print("Dispensing product!")

class NoCoinState(State):
    def insert_coin(self, machine):
        print("Coin inserted")
        machine.set_state(HasCoinState())
    
    def eject_coin(self):
        print("No coin to return")
    
    def turn_crank(self):
        print("Insert coin first")

class HasCoinState(State):
    def insert_coin(self):
        print("Coin already inserted")
    
    def eject_coin(self):
        print("Coin returned")
        machine.set_state(NoCoinState())
    
    def turn_crank(self):
        print("Crank turned")
        machine.set_state(SoldState())
        machine.dispense()

class SoldState(State):
    def insert_coin(self):
        print("Please wait")
    
    def eject_coin(self):
        print("Already turned crank")
    
    def turn_crank(self):
        print("Already turned crank")

# 使用
machine = VendingMachine()
machine.insert_coin()  # Coin inserted
machine.turn_crank()  # Crank turned, Dispensing product!
machine.insert_coin()  # Insert coin first
```

### 6. Iterator（迭代器）

提供一種方法順序訪問集合元素而不暴露底層表示：

```python
from abc import ABC, abstractmethod

class Iterator(ABC):
    @abstractmethod
    def has_next(self) -> bool:
        pass
    
    @abstractmethod
    def next(self):
        pass

class ArrayIterator(Iterator):
    def __init__(self, data: list):
        self._data = data
        self._index = 0
    
    def has_next(self) -> bool:
        return self._index < len(self._data)
    
    def next(self):
        if self.has_next():
            item = self._data[self._index]
            self._index += 1
            return item
        return None

class TreeNode:
    def __init__(self, value, left=None, right=None):
        self.value = value
        self.left = left
        self.right = right

class TreeIterator(Iterator):
    def __init__(self, root: TreeNode):
        self._stack = []
        self._current = root
    
    def has_next(self) -> bool:
        return self._current is not None or len(self._stack) > 0
    
    def next(self):
        while self._current is not None:
            self._stack.append(self._current)
            self._current = self._current.left
        
        if len(self._stack) == 0:
            return None
        
        node = self._stack.pop()
        self._current = node.right
        return node.value

# 使用
# Array Iterator
data = [1, 2, 3, 4, 5]
iterator = ArrayIterator(data)
while iterator.has_next():
    print(iterator.next(), end=" ")  # 1 2 3 4 5

print("\n")

# Tree Iterator
#       4
#      / \
#     2   6
#    / \ / \
#   1  3 5  7
root = TreeNode(4, 
                TreeNode(2, TreeNode(1), TreeNode(3)),
                TreeNode(6, TreeNode(5), TreeNode(7)))

tree_iter = TreeIterator(root)
while tree_iter.has_next():
    print(tree_iter.next(), end=" ")  # 1 2 3 4 5 6 7
```

## 設計模式的優缺點

### 優點

- **經過驗證**：來自業界最佳實踐
- **提高可維護性**：促進清晰的設計
- **促進重用**：提供通用的解決方案
- **改善溝通**：提供共享的術語

### 缺點

- **過度設計**：不必要的複雜性
- **學習曲線**：需要理解多種模式
- **可能過度抽象**：降低可讀性
