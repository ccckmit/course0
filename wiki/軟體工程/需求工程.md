# 需求工程：系統開發的起點

## 概述

需求工程是軟體開發生命週期的起始階段，決定了軟體系統應該做什麼。良好的需求工程能確保開發團隊理解用戶的真正需求，避免代價昂貴的需求變更。需求工程包括需求獲取、需求分析、需求規格說明和需求驗證四個主要活動。

## 需求類型

```markdown
# 功能需求 (Functional Requirements)
- 系統應該做什麼
- 示例：用戶應該能夠登入系統

# 非功能需求 (Non-Functional Requirements)
- 系統的質量屬性
- 性能：系統應該在 3 秒內響應
- 安全：用戶密碼應該加密存儲
- 可用性：系統應該易於使用
- 可靠性：系統應該 99.9% 可用

# 業務需求 (Business Requirements)
- 組織的高層目標
- 示例：公司希望提高客戶滿意度

# 用戶需求 (User Requirements)
- 用戶希望能夠完成的任務
- 示例：作為管理員，我想管理員工帳戶
```

## 用戶故事

```markdown
# 用戶故事模板
作為 <角色>，我希望 <功能>，以便 <收益>

# 示例
作為一名線上購物者，我希望能夠使用優惠券，以便節省購物費用。

# 接受標準
假設 優惠券代碼存在且有效
當 我在結帳頁面輸入優惠券代碼
然後 系統應該扣除相應金額
而且 顯示最終價格

# 估算 (Planning Poker)
- XS: 0.5 天
- S: 1 天
- M: 3 天
- L: 5 天
- XL: 2 週
```

## 總結

需求工程是軟體開發成功的關鍵。高質量的需求應該是完整、一致、可測試和可追蹤的。使用用戶故事和接受標準能幫助團隊更好地理解需求。
