# 雲端原生 (Cloud-Native)

## 概述

雲端原生是一種軟體開發方法論，利用雲端運算的優勢——彈性、可擴展性、自動化——來構建和執行應用程式。Cloud Native Computing Foundation (CNCF) 定義了雲端原生的核心技術。

## 核心原則

1. **容器化 (Containerization)**：將應用及其依賴封裝為容器
2. **微服務架構 (Microservices)**：將大型應用拆分成小型服務
3. **動態管理 (Dynamic Orchestration)**：自動調度和管理容器
4. **韌性設計 (Resilience)**：假設故障會發生，設計容錯機制

## 十二要素應用 (The Twelve-Factor App)

雲端原生應用程式的設計原則：

| 因素 | 說明 | 範例 |
|------|------|------|
| I. 基準程式碼 | 單一程式碼庫，多個部署 | Git repo |
| II. 依賴 | 明確宣告依賴 | requirements.txt |
| III. 設定 | 環境變數存放設定 | DATABASE_URL |
| IV. 後端服務 | 把後端當附屬資源 | 資料庫、API |
| V. 建構、發布、執行 | 嚴格分離階段 | CI/CD Pipeline |
| VI. 處理序 | 無狀態、共享nothing | Redis session |
| VII. 連接埠綁定 | 自包含服務 | 8080:3000 |
| VIII. 併發 | 水平擴展 | replicas: 3 |
| IX. 易拋棄 | 快速啟動、優雅關閉 | SIGTERM 處理 |
| X. 開發/生產一致 | 環境一致性 | Docker |
| XI. 日誌 | 事件串流作為日誌 | stdout |
| XII. 管理程序 | 一次性管理工作 | Admin scripts |

## 雲端原生模式

### .sidecar 模式
為每個主要容器附加輔助容器處理橫切關注點：

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-app
spec:
  containers:
  - name: main-app
    image: my-app:latest
  - name: proxy
    image: envoy-proxy:latest
  - name: logger
    image: fluentd:latest
```

### 領導者選舉模式
分散式系統中選舉領導者處理協調整性任務。

### 熔斷器模式 (Circuit Breaker)
```python
import circuitbreaker

@circuitbreaker(failure_threshold=5, recovery_timeout=30)
def call_external_service():
    response = requests.get("http://external-api/data")
    return response.json()
```

## CNCF 景觀

主要專案：
- **Kubernetes**：容器編排
- **Prometheus**：監控
- **Grafana**：視覺化
- **Jaeger**：分散式追蹤
- **Envoy**：服務代理
- **Helm**：套件管理

## 實踐建議

1. **從小處著手**：先從單一服務容器化開始
2. **建立 SRE 文化**：強調可靠性目標 (SLO)
3. **observability first**：從第一天就考慮監控和追蹤
4. **漸進式遷移**：strangler fig pattern
