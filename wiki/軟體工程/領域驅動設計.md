# 領域驅動設計：複雜系統的設計方法

## 概述

領域驅動設計（Domain-Driven Design, DDD）是由 Eric Evans 提出的軟體設計方法學，強調軟體應該緊密反映業務領域概念。DDD 適用於處理複雜業務邏輯的系統，通過建立通用語言、識別限界上下文和設計聚合等戰術模式，幫助團隊構建高內聚、低耦合的軟體架構。

## 核心概念

```python
# 實體 (Entity) - 具有唯一標識的對象
class Order:
    def __init__(self, order_id, customer):
        self._order_id = order_id
        self._customer = customer
        self._items = []
        self._status = 'pending'
    
    @property
    def order_id(self):
        return self._order_id
    
    def add_item(self, product, quantity):
        if self._status != 'pending':
            raise ValueError("Cannot modify completed order")
        self._items.append(OrderItem(product, quantity))
    
    def total(self):
        return sum(item.subtotal() for item in self._items)

# 值對象 (Value Object) - 由其屬性定義的不可變對象
class Money:
    def __init__(self, amount, currency='USD'):
        self._amount = amount
        self._currency = currency
    
    @property
    def amount(self):
        return self._amount
    
    def __eq__(self, other):
        return self._amount == other._amount and self._currency == other._currency
    
    def __add__(self, other):
        if self._currency != other._currency:
            raise ValueError("Cannot add different currencies")
        return Money(self._amount + other._amount, self._currency)

# 聚合 (Aggregate) - 一組相關對象的集合
class OrderAggregate:
    def __init__(self, order_id, customer):
        self._order = Order(order_id, customer)
        self._invoice = None  # 延遲創建
    
    def add_item(self, product, quantity):
        self._order.add_item(product, quantity)
    
    def get_total(self):
        return self._order.total()
    
    def confirm(self):
        """工廠方法：創建 Invoice"""
        if not self._invoice:
            self._invoice = Invoice.create(self._order)
        return self._invoice

# 領域服務 (Domain Service) - 不屬於任何實體的操作
class PricingService:
    def calculate_discount(self, customer, order):
        if customer.loyalty_tier == 'gold':
            return order.total() * 0.1
        return Money(0)
```

"""
電商系統的限界上下文：
- 產品目錄上下文

- 支付上下文
- 庫存上下文
- 物流上下文

每個上下文有：
- 自己的通用語言
- 明確的邊界
- 自主的數據存儲
"""

# 上下文映射
class ContextMapping:
    """上下文之間的關係"""
    
    # 共享內核 (Shared Kernel)
    # 兩個團隊共享一部分模型
    
    # 客戶-供應商 (Customer-Supplier)
    # 上游團隊提供下游團隊需要的接口
    
    # 順應者 (Conformist)
    # 下游團隊適配上游客服
    
    # 防腐層 (Anti-Corruption Layer)
    # 在兩個上下文之間轉換，翻譯接口
    pass

class AntiCorruptionLayer:
    """防腐層 - 隔離外部系統"""
    
    def __init__(self, external_adapter):
        self._adapter = external_adapter
    
    def to_internal(self, external_data):
        """將外部數據轉換為內部模型"""
        return InternalOrder(
            id=external_data['orderId'],
            items=[self._translate_item(i) for i in external_data['items']]
        )
    
    def _translate_item(self, item):
        return InternalItem(
            product_id=item['sku'],
            quantity=item['qty']
        )

## 總結

DDD 通過建立通用語言、識別限界上下文和設計聚合等模式，幫助團隊處理複雜業務邏輯。DDD 不是銀彈，適用於業務複雜度高的系統。
