# 災難復原：確保業務連續性

## 概述

災難復原（Disaster Recovery, DR）是一組策略和過程，用於在災難發生後恢復 IT 基礎設施和數據。災難可能包括自然災害、硬體故障、網路攻擊或人為錯誤。

## DR 策略

```markdown
# RPO (Recovery Point Objective)
- 可接受的數據丟失量
- 示例：RPO = 1 小時 = 最多丟失 1 小時的數據

# RTO (Recovery Time Objective)
- 可接受的系統停機時間
- 示例：RTO = 4 小時 = 系統應在 4 小時內恢復

# DR 策略分級
- Tier 0: 無恢復能力
- Tier 1: 備份和恢復
- Tier 2: 暖備份（部分系統）
- Tier 3: 熱備份（大部分系統）
- Tier 4: 高可用性

# 備份類型
- 完整備份：備份所有數據
- 增量備份：只備份自上次備份以來變更的數據
- 差異備份：只備份自上次完整備份以來變更的數據
```

## 備份策略實現

```python
import boto3
from datetime import datetime

class BackupManager:
    """AWS 備份管理器"""
    
    def __init__(self):
        self.s3 = boto3.client('s3')
        self.rds = boto3.client('rds')
    
    def create_rds_snapshot(self, db_instance_id):
        """創建 RDS 數據庫快照"""
        timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
        snapshot_id = f'{db_instance_id}-snapshot-{timestamp}'
        
        response = self.rds.create_db_snapshot(
            DBSnapshotIdentifier=snapshot_id,
            DBInstanceIdentifier=db_instance_id
        )
        return response['DBSnapshot']['DBSnapshotArn']
    
    def backup_to_s3(self, bucket, local_path, key_prefix):
        """備份文件到 S3"""
        timestamp = datetime.now().strftime('%Y-%m-%d')
        key = f'{key_prefix}/{timestamp}/backup.tar.gz'
        
        self.s3.upload_file(local_path, bucket, key)
        return key
    
    def restore_from_snapshot(self, snapshot_id, target_instance_id):
        """從快照恢復 RDS 實例"""
        self.rds.restore_db_instance_from_db_snapshot(
            DBInstanceIdentifier=target_instance_id,
            DBSnapshotIdentifier=snapshot_id
        )
```

## 總結

災難復原是業務連續性計劃的核心部分。定義清晰的 RPO 和 RTO，選擇合適的備份策略，並定期測試災難恢復流程。
