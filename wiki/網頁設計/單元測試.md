# 單元測試：自動化測試實踐

## 概述

單元測試是軟體品質保證的基石，它針對軟體中最小的可測試單元（如函數、方法或類）進行驗證，確保每個單元按預期工作。高品質的單元測試能夠及早發現錯誤、防止回歸問題、提供文件作用並支援重構。現代前端開發中，常用的單元測試框架包括 Jest（Facebook 開發，React 生態首選）和 Vitest（Vite 原生支援，現代 JavaScript 框架的好選擇）。

單元測試的核心原則是：**隔離性**（每個測試獨立運行，不依賴其他測試結果）、**可重複性**（相同測試每次運行應得到相同結果）、**快速執行**（單元測試應能在毫秒級完成）、以及**自我驗證**（測試應能自動判斷通過或失敗）。遵循這些原則能確保測試套件的可靠性和可維護性。

## Jest 框架詳解

### 基礎配置與結構

Jest 是 Facebook 開發的 JavaScript 測試框架，以其零配置、快照測試和即時反饋的特性著稱。它內置了測試運行器、斷言庫、模擬功能，並支援 TypeScript、JSX 和 Vue 組件測試。

### 斷言與匹配器

Jest 的斷言庫（Expect）提供了豐富的匹配器來驗證各種條件。掌握這些匹配器是撰寫有效測試的關鍵。

```javascript
// tests/example.test.js
describe('Jest 斷言示例', () => {
  
  // 通用匹配器
  test('基本斷言', () => {
    expect(2 + 2).toBe(4);           // 嚴格相等（基本類型）
    expect({ name: 'test' }).toEqual({ name: 'test' }); // 深比較（引用類型）
    expect(null).toBeNull();          // null
    expect(undefined).toBeUndefined(); // undefined
    expect(1).toBeDefined();          // 已定義
    expect(true).toBeTruthy();        // 真值
    expect(0).toBeFalsy();            // 假值
  });
  
  // 數值匹配器
  test('數值斷言', () => {
    expect(0.1 + 0.2).toBeCloseTo(0.3); // 浮點數比較
    expect(10).toBeGreaterThan(5);      // 大於
    expect(10).toBeGreaterThanOrEqual(10); // 大於等於
    expect(5).toBeLessThan(10);         // 小於
    expect(5).toBeLessThanOrEqual(5);  // 小於等於
  });
  
  // 字串匹配器
  test('字串匹配', () => {
    expect('Hello World').toMatch(/World/);      // 正則表達式匹配
    expect('Hello World').toContain('Hello');    // 包含子串
    expect(['a', 'b', 'c']).toContain('a');      // 數組包含
    expect('hello').toHaveLength(5);             // 長度匹配
  });
  
  // 數組和可迭代對象
  test('數組匹配', () => {
    const users = [
      { name: 'Alice', age: 25 },
      { name: 'Bob', age: 30 },
    ];
    expect(users).toContainEqual({ name: 'Alice', age: 25 });
    expect(users).toHaveLength(2);
    expect([1, 2, 3]).toEqual(
      expect.arrayContaining([1, 3]) // 數組包含特定元素
    );
  });
  
  // 對象匹配
  test('對象匹配', () => {
    const user = { name: 'Alice', age: 25, email: 'alice@example.com' };
    expect(user).toMatchObject({
      name: 'Alice',
      age: 25,
    }); // 對象包含指定屬性
    expect(user).toHaveProperty('name', 'Alice');
    expect(user).toEqual(
      expect.objectContaining({
        name: expect.stringContaining('Ali'),
      })
    );
  });
  
  // 例外處理
  test('異常斷言', () => {
    function throwError() {
      throw new Error('Test error');
    }
    expect(throwError).toThrow();           // 拋出任何異常
    expect(throwError).toThrow(Error);     // 拋出特定類型
    expect(throwError).toThrow('Test');    // 錯誤信息包含文字
    expect(throwError).toThrow(/test/i);   // 正則表達式匹配
  });
  
  // 非匹配（否定）
  test('否定匹配', () => {
    expect(1).not.toBe(2);
    expect('hello').not.toMatch(/world/);
    expect([1, 2, 3]).not.toContain(4);
  });
  
  // 異步斷言
  test('Promise 斷言', async () => {
    await expect(Promise.resolve(4)).resolves.toBe(4);
    await expect(Promise.reject(new Error('Error'))).rejects.toThrow();
    
    // 或使用 async/await
    const result = await fetchData();
    expect(result).toBeDefined();
  });
  
  // 合成匹配器
  test('自定義斷言', () => {
    expect.extend({
      toBeWithinRange(received, min, max) {
        const pass = received >= min && received <= max;
        return {
          pass,
          message: () => `expected ${received} ${pass ? 'not ' : ''}to be within range ${min} - ${max}`,
        };
      },
    });
    expect(100).toBeWithinRange(90, 110);
  });
});
```

### 模擬函數（Mock Functions）

模擬函數是隔離測試和控制函數行為的核心工具。Jest 提供了完整的模擬機制，包括手動模擬、模擬函數和模組模擬。

```javascript
// src/calculator.js
export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

export async function fetchData(url) {
  const response = await fetch(url);
  return response.json();
}

// src/api.js
import axios from 'axios';

export async function getUser(id) {
  const response = await axios.get(`/api/users/${id}`);
  return response.data;
}

describe('模擬函數測試', () => {
  test('基本模擬', () => {
    const mockFn = jest.fn();
    
    mockFn('hello');
    mockFn('world');
    
    // 調用次數
    expect(mockFn).toHaveBeenCalledTimes(2);
    // 特定參數調用
    expect(mockFn).toHaveBeenCalledWith('hello');
    expect(mockFn).toHaveBeenCalledWith('world');
    // 最近一次調用
    expect(mockFn).toHaveBeenLastCalledWith('world');
  });
  
  test('模擬返回值', () => {
    const mockFn = jest.fn();
    mockFn.mockReturnValue(10);
    mockFn.mockResolvedValue(20);
    mockFn.mockRejectedValue(new Error('Error'));
    
    expect(mockFn()).toBe(10);
    await expect(mockFn()).resolves.toBe(20);
    await expect(mockFn()).rejects.toThrow('Error');
  });
  
  test('模擬實現', () => {
    const mockFn = jest.fn((a, b) => a + b);
    expect(mockFn(1, 2)).toBe(3);
  });
  
  test('部分模擬', () => {
    const each = jest.fn();
    [1, 2, 3].forEach(each);
    expect(each).toHaveBeenCalledTimes(3);
  });

    jest.useRealTimers();

    jest.runAllTimers();
    
    expect(callback).toHaveBeenCalled();
    expect(callback).toHaveBeenCalledAfter(900);

  test('模擬 setInterval', () => {
    const callback = jest.fn();
    
    const id = setInterval(callback, 500);
    jest.advanceTimersByTime(1500);

    jest.useFakeTimers();
    
    const promise = new Promise(resolve => setTimeout(() => resolve(42), 1000));

    const result = await promise;
    
    expect(result).toBe(42);
```

## Vitest 框架詳解

### 基礎配置

Vitest 是 Vite 的原生測試框架，與 Vite 共享相同的配置格式和插件系統，提供極快的測試運行體驗。

```typescript

    setupFiles: ['./tests/setup.ts'],
    
    // 測試檔案
    include: [

    ],
    exclude: ['**/node_modules/**', '**/dist/**'],
    
    // Coverage 配置
    coverage: {
      provider: 'v8', // 或 'istanbul'
      reporter: ['text', 'json', 'html', 'lcov'],
      reportsDirectory: './coverage',
      include: ['src/**/*.{js,ts}'],
      exclude: ['**/*.d.ts', '**/*.config.*', '**/node_modules/**'],

        functions: 80,

      json: './coverage/test-results.json',

    testTimeout: 10000,

    expect: {
      timeout: 5000,
    },
    
    // 序列或並行
    sequence: {
      hooks: 'list', // 或 'always' 或 'never'
    },
    
    // 類型檢查

      enabled: true,
      tsconfig: './tsconfig.json',
    },
    
    // 快照
```

## Vue 組件測試

### Vue Test Utils 使用

```typescript
// tests/components/UserCard.spec.ts
import { describe, it, expect, vi } from 'vitest';
import { mount, shallowMount, VueWrapper } from '@vue/test-utils';
import { h } from 'vue';
import UserCard from '@/components/UserCard.vue';

describe('UserCard', () => {
  const defaultProps = {
    user: {
      id: 1,
      name: 'Alice',
      email: 'alice@example.com',
      avatar: 'https://example.com/avatar.jpg',
    },
    editable: false,
  };
  
  it('渲染用戶信息', () => {
    const wrapper = mount(UserCard, {
      props: defaultProps,
    });
    
    expect(wrapper.find('.user-name').text()).toBe('Alice');

    expect(wrapper.emitted('edit')).toBeTruthy();
    expect(wrapper.emitted('edit')![0]).toEqual([{ id: 1 }]);

    const wrapper = mount(UserCard, {
      props: defaultProps,
      slots: {
        footer: h('div', { class: 'custom-footer' }, 'Custom Footer Content'),
      },
    });
    
    expect(wrapper.find('.custom-footer').exists()).toBe(true);

  it('過濾敏感信息', () => {

    expect(wrapper.emitted('submit')![0][0]).toMatchObject({
      name: 'Bob',
    });
  });
  
  describe('邊界情況', () => {
    it('用戶數據為空', () => {
      const wrapper = mount(UserCard, {
        props: { user: null },
      });
      
      expect(wrapper.find('.user-name').text()).toBe('Anonymous');
    });
    
    it('頭像加載失敗', async () => {
      const wrapper = mount(UserCard, {
        props: defaultProps,
      });
      
      await wrapper.find('img').trigger('error');
```

```typescript
// tests/components/Button.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render, screen, fireEvent, waitFor, within } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { Button } from '@/components/Button';
import * as React from 'react';

describe('Button', () => {
  it('渲染正確的文本', () => {
    render(<Button>Click me</Button>);
    expect(screen.getByRole('button', { name: /click me/i })).toBeInTheDocument();
  });
  
  it('點擊事件處理', async () => {
    const handleClick = vi.fn();
    const user = userEvent.setup();
    
    render(<Button onClick={handleClick}>Click</Button>);
    
    await user.click(screen.getByRole('button'));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });
  
  it('禁用狀態不可點擊', async () => {
    const handleClick = vi.fn();
    const user = userEvent.setup();
    
    render(<Button disabled onClick={handleClick}>Disabled</Button>);
    
    await user.click(screen.getByRole('button'));
    expect(handleClick).not.toHaveBeenCalled();
  });
  
  it('加載狀態顯示', () => {
    render(<Button loading>Submit</Button>);
    
    expect(screen.getByRole('button')).toBeDisabled();
    expect(screen.getByLabelText(/loading/i)).toBeInTheDocument();
  });
});

describe('Form', () => {
  it('表單提交', async () => {
    const handleSubmit = vi.fn();
    const user = userEvent.setup();
    
    render(<form onSubmit={handleSubmit}>
      <input name="email" type="email" />
      <button type="submit">Submit</button>
    </form>);
    
    await user.type(screen.getByRole('textbox', { name: /email/i }), 'test@example.com');

  it('表單驗證', async () => {
    const user = userEvent.setup();

    </form>);
    
    await user.click(screen.getByRole('button'));
    
    await waitFor(() => {
      expect(screen.getByText(/invalid email/i)).toBeInTheDocument();
    });
  });
});
```

## 總結

單元測試是確保程式碼品質的關鍵實踐。Jest 和 Vitest 提供了強大而靈活的測試框架，選擇哪個應根據項目需求和技術棧決定。對於 React 項目，Jest 是首選；對於 Vue 3 或使用 Vite 的項目，Vitest 提供了更好的整合。關鍵是建立全面的測試策略，確保測試覆蓋關鍵功能，同時保持測試的可維護性和執行速度。
