# 後端開發

**標籤**: #後端 #伺服器 #API #資料庫
**日期**: 2024-01-15

後端開發 (Backend Development) 指的是處理伺服器端的邏輯、資料庫管理與 API 開發的技術領域。與前端開發專注於使用者介面不同，後端開發主要負責資料處理、業務邏輯、資料儲存和系統整合等核心功能。

## 後端開發的核心概念

### 客戶端-伺服器架構

現代網頁應用程式大多採用客戶端-伺服器架構。前端（客戶端）負責呈現使用者介面，收集使用者輸入，並將請求發送給後端（伺服器）。後端處理請求後，執行相應的業務邏輯，與資料庫互動，最後將結果返回給客戶端。

### 常見的後端職責

| 職責 | 說明 |
|------|------|
| API 開發 | 設計和實作 RESTful 或 GraphQL API |
| 資料庫管理 | 設計資料庫結構、優化查詢、確保資料完整性 |
| 業務邏輯 | 實現應用程式的核心功能和規則 |
| 身份認證 | 使用者登入、權限管理、Session 處理 |
| 效能優化 | 快取、負載平衡、效能監控 |
| 安全性 | 輸入驗證、SQL 注入防護、XSS 防護 |

## 後端程式語言

### Node.js

Node.js 是基於 Chrome V8 引擎的 JavaScript 執行環境，讓 JavaScript 能夠在伺服器端執行。其事件驅動、非阻塞 I/O 的特性使其在處理高並發應用時表現優異。

**熱門框架**:
- Express.js: 簡單靈活的 Node.js Web 框架
- NestJS: 漸進式 Node.js 框架，支援 TypeScript
- Fastify: 高效能的 Node.js 框架

**Express.js 範例**:
```javascript
const express = require('express');
const app = express();

app.use(express.json());
app.use(express.urlencoded({ extended: true }));

app.get('/api/users', (req, res) => {
    res.json([
        { id: 1, name: '王小明', email: 'wang@example.com' },
        { id: 2, name: '李小華', email: 'lee@example.com' }
    ]);
});

app.get('/api/users/:id', (req, res) => {
    const userId = parseInt(req.params.id);
    const user = getUserById(userId);
    
    if (!user) {
        return res.status(404).json({ error: '使用者不存在' });
    }
    
    res.json(user);
});

app.post('/api/users', (req, res) => {
    const { name, email } = req.body;
    
    if (!name || !email) {
        return res.status(400).json({ error: '姓名和 Email 為必填欄位' });
    }
    
    const newUser = createUser({ name, email });
    res.status(201).json(newUser);
});

app.put('/api/users/:id', (req, res) => {
    const userId = parseInt(req.params.id);
    const { name, email } = req.body;
    
    const updatedUser = updateUser(userId, { name, email });
    res.json(updatedUser);
});

app.delete('/api/users/:id', (req, res) => {
    const userId = parseInt(req.params.id);
    deleteUser(userId);
    res.status(204).send();
});

app.listen(3000, () => {
    console.log('伺服器執行於 http://localhost:3000');
});
```

```python
from django.db import models

class User(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    created_at = models.DateTimeField(auto_now_add=True)
    
    def __str__(self):

# views.py
from django.http import JsonResponse
from django.views.decorators.http import require_http_methods
from .models import User
import json

@require_http_methods(["GET"])
def user_list(request):
    users = User.objects.all().values('id', 'name', 'email')
    return JsonResponse(list(users), safe=False)

@require_http_methods(["POST"])
def user_create(request):
    data = json.loads(request.body)
    user = User.objects.create(
        name=data['name'],
        email=data['email']
    )
    return JsonResponse({
        'id': user.id,
        'name': user.name,
        'email': user.email
    }, status=201)
```

**Flask 範例**:

```python
from flask import Flask, request, jsonify
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///app.db'
db = SQLAlchemy(app)

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    email = db.Column(db.String(100), unique=True, nullable=False)

@app.route('/api/users', methods=['GET'])
def get_users():
    users = User.query.all()
    return jsonify([{
        'id': u.id,
        'name': u.name,
        'email': u.email
    } for u in users])

@app.route('/api/users', methods=['POST'])
def create_user():
    data = request.get_json()
    user = User(name=data['name'], email=data['email'])
    db.session.add(user)
    db.session.commit()
    return jsonify({'id': user.id}), 201

if __name__ == '__main__':
    app.run(debug=True)
```

### Go

Go (Golang) 是 Google 開發的程式語言，以其高效能、簡潔的語法和优秀的並發支援而受到歡迎，適合開發高並發的微服務。

**熱門框架**:

**Gin 範例**:
```go
package main

import (
    "net/http"
    "strconv"
    
    "github.com/gin-gonic/gin"
)

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

var users = []User{
    {ID: 1, Name: "王小明", Email: "wang@example.com"},
    {ID: 2, Name: "李小華", Email: "lee@example.com"},
}

func main() {
    r := gin.Default()
    
    r.GET("/api/users", func(c *gin.Context) {
        c.JSON(http.StatusOK, users)
    })
    
    r.GET("/api/users/:id", func(c *gin.Context) {
        id, _ := strconv.Atoi(c.Param("id"))
        for _, user := range users {
            if user.ID == id {
                c.JSON(http.StatusOK, user)
                return
            }
        }
        c.JSON(http.StatusNotFound, gin.H{"error": "使用者不存在"})
    })
    
    r.POST("/api/users", func(c *gin.Context) {
        var newUser User
        if err := c.ShouldBindJSON(&newUser); err != nil {
            c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
            return
        }
        newUser.ID = len(users) + 1
        users = append(users, newUser)
        c.JSON(http.StatusCreated, newUser)
    })
    
    r.DELETE("/api/users/:id", func(c *gin.Context) {
        id, _ := strconv.Atoi(c.Param("id"))
        for i, user := range users {
            if user.ID == id {
                users = append(users[:i], users[i+1:]...)
                c.Status(http.StatusNoContent)
                return
            }
        }
        c.JSON(http.StatusNotFound, gin.H{"error": "使用者不存在"})
    })
    
    r.Run(":3000")
}
```

**Spring Boot 範例**:
```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping
    public List<User> getAllUsers() {
        return userService.findAll();
    }
    
    @GetMapping("/{id}")
    public ResponseEntity<User> getUserById(@PathVariable Long id) {
        return userService.findById(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }
    
    @PostMapping
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
    
    @PutMapping("/{id}")
    public ResponseEntity<User> updateUser(
            @PathVariable Long id,
            @RequestBody User userDetails) {
        return userService.findById(id)
            .map(user -> {
                user.setName(userDetails.getName());
                user.setEmail(userDetails.getEmail());
                return ResponseEntity.ok(userService.save(user));
            })
            .orElse(ResponseEntity.notFound().build());
    }
    
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        if (userService.existsById(id)) {
            userService.deleteById(id);
            return ResponseEntity.noContent().build();
        }
        return ResponseEntity.notFound().build();
    }
}
```

## API 設計原則

### RESTful API

REST (Representational State Transfer) 是一種架構風格，RESTful API 遵循以下原則：

| 原則 | 說明 |
|------|------|
| 資源導向 | 使用名詞而非動詞，如 `/users` 而非 `/getUsers` |
| HTTP 方法 | GET (讀取)、POST (建立)、PUT (更新)、DELETE (刪除) |
| 無狀態 | 每個請求都包含所有必要的資訊 |
| 回應狀態碼 | 使用標準 HTTP 狀態碼表示結果 |
| 版本控制 | 在 URL 中包含版本號，如 `/api/v1/users` |

### HTTP 狀態碼

```http
// 成功回應
200 OK              - 請求成功
201 Created        - 資源建立成功
204 No Content     - 請求成功但無回應內容

// 用戶端錯誤
400 Bad Request    - 請求格式錯誤
401 Unauthorized   - 需要身份驗證
403 Forbidden      - 權限不足
404 Not Found      - 資源不存在
422 Unprocessable Entity - 請求語法正確但無法處理

// 伺服器錯誤
500 Internal Server Error - 伺服器內部錯誤
503 Service Unavailable - 服務不可用
```

### GraphQL

GraphQL 是 Facebook 開發的 API 查詢語言，允許客戶端精確指定需要的資料，減少網路傳輸量。

**GraphQL 範例**:
```javascript
const { graphql, buildSchema } = require('graphql');

const schema = buildSchema(`
  type User {
    id: ID!
    name: String!
    email: String!
    posts: [Post]
  }
  
  type Post {
    id: ID!
    title: String!
    content: String!
  }
  
  type Query {
    users: [User]
    user(id: ID!): User
  }
  
  type Mutation {
    createUser(name: String!, email: String!): User
    deleteUser(id: ID!): Boolean
  }
`);

const root = {
  users: () => {
    return getAllUsers();
  },
  user: ({ id }) => {
    return getUserById(id);
  },
  createUser: ({ name, email }) => {
    return createUser({ name, email });
  },
  deleteUser: ({ id }) => {
    return deleteUser(id);
  }
};

// 使用方式
graphql(schema, `
  query {
    users {
      name
      email
      posts {
        title
      }
    }
  }
`, root).then(response => {
  console.log(response);
});
```

## 資料庫整合

### SQL 資料庫

關聯式資料庫使用 SQL (Structured Query Language) 進行資料操作，常見的資料庫系統包括 PostgreSQL、MySQL、SQL Server 等。

**Node.js + PostgreSQL 範例**:
```javascript
const { Pool } = require('pg');

const pool = new Pool({
    host: 'localhost',
    database: 'mydb',
    user: 'user',
    password: 'password',
    port: 5432,
});

async function getUsers() {
    const result = await pool.query('SELECT * FROM users');
    return result.rows;
}

async function createUser(name, email) {
    const result = await pool.query(
        'INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *',
        [name, email]
    );
    return result.rows[0];
}

async function updateUser(id, name, email) {
    const result = await pool.query(
        'UPDATE users SET name = $1, email = $2 WHERE id = $3 RETURNING *',
        [name, email, id]
    );
    return result.rows[0];
}

async function deleteUser(id) {
    await pool.query('DELETE FROM users WHERE id = $1', [id]);
}
```

### NoSQL 資料庫

NoSQL 資料庫提供靈活的資料模型，適合處理大量非結構化資料。常見的 NoSQL 資料庫包括 MongoDB、Redis、Cassandra 等。

**Node.js + MongoDB 範例**:
```javascript
const { MongoClient } = require('mongodb');

const client = new MongoClient('mongodb://localhost:27017');
const dbName = 'mydb';

async function connect() {
    await client.connect();
    return client.db(dbName);
}

async function getUsers() {
    const db = await connect();
    const users = db.collection('users');
    return users.find({}).toArray();
}

async function createUser(userData) {
    const db = await connect();
    const users = db.collection('users');
    const result = await users.insertOne(userData);
    return result.insertedId;
}

async function updateUser(id, userData) {
    const db = await connect();
    const users = db.collection('users');
    const ObjectId = require('mongodb').ObjectId;
    const result = await users.updateOne(
        { _id: new ObjectId(id) },
        { $set: userData }
    );
    return result.modifiedCount > 0;
}

async function deleteUser(id) {
    const db = await connect();
    const users = db.collection('users');
    const ObjectId = require('mongodb').ObjectId;
    const result = await users.deleteOne({ _id: new ObjectId(id) });
    return result.deletedCount > 0;
}
```

**Node.js + Redis 範例**:
```javascript
const redis = require('redis');

const client = redis.createClient();

client.on('error', (err) => {
    console.log('Redis Client Error', err);
```

## 身份認證與授權

### JWT (JSON Web Token)

JWT 是一種開放標準，用於在各方之間安全地傳輸資訊。

**Node.js JWT 範例**:
```javascript
const jwt = require('jsonwebtoken');
const bcrypt = require('bcrypt');

const secretKey = 'your-secret-key';

// 使用者登入
app.post('/api/login', async (req, res) => {
    const { email, password } = req.body;
    
    const user = await getUserByEmail(email);
    if (!user) {
        return res.status(401).json({ error: 'Invalid credentials' });
    }
    
    const isValid = await bcrypt.compare(password, user.password);
    if (!isValid) {
        return res.status(401).json({ error: 'Invalid credentials' });
    }
    
    const token = jwt.sign(
        { userId: user.id, email: user.email },
        secretKey,
        { expiresIn: '24h' }
    );
    
    res.json({ token });
});

// 中間件驗證 JWT
function authenticateToken(req, res, next) {
    const authHeader = req.headers['authorization'];
    const token = authHeader && authHeader.split(' ')[1];
    
    if (!token) {
        return res.status(401).json({ error: 'Token required' });
    }
    
    jwt.verify(token, secretKey, (err, user) => {
        if (err) {
            return res.status(403).json({ error: 'Invalid token' });
        }
        req.user = user;
        next();
    });
}

// 受保護的路由
app.get('/api/profile', authenticateToken, (req, res) => {
    res.json({ user: req.user });
});
```

### OAuth 2.0

OAuth 2.0 是授權框架，允許應用程式獲得對使用者帳戶的有限存取權限。

**OAuth 流程**:
1. 使用者點擊「使用 Google 登入」
2. 應用程式重新導向到 Google 授權頁面
3. 使用者授權後，Google 回调带回授權碼
4. 應用程式用授權碼兌換存取權杖
5. 使用存取權杖調用 Google API 取得使用者資訊

## 安全性考量

### 輸入驗證

```javascript
const Joi = require('joi');

const userSchema = Joi.object({
    name: Joi.string().min(2).max(50).required(),
    email: Joi.string().email().required(),
    password: Joi.string().min(8).pattern(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/).required(),
    age: Joi.number().integer().min(0).max(150)
});

function validateUser(data) {
    const { error, value } = userSchema.validate(data);
    if (error) {
        throw new Error(error.details[0].message);
    }
    return value;
}
```

### SQL 注入防護

使用參數化查詢或 ORM 來防止 SQL 注入：

```javascript
// 使用參數化查詢（安全）
const result = await pool.query(
    'SELECT * FROM users WHERE email = $1',
    [email]
);

// 使用 ORM（安全）
const user = await User.findOne({ where: { email } });
```

## 後端架構模式

### MVC 架構

MVC (Model-View-Controller) 是經典的後端架構模式：
- Model: 負責資料模型和業務邏輯

### 微服務架構

微服務將應用程式拆分為小型、獨立的服務，每個服務負責特定功能。

**微服務優點**:
- 獨立部署和擴展
- 技術多樣性
- 故障隔離
- 更快的開發週期

**微服務挑戰**:
- 服務發現
- 網路延遲
- 資料一致性
- 部署複雜度

## 效能優化

### 快取策略

```javascript
const cache = require('memory-cache');

function cacheMiddleware(duration) {
    return (req, res, next) => {
        const key = req.originalUrl;
        const cached = cache.get(key);
        
        if (cached) {
            return res.json(cached);
        }
        
        const originalJson = res.json;
        res.json = (data) => {
            cache.put(key, data, duration * 1000);
            return originalJson.call(res, data);
        };
        
        next();
    };
}

app.get('/api/users', cacheMiddleware(60), (req, res) => {
    const users = getAllUsersFromDB();
    res.json(users);
});
```
