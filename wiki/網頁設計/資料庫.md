# 資料庫

**標籤**: #資料庫 #SQL #NoSQL #ORM #MySQL #PostgreSQL #MongoDB
**日期**: 2024-01-15

資料庫是組織、儲存和管理資料的系統，它是現代軟體應用程式的核心基礎設施。從簡單的個人筆記應用到複雜的企業系統，幾乎所有軟體都需要某種形式的資料持久化。資料庫管理系統 (DBMS) 提供了資料的存取、查詢、更新和保護功能。

## 資料庫的發展歷程

### 層次與網路資料庫 (1960s-1970s)

最早的資料庫系統採用層次結構（如 IBM IMS）和網路結構（如 CODASYL），雖然解決了資料組織問題，但缺乏靈活性。

### 關聯式資料庫 (1970s-現在)

1970 年，IBM 研究員 Edgar F. Codd 發表了關聯式資料庫模型論文，奠定了現代關聯式資料庫的基礎。1980 年代後，Oracle、MySQL、PostgreSQL 等關聯式資料庫開始普及。

### NoSQL 資料庫 (2000s-現在)

隨著 Web 2.0 和大資料時代的來臨，傳統關聯式資料庫在處理海量非結構化資料時顯得力不從心，NoSQL 資料庫應運而生。

### 多模資料庫 (2010s-現在)

現代資料庫系統趨向於支援多種資料模型，提供更大的靈活性。

## 關聯式資料庫 (RDBMS)

關聯式資料庫將資料組織成表格（Table），表格之間透過鍵（Key）建立關係。使用結構化查詢語言 (SQL) 進行資料操作。

### 核心概念

**表格結構**:
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    age INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    total DECIMAL(10, 2) NOT NULL,
    status ENUM('pending', 'paid', 'shipped', 'delivered') DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### SQL 基本操作

**查詢資料 (SELECT)**:
```sql
-- 基本查詢
SELECT * FROM users;

-- 指定欄位
SELECT name, email FROM users;

-- 條件篩選
SELECT * FROM users WHERE age >= 18;

-- 排序
SELECT * FROM users ORDER BY created_at DESC;

-- 限制結果
SELECT * FROM users LIMIT 10 OFFSET 20;

-- 聚合函數
SELECT COUNT(*) as total_users,
       AVG(age) as average_age,
       MIN(age) as youngest,
       MAX(age) as oldest
FROM users;

-- 分組
SELECT status, COUNT(*) as count
FROM orders
GROUP BY status
HAVING COUNT(*) > 10;
```

**新增資料 (INSERT)**:
```sql
-- 單筆新增
INSERT INTO users (name, email, age)
VALUES ('王小明', 'wang@example.com', 25);

-- 多筆新增
INSERT INTO users (name, email, age)
VALUES 
    ('李小華', 'lee@example.com', 30),
    ('張小華', 'zhang@example.com', 28);

-- 查詢後新增
INSERT INTO users (name, email, age)
SELECT name, email, age FROM temp_users WHERE age >= 18;
```

**更新資料 (UPDATE)**:
```sql
-- 條件更新
UPDATE users SET age = 26 WHERE name = '王小明';

-- 多欄位更新
UPDATE users 
SET name = '王小明', email = 'wang.new@example.com'
WHERE id = 1;

-- 基於其他表格更新
UPDATE users u
SET age = (
    SELECT AVG(age) FROM users WHERE department = u.department
)
WHERE u.department = 'Sales';
```

**刪除資料 (DELETE)**:
```sql
-- 條件刪除
DELETE FROM users WHERE age < 18;

-- 刪除重複資料
DELETE FROM users
WHERE id NOT IN (
    SELECT MIN(id) FROM users GROUP BY email
);
```

### JOIN 查詢

```sql
-- INNER JOIN: 只回傳符合條件的記錄
SELECT u.name, o.total, o.status
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

-- LEFT JOIN: 回傳左表所有記錄
SELECT u.name, o.total
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;

-- RIGHT JOIN: 回傳右表所有記錄
SELECT u.name, o.total
FROM users u
RIGHT JOIN orders o ON u.id = o.user_id;

-- 多表 JOIN
SELECT u.name, o.total, p.name as product_name
FROM users u
JOIN orders o ON u.id = o.user_id
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id;
```

### 交易 (Transaction)

```sql
START TRANSACTION;

INSERT INTO users (name, email) VALUES ('新使用者', 'new@example.com');
SET @user_id = LAST_INSERT_ID();

INSERT INTO orders (user_id, total) VALUES (@user_id, 100.00);

COMMIT;  -- 確認交易
-- ROLLBACK;  -- 取消交易
```

### 索引

```sql
-- 單欄索引
CREATE INDEX idx_users_email ON users(email);

-- 複合索引
CREATE INDEX idx_orders_user_status ON orders(user_id, status);

-- 唯一索引
CREATE UNIQUE INDEX idx_users_email ON users(email);

-- 全文索引 (MySQL)
CREATE FULLTEXT INDEX idx_content ON articles(content);

-- 查詢效能分析
EXPLAIN SELECT * FROM users WHERE email = 'wang@example.com';
```

### PostgreSQL 進階功能

```sql
-- CTE (Common Table Expression)
WITH recent_orders AS (
    SELECT user_id, COUNT(*) as order_count
    FROM orders
    WHERE created_at > NOW() - INTERVAL '30 days'
    GROUP BY user_id
)
SELECT u.name, ro.order_count
FROM users u
JOIN recent_orders ro ON u.id = ro.user_id
ORDER BY ro.order_count DESC;

-- 視窗函數
SELECT 
    name,
    department,
    salary,
    AVG(salary) OVER (PARTITION BY department) as dept_avg,
    RANK() OVER (ORDER BY salary DESC) as salary_rank
FROM employees;

-- JSON 支援
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255),
    attributes JSONB
);

INSERT INTO products (name, attributes)
VALUES ('iPhone', '{"color": "black", "storage": "256GB"}'::jsonb);

SELECT * FROM products WHERE attributes->>'color' = 'black';
```

## NoSQL 資料庫

NoSQL (Not Only SQL) 資料庫放棄了傳統關聯式資料庫的嚴格結構，提供更高的擴展性和靈活性。

### 文件資料庫 (Document Database)

**MongoDB** 是最流行的文件資料庫，資料以類似 JSON 的 BSON 格式儲存。

```javascript
// MongoDB 文件結構
{
    "_id": ObjectId("..."),
    "name": "王小明",
    "email": "wang@example.com",
    "age": 25,
    "address": {
        "city": "台北市",
        "district": "信義區"
    },
    "tags": ["developer", "javascript"],
    "orders": [
        {
            "id": 1,
            "total": 1000,
            "status": "delivered"
        }
    ]
}
```

```javascript
// Node.js + MongoDB 操作
const { MongoClient } = require('mongodb');

async function main() {
    const client = new MongoClient('mongodb://localhost:27017');
    await client.connect();
    
    const db = client.db('mydb');
    const users = db.collection('users');
    
    // 新增
    const result = await users.insertOne({
        name: '王小明',
        email: 'wang@example.com',
        age: 25
    });
    
    // 查詢
    const user = await users.findOne({ _id: result.insertedId });
    const allUsers = await users.find({ age: { $gte: 18 } }).toArray();
    
    // 更新
    await users.updateOne(
        { _id: result.insertedId },
        { $set: { age: 26 } }
    );
    
    // 刪除
    await users.deleteOne({ _id: result.insertedId });
    
    await client.close();
}
```

### 鍵值資料庫 (Key-Value Database)

**Redis** 是記憶體鍵值資料庫，支援多種資料結構，適合作為快取或即時系統。

```javascript
const redis = require('redis');

const client = redis.createClient();

client.on('error', (err) => console.error(err));

// 字串操作
client.set('user:1:name', '王小明', redis.print);
client.get('user:1:name', (err, reply) => console.log(reply));

// 雜湊操作
client.hset('user:1', 'name', '王小明', redis.print);
client.hset('user:1', 'email', 'wang@example.com', redis.print);
client.hgetall('user:1', (err, object) => console.log(object));

// 列表操作
client.rpush('tasks', 'task1', 'task2', 'task3', redis.print);
client.lrange('tasks', 0, -1, (err, replies) => console.log(replies));

// 集合操作
client.sadd('tags', 'javascript', 'python', 'java', redis.print);
client.smembers('tags', (err, members) => console.log(members));

// _sorted set_
client.zadd('leaderboard', 100, 'player1', redis.print);
client.zadd('leaderboard', 200, 'player2', redis.print);
client.zrevrange('leaderboard', 0, -1, 'WITHSCORES', (err, result) => {
    console.log(result);
});
```

### 寬欄資料庫 (Wide-Column Database)

**Apache Cassandra** 適合處理大量資料和寫入密集型工作負載。

```sql
CREATE KEYSPACE myapp WITH REPLICATION = {
    'class': 'SimpleStrategy',
    'replication_factor': 3
};

CREATE TABLE myapp.users (
    user_id uuid PRIMARY KEY,
    name text,
    email text,
    age int
);

INSERT INTO myapp.users (user_id, name, email, age)
VALUES (uuid(), '王小明', 'wang@example.com', 25);

SELECT * FROM myapp.users WHERE user_id = ?;
```

### 圖形資料庫 (Graph Database)

**Neo4j** 專門設計來處理複雜的圖形關係。

```cypher
// 建立節點
CREATE (alice:Person {name: '王小明', age: 25})
CREATE (bob:Person {name: '李小華', age: 30})
CREATE (movie:Movie {title: '電影A', year: 2023})

// 建立關係
CREATE (alice)-[:KNOWS {since: 2020}]->(bob)
CREATE (alice)-[:WATCHED]->(movie)
CREATE (bob)-[:WATCHED]->(movie)

// 查詢
MATCH (p:Person)-[:WATCHED]->(m:Movie)
WHERE m.year = 2023
RETURN p.name, m.title

// 短路徑查詢
MATCH (a:Person {name: '王小明'}), (b:Person {name: '李小華'})
MATCH path = shortestPath((a)-[*]-(b))
RETURN path
```

## ORM (Object-Relational Mapping)

ORM 是將物件導向程式語言中的物件與關聯式資料庫中的表格進行映射的技術。

### Sequelize (Node.js)

```javascript
const { Sequelize, DataTypes } = require('sequelize');

const sequelize = new Sequelize('database', 'username', 'password', {
    host: 'localhost',
    dialect: 'mysql',
    logging: false
});

const User = sequelize.define('User', {
    name: {
        type: DataTypes.STRING,
        allowNull: false
    },
    email: {
        type: DataTypes.STRING,
        allowNull: false,
        unique: true,
        validate: {
            isEmail: true
        }
    },
    age: {
        type: DataTypes.INTEGER,
        validate: {
            min: 0,
            max: 150
        }
    }
});

const Order = sequelize.define('Order', {
    total: {
        type: DataTypes.DECIMAL(10, 2),
        allowNull: false
    },
    status: {
        type: DataTypes.ENUM('pending', 'paid', 'shipped', 'delivered'),
        defaultValue: 'pending'
    }
});

User.hasMany(Order);
Order.belongsTo(User);

// 同步
sequelize.sync();

// 新增
const user = await User.create({
    name: '王小明',
    email: 'wang@example.com',
    age: 25
});

// 查詢
const users = await User.findAll({
    where: { age: { [Op.gte]: 18 } },
    order: [['createdAt', 'DESC']],
    limit: 10
});

// 更新
await user.update({ age: 26 });

// 刪除
await user.destroy();
```

### SQLAlchemy (Python)

```python
from sqlalchemy import create_engine, Column, Integer, String, ForeignKey, DateTime
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker, relationship
from datetime import datetime

Base = declarative_base()

class User(Base):
    __tablename__ = 'users'
    
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    email = Column(String(255), unique=True, nullable=False)
    age = Column(Integer)
    created_at = Column(DateTime, default=datetime.utcnow)
    
    orders = relationship('Order', back_populates='user')

class Order(Base):
    __tablename__ = 'orders'
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey('users.id'))
    total = Column(Integer, nullable=False)
    status = Column(String(50), default='pending')
    created_at = Column(DateTime, default=datetime.utcnow)
    
    user = relationship('User', back_populates='orders')

engine = create_engine('sqlite:///mydb.sqlite')
Base.metadata.create_all(engine)

Session = sessionmaker(bind=engine)
session = Session()

# 新增
user = User(name='王小明', email='wang@example.com', age=25)
session.add(user)
session.commit()

# 查詢
users = session.query(User).filter(User.age >= 18).all()
user = session.query(User).filter_by(email='wang@example.com').first()

# 更新
user.age = 26
session.commit()

# 刪除
session.delete(user)
session.commit()
```

### Prisma (現代 ORM)

```prisma
// schema.prisma
datasource db {
    provider = "postgresql"
    url      = env("DATABASE_URL")
}

generator client {
    provider = "prisma-client-js"
}

model User {
    id        Int      @id @default(autoincrement())
    name      String
    email     String   @unique
    age       Int?
    orders    Order[]
    createdAt DateTime @default(now())
    updatedAt DateTime @updatedAt
}

model Order {
    id        Int      @id @default(autoincrement())
    userId    Int
    user      User     @relation(fields: [userId], references: [id])
    total     Int
    status    String   @default("pending")
    createdAt DateTime @default(now())
}
```

## 資料庫選擇指南

### 選擇關聯式資料庫的情況

- 需要複雜的查詢和 JOIN 操作
- 需要嚴格的資料一致性 (ACID)
- 資料結構相對穩定
- 需要事務支援

### 選擇 NoSQL 資料庫的情況

- 資料結構靈活多變
- 需要高效處理大量寫入
- 需要水平擴展能力
- 快取和即時系統

### 常見資料庫比較

| 資料庫 | 類型 | 優點 | 適用場景 |
|--------|------|------|----------|
| PostgreSQL | 關聯式 | 功能強大、標準相容 | 企業應用、資料分析 |
| MySQL | 關聯式 | 簡單易用、效能好 | Web 應用 |
| SQLite | 關聯式 | 輕量、無伺服器 | 嵌入式應用、測試 |
| MongoDB | 文件 | 靈活、易上手 | 原型、MongoDB |
| Redis | 鍵值 | 極速、支援多種結構 | 快取、會話 |
| Cassandra | 寬欄 | 寫入快、可擴展 | 物聯網、事件記錄 |
| Neo4j | 圖形 | 關係查詢 | 社交網路推薦系統 |
