# 貪婪法測試說明

## 執行方式

```bash
python 01-greedy.py
```

## 測試項目

### 1. activity_selection - 活動選擇
- **輸入**: starts=[1,3,0,5,3,5,6,8,8,2], finishes=[4,5,6,7,9,9,10,11,12,14]
- **預期輸出**: 最大不相交活動數
- **說明**: 貪心選擇結束最早的活動

### 2. coin_change_greedy - 硬幣找零
- **輸入**: amount=63, coins=[25,10,5,1]
- **預期輸出**: [(25,2), (10,1), (1,3)], remaining=0
- **說明**: 貪心不一定最優（某些金額會失敗）

### 3. fractional_knapsack - 分數背包
- **輸入**: values=[60,100,120], weights=[10,20,30], capacity=50
- **預期輸出**: 總價值 ~220
- **說明**: 可取部分物品，貪心最優

### 4. huffman_coding - 霍夫曼編碼
- **輸入**: freq={'a':45,'b':13,'c':12,'d':16,'e':9,'f':5}
- **預期輸出**: 每個字元的編碼
- **說明**: 最優前綴碼

### 5. dijkstra_shortest_path - Dijkstra 最短路徑
- **輸入**: graph, start
- **預期輸出**: 各節點的最短距離
- **說明**: 貪心選擇最近的未訪問節點

## 預期輸出範例

```
活動選擇: [(1, 4), (5, 7), (8, 11), (12, 14)]
硬幣找零 63: [(25, 2), (10, 1), (1, 3)], 剩餘: 0
分數背包總價值: 220.0
取用: [(120, 30, 0.666...), (100, 20, 1.0), (60, 10, 1.0)]
霍夫曼編碼: {'a': '0', 'b': '101', 'c': '100', 'd': '111', 'e': '1101', 'f': '1100'}
```
