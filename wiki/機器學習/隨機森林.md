# 隨機森林 (Random Forest)

## 概述

隨機森林是一種基於決策樹的集成學習方法，由 Leo Breiman 於 2001 年提出。它透過建構多棵決策樹並匯總它們的預測結果，來提高模型的準確性和穩定性。隨機森林的核心思想是「三個臭皮匠，勝過諸葛亮」——通過組合多個弱分類器（決策樹），形成一個強分類器。

隨機森林屬於 Bootstrap Aggregating（簡稱 Bagging）的一種變體，它在傳統 Bagging 的基礎上加入了隨機特徵選擇，进一步减少了各棵樹之間的相關性。

## 原理

隨機森林的建構過程可以總結為以下三個關鍵步驟：

### 1. Bootstrap Sampling（自助採樣）

對於每棵樹，我們從訓練資料中進行有放回的隨機抽樣，產生和原始資料集大小相同的訓練子集。這意味著：
- 每個訓練子集大約包含 63.2% 的原始樣本（因為每次抽樣不被選中的機率是 $(1 - 1/n)^n \approx 1/e$）
- 樣本會被重複選中，有些樣本則不會被選中

這些未被選中的樣本稱為「袋外樣本」（Out-of-Bag, OOB），可用於評估模型效能。

### 2. 特徵隨機選擇（Random Feature Selection）

這是隨機森林與傳統 Bagging 的關鍵區別。在每個節點分裂時，我們不是從所有特徵中選擇最佳分裂特徵，而是從隨機選擇的特徵子集中選擇。這有兩個主要參數：
- `max_features`：每次分裂時考慮的特徵數量
- 一般設為 $\sqrt{n\_features}$（分類）或 $n\_features$（回歸）

### 3. 建構決策樹

每棵樹在隨機選擇的訓練子集上，使用隨機選擇的特徵子集，完整建構一棵決策樹（不進行剪枝）。

### 4. 預測匯總

對於分類問題：多數投票（Majority Voting）
- 每棵樹產生一個類別預測
- 最終結果是出現次數最多的類別

對於回歸問題：平均值
- 每棵樹產生一個數值預測
- 最終結果是所有樹預測的平均值

## 為什麼隨機森林有效？

隨機森林的有效性基於兩個統計學原理：

### 偏差-變異權衡（Bias-Variance Tradeoff）

單棵決策樹通常有較低的偏差但较高的變異性（容易過擬合）。透過平均多棵樹的預測：
- 保持較低的偏差（因為每棵樹都是在完整資料上訓練的）
- 大幅降低變異性（因為各棵樹的錯誤会互相抵消）

### 多樣性來源

隨機森林的多樣性來自三個隨機性來源：
1. **訓練樣本的隨機性**：每棵樹使用不同的 Bootstrap 樣本
2. **特徵的隨機性**：每個節點考慮不同的特徵子集
3. **分裂閾值的隨機性**：雖然每棵樹是確定性建構的，但起點不同

## 完整實現

```python
import numpy as np
from collections import Counter

class RandomForestClassifier:
    """隨機森林分類器"""
    
    def __init__(self, n_estimators=100, max_depth=10, 
                 min_samples_split=2, min_samples_leaf=1,
                 max_features='sqrt', random_state=None):
        self.n_estimators = n_estimators
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.min_samples_leaf = min_samples_leaf
        self.max_features = max_features
        self.random_state = random_state
        self.trees = []
    
    def _get_n_features(self, n_features):
        """計算每次分裂考慮的特徵數"""
        if self.max_features == 'sqrt':
            return int(np.sqrt(n_features))
        elif self.max_features == 'log2':
            return int(np.log2(n_features + 1))
        else:
            return n_features
    
    def _bootstrap(self, X, y):
        """Bootstrap 採樣"""
        n_samples = X.shape[0]
        indices = np.random.choice(n_samples, n_samples, replace=True)
        return X[indices], y[indices]
    
    def _find_best_split(self, X, y, features):
        """找到最佳分裂"""
        best_gain = -1
        best_feature = None
        best_threshold = None
        
        n_features = X.shape[1]
        n_candidates = self._get_n_features(n_features)
        
        # 隨機選擇特徵子集
        candidate_features = np.random.choice(
            n_features, n_candidates, replace=False
        )
        
        for feature in candidate_features:
            values = X[:, feature]
            unique_values = np.unique(values)
            
            for i in range(len(unique_values) - 1):
                threshold = (unique_values[i] + unique_values[i + 1]) / 2
                
                left_mask = values <= threshold
                right_mask = ~left_mask
                
                if np.sum(left_mask) < self.min_samples_leaf or \
                   np.sum(right_mask) < self.min_samples_leaf:
                    continue
                
                # 計算基尼不純度
                parent_impurity = self._gini(y)
                n_left = np.sum(left_mask)
                n_right = np.sum(right_mask)
                child_impurity = (n_left / len(y)) * self._gini(y[left_mask]) + \
                              (n_right / len(y)) * self._gini(y[right_mask])
                
                gain = parent_impurity - child_impurity
                
                if gain > best_gain:
                    best_gain = gain
                    best_feature = feature
                    best_threshold = threshold
        
        return best_feature, best_threshold, best_gain
    
    def _gini(self, y):
        """計算基尼不純度"""
        if len(y) == 0:
            return 0
        counts = np.bincount(y)
        probs = counts / len(y)
        return 1 - np.sum(probs ** 2)
    
    def _build_tree(self, X, y, depth=0):
        """建構決策樹"""
        if (depth >= self.max_depth or 
            len(y) < self.min_samples_split or
            len(np.unique(y)) == 1):
            return {'leaf': True, 'class': np.bincount(y).argmax()}
        
        feature, threshold, gain = self._find_best_split(X, y, None)
        
        if feature is None:
            return {'leaf': True, 'class': np.bincount(y).argmax()}
        
        left_mask = X[:, feature] <= threshold
        right_mask = ~left_mask
        
        left_tree = self._build_tree(X[left_mask], y[left_mask], depth + 1)
        right_tree = self._build_tree(X[right_mask], y[right_mask], depth + 1)
        
        return {
            'leaf': False,
            'feature': feature,
            'threshold': threshold,
            'left': left_tree,
            'right': right_tree
        }
    
    def fit(self, X, y):
        """訓練模型"""
        np.random.seed(self.random_state)
        self.trees = []
        
        for i in range(self.n_estimators):
            # Bootstrap 採樣
            X_boot, y_boot = self._bootstrap(X, y)
            
            # 建構決策樹
            tree = self._build_tree(X_boot, y_boot)
            self.trees.append(tree)
        
        return self
    
    def _predict_single(self, x, node):
        """單個樣本預測"""
        if node['leaf']:
            return node['class']
        
        if x[node['feature']] <= node['threshold']:
            return self._predict_single(x, node['left'])
        else:
            return self._predict_single(x, node['right'])
    
    def predict(self, X):
        """��測"""
        predictions = np.array([
            [self._predict_single(x, tree) for tree in self.trees]
            for x in X
        ])
        
        # 多數投票
        result = []
        for i in range(X.shape[0]):
            vote = Counter(predictions[i])
            result.append(vote.most_common(1)[0][0])
        
        return np.array(result)
    
    def predict_proba(self, X):
        """預測機率"""
        probas = np.zeros((X.shape[0], len(np.unique([t for tree in self.trees for t in np.arange(100)]))))
        
        for i, x in enumerate(X):
            votes = [self._predict_single(x, tree) for tree in self.trees]
            for vote in votes:
                probas[i, vote] += 1
        
        return probas / self.n_estimators
```

## 使用 sklearn

```python
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor

# 分類
clf = RandomForestClassifier(
    n_estimators=100,      # 樹的數量
    max_depth=None,         # 最大深度，None 表示不限制
    min_samples_split=2,   # 內部節點最小樣本數
    min_samples_leaf=1,     # 葉節點最小樣本數
    max_features='sqrt',    # 每個分裂考慮的特徵數
    bootstrap=True,         # 是否使用 bootstrap
    oob_score=True,        # 計算袋外得分
    random_state=42
)

clf.fit(X_train, y_train)
predictions = clf.predict(X_test)

# 袋外得分（評估模型效能）
oob_score = clf.oob_score_

# 特徵重要性
importances = clf.feature_importances_
```

```python
# 回歸
reg = RandomForestRegressor(
    n_estimators=100,
```

## 特徵重要性

隨機森林的一個重要優勢是可以計算特徵重要性。這對於了解哪些特徵對預測最重要、以及進行特徵選擇非常有用。

### 基於不純度減少的重要性

```python
def feature_importance(forest, feature_names):
    """計算特徵重要性"""
    importances = {}
    
    for tree in forest.trees:
        self._extract_importance(tree, importances)
    
    # 正規化
    total = sum(importances.values())
    return {k: v / total for k, v in importances.items()}
```

sklearn 提供的特徵重要性：

```python
import matplotlib.pyplot as plt

# 繪製特徵重要性
importances = clf.feature_importances_
indices = np.argsort(importances)[::-1]

plt.figure(figsize=(10, 6))
plt.title('Feature Importances')
plt.bar(range(len(importances)), importances[indices])
plt.xticks(range(len(importances)), [feature_names[i] for i in indices], rotation=45)
plt.tight_layout()
plt.show()
```

### 基於排列的重要性（Permutation Importance）

另一種計算特徵重要性的方法是通过隨機打亂特徵值來測量模型效能的下降：

```python
from sklearn.inspection import permutation_importance

# 基於排列的重要性
result = permutation_importance(clf, X_test, y_test, 
                            n_repeats=10, random_state=42)
```

## 調參數指南

### 1. n_estimators

樹的數量：
- 越多越好，但回報遞減
- 一般 100-500 足夠
- 可以用 OOB 得分來選擇

```python
# 使用 OOB 選擇最佳數量
oob_scores = []
for n in [50, 100, 200, 300, 500]:
    clf = RandomForestClassifier(n_estimators=n, oob_score=True)
    clf.fit(X, y)
    oob_scores.append(clf.oob_score_)
```

### 2. max_depth

最大深度：
- None：不限制深度（可能過擬合）
- 較小值：更保守的模型
- 一般 10-30 是合理的起始點

### 3. max_features

每個分裂考慮的特徵數：
- `'sqrt'`：分類的經典選擇
- `'log2'`：特徵多時使用
- `0.3-0.7`：特徵比例

### 4. min_samples_split / min_samples_leaf

最小樣本數：
- 增加這些值會使模型更保守
- 有助於減少過擬合
- 典型值：2-10

## 隨機森林 vs 單棵決策樹

| 特性 | 隨機森林 | 單棵決策樹 |
|------|----------|-----------|
| 準確率 | 較高 | 較低 |
| 穩定性 | 高 | 低 |
| 過擬合 | 較不易 | 容易 |
| 訓練時間 | 較長 | 短 |
| 可解釋性 | 較低 | 高 |
| 特徵重要性 | 內建 | 需要額外計算 |

## 優點

1. **準確率高**：在多數問題上表現優秀
2. **穩定性強**：對資料的小變化不敏感
3. **可處理高維資料**：在維度較高時依然有效
4. **可處理缺失值**：透過增加指示特徵
5. **可並行訓練**：每棵樹相互獨立
6. **內建特徵重要性**：便於理解模型
7. **可處理類別和數值特徵**

## 缺點

1. **訓練時間較長**：需要訓練多棵樹
2. **預測時間較長**：需要遍歷所有樹
3. **可解釋性較低**：不能直接轉換為簡單規則
4. **可能過擬合**：在噪音較大的資料上
