# 監督式學習

監督式學習（Supervised Learning）是機器學習中最基本也是最重要的學習範式之一。在監督式學習中，演算法使用帶有標籤的訓練資料進行學習，目標是學習從輸入特徵到輸出標籤的映射函數。這種學習方式就像是有老師指導的學習過程，訓練資料中的標籤就像是老師提供的「標準答案」，演算法透過比較預測結果與標準答案來不斷調整模型參數，最終希望能夠對新的、未見過的資料做出準確的預測。

## 監督式學習的基本概念

### 學習框架

監督式學習的核心是建立一個模型，這個模型能夠將輸入特徵映射到輸出標籤：

```
┌─────────────────────────────────────────────────────────────────┐
│                      監督式學習框架                             │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  訓練階段：                                                   │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │   輸入 X     │ →  │     模型     │ →  │   輸出 Y     │   │
│  │ (特徵向量)   │    │  f(x; θ)     │    │ (預測標籤)   │   │
│  └──────────────┘    └──────┬───────┘    └──────────────┘   │
│                              │                                 │
│                        ┌─────┴─────┐                          │
│                        │ 損失函數   │                          │
│                        │ L(y, ŷ)   │                          │
│                        └─────┬─────┘                          │
│                              │                                 │
│                        ┌─────┴─────┐                          │
│                        │ 最佳化器   │                          │
│                        └─────┬─────┘                          │
│                              │                                 │
│                        ┌─────┴─────┐                          │
│                        │ 更新 θ    │                          │
│                        └───────────┘                          │
│                                                              │
│  推論階段：                                                   │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │   輸入 x*    │ →  │ 訓練好的模型   │ →  │   輸出 y*   │   │
│  │ (新特徵)     │    │   f(x*; θ*)  │    │ (預測結果)   │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 訓練資料

監督式學習需要大量的訓練資料，每筆資料都由輸入特徵和對應的標籤組成：

```python
# 訓練資料格式
# D = {(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)}
# 其中 xᵢ 是特徵向量，yᵢ 是標籤

# 範例：房價預測
training_data = [
    {"面積": 100, "房間": 3, "屋齡": 5, "價格": 500},
    {"面積": 150, "房間": 4, "屋齡": 10, "價格": 700},
    {"面積": 80, "房間": 2, "屋齡": 2, "價格": 350},
    # ... 更多資料
]

# 範例：圖片分類
# (圖片像素, 類別標籤)
# (image_pixels, "cat")
# (image_pixels, "dog")
# (image_pixels, "bird")
```

## 監督式學習的類型

監督式學習主要分為兩大類型：回歸和分類。這兩種類型的目標不同，處理方式也有所差異。

### 回歸（Regression）

回歸問題的輸出是連續值，目標是預測一個或多個連續變數：

```
┌─────────────────────────────────────────────────────────────────┐
│                      回歸問題範例                               │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  • 房價預測：輸入房屋特徵（面積、房間數、位置等）               │
│              輸出房價（連續值，如 500 萬）                     │
│                                                              │
│  • 溫度預測：輸入氣象資料（濕度、風速、氣壓等）                 │
│              輸出溫度（連續值，如 25.5°C）                     │
│                                                              │
│  • 銷售預測：輸入時間特徵（月份、季節、促銷活動）               │
│              輸出銷售額（連續值）                              │
│                                                              │
│  • 股價預測：輸入歷史股價和財經指標                            │
│              輸出未來股價（連續值）                            │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 分類（Classification）

分類問題的輸出是離散類別，目標是將輸入分配到預定義的類別中：

```
┌─────────────────────────────────────────────────────────────────┐
│                      分類問題範例                               │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  二分類（Binary Classification）：                              │
│  • 郵件垃圾過濾：輸入郵件內容，輸出是否為垃圾郵件（是/否）       │
│  • 疾病診斷：輸入症狀資料，輸出是否患病（是/否）               │
│  • 信用風險：輸入申請人資料，輸出是否違約（是/否）             │
│                                                              │
│  多分類（Multi-class Classification）：                         │
│  • 圖片分類：輸入圖片，輸出類別（貓、狗、鳥等）                 │
│  • 手寫數字識別：輸入手寫數字影像，輸出 0-9                     │
│  • 語音識別：輸入音頻特徵，輸出文字                            │
│                                                              │
│  多標籤分類（Multi-label Classification）：                      │
│  • 圖片標籤：一張圖片可能同時包含多個標籤                      │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 訓練流程

監督式學習的訓練過程是一個迭代優化的過程：

### 流程步驟

```python
# 監督式學習訓練流程

# 1. 準備資料
X_train, y_train = load_data()           # 載入訓練資料
X_val, y_val = split_validation(X, y)   # 分割驗證集

# 2. 選擇模型
model = LinearRegression()               # 選擇初始模型
# 或
model = NeuralNetwork(hidden_layers=[128, 64])

# 3. 定義損失函數
loss_fn = MeanSquaredError()              # 回歸用
# 或
loss_fn = CrossEntropyLoss()              # 分類用

# 4. 選擇最佳化器
optimizer = Adam(model.parameters(), lr=0.001)

# 5. 訓練循環
for epoch in range(num_epochs):
    # 前向傳播
    predictions = model(X_train)
    loss = loss_fn(predictions, y_train)
    
    # 反向傳播
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    # 驗證
    val_pred = model(X_val)
    val_loss = loss_fn(val_pred, y_val)
    
    # early stopping
    if improvement < threshold:
        break

# 6. 測試評估
test_predictions = model(X_test)
accuracy = evaluate(test_predictions, y_test)
```

### 損失函數

不同類型的問題使用不同的損失函數：

```python
# 回歸損失函數
# 均方誤差 (MSE)
mse = (1/n) * Σ(yᵢ - ŷᵢ)²

# 平均絕對誤差 (MAE)
mae = (1/n) * Σ|yᵢ - ŷᵢ|

# Huber 損失（結合 MSE 和 MAE）
huber = L(y, ŷ) = {
    0.5*(y-ŷ)²  if |y-ŷ| ≤ δ
    δ*|y-ŷ| - 0.5*δ²  otherwise
}

# 分類損失函數
# 交叉熵 (Cross Entropy)
cross_entropy = -Σ yᵢ * log(ŷᵢ)

# Hinge 損失（SVM 用）
hinge = max(0, 1 - y * ŷ)
```

## 常見演算法

監督式學習涵蓋了多種經典演算法，從簡單的線性模型到複雜的神經網路：

### 線性模型

```python
from sklearn.linear_model import LinearRegression, Ridge, Lasso, LogisticRegression

# 線性回歸
lr = LinearRegression()
lr.fit(X_train, y_train)
predictions = lr.predict(X_test)

# 嶺回歸（L2 正則化）
ridge = Ridge(alpha=1.0)
ridge.fit(X_train, y_train)

# Lasso 回歸（L1 正則化，可做特徵選擇）
lasso = Lasso(alpha=1.0)
lasso.fit(X_train, y_train)

# 邏輯回歸（分類）
logreg = LogisticRegression()
logreg.fit(X_train, y_train)
predictions = logreg.predict(X_test)
```

### 支持向量機

```python
from sklearn.svm import SVC, SVR
```

### 基於樹的方法

```python
from sklearn.tree import DecisionTreeClassifier
```

### 神經網路

```python
import torch
import torch.nn as nn

class NeuralNetwork(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super().__init__()
        self.layer1 = nn.Linear(input_size, hidden_size)
        self.relu = nn.ReLU()
        self.layer2 = nn.Linear(hidden_size, output_size)
    
    def forward(self, x):
        x = self.layer1(x)
        x = self.relu(x)
        x = self.layer2(x)
        return x

model = NeuralNetwork(784, 256, 10)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

# 訓練循環
for epoch in range(num_epochs):
    outputs = model(inputs)
    loss = criterion(outputs, labels)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

## 常見問題與解決方案

### 過擬合

過擬合是指模型在訓練資料上表現很好，但在測試資料上表現不佳：

```
┌─────────────────────────────────────────────────────────────────┐
│                        過擬合 vs 欠擬合                         │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  過擬合 (Overfitting):                                        │
│  - 模型過於複雜，記憶了訓練資料的雜訊                          │
│  - 訓練誤差低，測試誤差高                                      │
│  - 解決方案：正則化、 dropout、增加資料、 early stopping       │
│                                                              │
│  欠擬合 (Underfitting):                                       │
│  - 模型過於簡單，無法捕捉資料規律                              │
│  - 訓練誤差和測試誤差都高                                      │
│  - 解決方案：增加模型複雜度、增加特徵、減少正則化               │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

```python
# 防止過擬合的方法
# 1. 正則化
ridge = Ridge(alpha=1.0)     # L2 正則化
lasso = Lasso(alpha=1.0)      # L1 正則化

# 2. Dropout (神經網路)
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.Dropout(0.5),         # 隨機丟棄 50% 神經元
    nn.ReLU(),
    nn.Linear(256, 10)
)

# 3. Early Stopping
early_stop = EarlyStopping(patience=5, min_delta=0.01)

# 4. 增加訓練資料
data_augmentation(images, labels)
```

### 類別不平衡

當不同類別的樣本數量差異很大時，模型可能會偏向多數類：

```python
# 處理類別不平衡
# 1. 過採樣
from imblearn.over_sampling import SMOTE
X_resampled, y_resampled = SMOTE().fit_resample(X, y)

# 2. 欠採樣
from imblearn.under_sampling import RandomUnderSampler
X_resampled, y_resampled = RandomUnderSampler().fit_resample(X, y)

# 3. 類別權重
class_weights = {0: 1, 1: 10}  # 給少數類更高權重
model = LogisticRegression(class_weight=class_weights)
```
