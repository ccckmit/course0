# 決策樹 (Decision Tree)

## 概述

決策樹是一種樹狀結構的監督式學習模型，透過遞迴地選擇最佳特徵進行分裂，將資料劃分為越來越純淨的子集合。每個內部節點代表一個特徵上的判斷條件，每個分支代表條件的輸出，每個葉節點代表一個類別（分類樹）或一個數值（回歸樹）。

決策樹的最大優勢是其可解釋性——我們可以將決策過程可視化為一系列「如果-那麼」的規則，這使得它在需要模型可解釋性的應用場景中非常受歡迎，例如醫療診斷、金融風控等領域。

## 原理

決策樹的核心理念是「分而治之」（Divide and Conquer）。從根節點開始，我們選擇一個特徵和分裂條件，將資料劃分為多個子集合。這個過程在每個子集合上遞迴重複，直到達到停止條件。

決策樹的學習過程可以形式化為以下問題：**給定一個資料集，應該選擇哪個特徵來進行分裂？** 這個問題的答案取決於我們如何定義「最好的分裂」。一般來說，我們希望每次分裂後，子集合的「純淨度」能夠提高。

## 不純度度量

要衡量分裂的好壞，我們需要一個度量「純淨度」的指標。常用的有三種：

### 1. 熵 (Entropy)

熵源自資訊理論，表示資料的不確定性或混亂程度。熵越高，資料越混亂；熵越低，資料越純淨。當所有樣本屬於同一類別時，熵為 0。

$$H(S) = -\sum_{i=1}^{c} p_i \log_2(p_i)$$

其中 $p_i$ 是類別 $i$ 在集合 $S$ 中的比例，$c$ 是類別總數。

```python
import numpy as np

def entropy(y):
    """計算熵"""
    if len(y) == 0:
        return 0
    
    counts = np.bincount(y)
    probs = counts[counts > 0] / len(y)
    return -np.sum(probs * np.log2(probs))

# 範例
# [1, 1, 1, 1] -> entropy = 0（完全純淨）
# [0, 1, 0, 1] -> entropy = 1（完全混亂）
```

### 2. 基尼不純度 (Gini Impurity)

基尼不純度是 CART（Classification and Regression Trees）演算法使用的度量。它測量從集合中隨機選擇兩個樣本，它們被錯誤分類的機率。

$$Gini(S) = 1 - \sum_{i=1}^{c} p_i^2$$

基尼不純度的範圍是 0 到 0.5，熵的範圍是 0 到 1。兩者在本質上非常相似，在實際應用中通常選擇哪一個影響不大。

```python
def gini_impurity(y):
    """計算基尼不純度"""
    if len(y) == 0:
        return 0
    
    counts = np.bincount(y)
    probs = counts / len(y)
    return 1 - np.sum(probs ** 2)
```

### 3. 分類錯誤率 (Classification Error)

這是最簡單的度量，但較少使用：

$$Error(S) = 1 - \max_i(p_i)$$

## 資訊增益 (Information Gain)

資訊增益衡量的是進行一次分裂後，父節點熵的減少量。我們選擇資訊增益最大的特徵進行分裂。

$$IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v)$$

其中：
- $A$ 是要評估的特徵
- $Values(A)$ 是特徵 $A$ 的所有可能值
- $S_v$ 是特徵 $A$ 的值為 $v$ 的子集合

```python
def information_gain(X, y, feature_idx, threshold):
    """計算資訊增益"""
    # 母節點熵
    parent_entropy = entropy(y)
    
    # 分裂
    left_mask = X[:, feature_idx] <= threshold
    right_mask = ~left_mask
    
    if np.sum(left_mask) == 0 or np.sum(right_mask) == 0:
        return 0
    
    # 子節點加權熵
    n = len(y)
    n_left, n_right = np.sum(left_mask), np.sum(right_mask)
    
    child_entropy = (n_left / n) * entropy(y[left_mask]) + \
                    (n_right / n) * entropy(y[right_mask])
    
    return parent_entropy - child_entropy
```

### 連續特徵的分裂

對於連續特徵，我們需要找到最佳的分裂點。通常的做法是：
1. 對特徵值進行排序
2. 嘗試每對相鄰值的中點作為分裂點
3. 選擇資訊增益最大的點

```python
def find_best_split(X, y, feature_idx):
    """找到連續特徵的最佳分裂點"""
    values = X[:, feature_idx]
    unique_values = np.unique(values)
    
    if len(unique_values) <= 1:
        return None, 0
    
    # 嘗試所有可能的分裂點
    best_gain = -1
    best_threshold = None
    
    for i in range(len(unique_values) - 1):
        threshold = (unique_values[i] + unique_values[i + 1]) / 2
        gain = information_gain(X, y, feature_idx, threshold)
        
        if gain > best_gain:
            best_gain = gain
            best_threshold = threshold
    
    return best_threshold, best_gain
```

## 決策樹建構流程

```
決策樹建構演算法：

1. 從根節點開始，將所有訓練資料放入此節點
2. 如果節點滿足停止條件，則標記為葉節點並返回
3. 對每個特徵：
   - 如果是類別特徵：評估每個類別作為分裂
   - 如果是連續特徵：找到最佳分裂點
   - 計算分裂後的資訊增益
4. 選擇資訊增益最大的特徵/分裂點
5. 建立子節點，遞迴執行步驟 2-5

停止條件：
- 所有樣本屬於同一類別
- 沒有特徵可供分裂
- 達到最大深度
- 節點樣本數少於最小分裂樣本數
```

## 完整實現

```python
import numpy as np

class DecisionTreeClassifier:
    """決策樹分類器"""
    
    def __init__(self, max_depth=10, min_samples_split=2, 
                 min_samples_leaf=1, criterion='gini'):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.min_samples_leaf = min_samples_leaf
        self.criterion = criterion
        self.tree = None
    
    def _impurity(self, y):
        """計算不純度"""
        if self.criterion == 'gini':
            return self._gini(y)
        else:
            return self._entropy(y)
    
    def _gini(self, y):
        """基尼不純度"""
        if len(y) == 0:
            return 0
        counts = np.bincount(y)
        probs = counts / len(y)
        return 1 - np.sum(probs ** 2)
    
    def _entropy(self, y):
        """熵"""
        if len(y) == 0:
            return 0
        counts = np.bincount(y)
        probs = counts[counts > 0] / len(y)
        return -np.sum(probs * np.log2(probs))
    
    def _information_gain(self, y, y_left, y_right):
        """計算資訊增益"""
        n = len(y)
        if n == 0:
            return 0
        
        n_left, n_right = len(y_left), len(y_right)
        
        if n_left == 0 or n_right == 0:
            return 0
        
        parent_impurity = self._impurity(y)
        child_impurity = (n_left / n) * self._impurity(y_left) + \
                         (n_right / n) * self._impurity(y_right)
        
        return parent_impurity - child_impurity
    
    def _find_best_split(self, X, y):
        """找到最佳分裂"""
        best_gain = -1
        best_feature = None
        best_threshold = None
        
        n_features = X.shape[1]
        
        for feature in range(n_features):
            # 獲取該特徵的所有唯一值
            thresholds = np.unique(X[:, feature])
            
            # 嘗試每個可能的分裂點
            for i in range(len(thresholds) - 1):
                threshold = (thresholds[i] + thresholds[i + 1]) / 2
                
                left_mask = X[:, feature] <= threshold
                right_mask = ~left_mask
                
                # 檢查是否滿足最小葉節點大小
                if np.sum(left_mask) < self.min_samples_leaf or \
                   np.sum(right_mask) < self.min_samples_leaf:
                    continue
                
                y_left = y[left_mask]
                y_right = y[right_mask]
                
                gain = self._information_gain(y, y_left, y_right)
                
                if gain > best_gain:
                    best_gain = gain
                    best_feature = feature
                    best_threshold = threshold
        
        return best_feature, best_threshold, best_gain
    
    def _build_tree(self, X, y, depth=0):
        """遞迴建構樹"""
        n_samples = len(y)
        
        # 停止條件
        if (depth >= self.max_depth or 
            n_samples < self.min_samples_split or
            len(np.unique(y)) == 1):
            # 葉節點：返回最常見的類別
            return {'leaf': True, 'class': np.bincount(y).argmax()}
        
        # 找最佳分裂
        feature, threshold, gain = self._find_best_split(X, y)
        
        # 如果沒有找到有效的分裂，返回葉節點
        if feature is None or gain <= 0:
            return {'leaf': True, 'class': np.bincount(y).argmax()}
        
        # 分裂資料
        left_mask = X[:, feature] <= threshold
        right_mask = ~left_mask
        
        # 遞迴建構子樹
        left_tree = self._build_tree(X[left_mask], y[left_mask], depth + 1)
        right_tree = self._build_tree(X[right_mask], y[right_mask], depth + 1)
        
        return {
            'leaf': False,
            'feature': feature,
            'threshold': threshold,
            'left': left_tree,
            'right': right_tree
        }
    
    def fit(self, X, y):
        """訓練模型"""
        self.tree = self._build_tree(X, y)
        return self
    
    def _predict_single(self, x, node):
        """預測單個樣本"""
        if node['leaf']:
            return node['class']
        
        if x[node['feature']] <= node['threshold']:
            return self._predict_single(x, node['left'])
        else:
            return self._predict_single(x, node['right'])
    
    def predict(self, X):
        """預測"""
        return np.array([self._predict_single(x, self.tree) for x in X])
```

## 使用 sklearn

```python
from sklearn.tree import DecisionTreeClassifier, plot_tree
import matplotlib.pyplot as plt

# 建立模型
clf = DecisionTreeClassifier(
    max_depth=5,           # 最大深度
    min_samples_split=10,  # 內部節點最小樣本數
    min_samples_leaf=5,    # 葉節點最小樣本數
    criterion='gini',      # 'gini' 或 'entropy'
    random_state=42
)

clf.fit(X_train, y_train)

# 預測
predictions = clf.predict(X_test)

# 評估
from sklearn.metrics import accuracy_score
accuracy = accuracy_score(y_test, predictions)

plt.figure(figsize=(20, 10))

          feature_names=['feature1', 'feature2', 'feature3'],

importances = clf.feature_importances_
```

## 防止過擬合

決策樹容易過擬合，特別是當樹深度過大時。以下是常見的正則化策略：

### 1. 預剪枝 (Pre-pruning)

在建樹過程中提前停止：
- 限制最大深度 (`max_depth`)
- 限制最小樣本數 (`min_samples_split`, `min_samples_leaf`)
- 限制最小資訊增益 (`min_impurity_decrease`)

### 2. 後剪枝 (Post-pruning)

先建構完整樹，然後移除不重要的節點：

```python
# sklearn 的 cost-complexity pruning
clf = DecisionTreeClassifier(random_state=42)
path = clf.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas = path.ccp_alphas

# 選擇最佳的 alpha
best_alpha = ccp_alphas[np.argmax(path.impurities[:-1])]
clf = DecisionTreeClassifier(ccp_alpha=best_alpha, random_state=42)
clf.fit(X_train, y_train)
```

## 回歸樹

決策樹也可以用於回歸任務，稱為回歸樹。不同之處在於：
- 葉節點不是類別，而是預測值（通常是該葉節點樣本的均值）
- 使用 MSE（均方誤差）作為分裂準則

```python
from sklearn.tree import DecisionTreeRegressor

reg = DecisionTreeRegressor(max_depth=5, random_state=42)
reg.fit(X_train, y_train)
predictions = reg.predict(X_test)
```

## 演算法變體

| 演算法 | 分裂準則 | 支援類型 | 特色 |
|--------|----------|----------|------|
| ID3 | 資訊增益 | 分類 | 只能處理類別特徵 |
| C4.5 | 資訊增益率 | 分類 | 支援連續特徵、處理缺失值 |
| CART | 基尼不純度/MSE | 分類+回歸 | 二叉樹、支援特徵重要性 |

## 決策樹的優點與缺點

### 優點
- **可解釋性強**：可以轉換為人類可理解的規則
- **不需要特徵縮放**：對特徵的尺度不敏感
- **能處理類別和數值特徵**
- **能處理多分類問題**
- **訓練時間相對較短**

### 缺點
- **容易過擬合**：需要仔細調整參數
- **對資料的小變化敏感**：可能產生完全不同的樹
- **傾向於偏向具有更多等級的特徵**
- **無法很好地外推到訓練資料範圍外的值**
