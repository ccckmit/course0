# 強化學習

強化學習（Reinforcement Learning，RL）是機器學習的一個重要分支，專注於讓智慧體（Agent）透過與環境的互動學習如何做出最佳決策。與監督式學習不同，強化學習不依賴帶有標籤的資料，而是透過試誤（Trial and Error）的方式，根據環境給予的獎賞或懲罰來學習。智慧體需要在探索（嘗試新動作）和利用（使用已知的最佳動作）之間取得平衡，這是強化學習的核心挑戰之一。強化學習在遊戲、機器人控制、推薦系統等領域有著廣泛的應用。

## 強化學習的核心概念

### 基本框架

強化學習的基本框架涉及智慧體、環境、狀態、動作、獎賞和策略之間的互動：

```
┌─────────────────────────────────────────────────────────────────┐
│                      強化學習框架                               │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│                    ┌───────────────┐                         │
│           ┌──────►│   智慧體      │◄──────┐                  │
│           │       │   (Agent)     │       │                  │
│           │       └───────┬───────┘       │                  │
│           │               │               │                  │
│           │               ▼               │                  │
│           │       ┌───────────────┐        │                  │
│           │  動作 │    環境       │ 狀態   │                  │
│           │       │ (Environment) │+獎賞  │                  │
│           │       └───────────────┘        │                  │
│           │               │               │                  │
│           │               ▼               │                  │
│           │       ┌───────────────┐        │                  │
│           └───────│   獎賞        │        │                  │
│                   │   (Reward)    │────────┘                  │
│                   └───────────────┘                           │
│                                                              │
│  馬可夫決策過程 (MDP):                                        │
│  • 狀態空間 S                                                │
│  • 動作空間 A                                                │
│  • 轉移機率 P(s'|s, a)                                       │
│  • 獎賞函數 R(s, a, s')                                      │
│  • 折扣因子 γ                                                │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 核心術語

```python
# 強化學習基本元素

# 狀態 (State): 環境的當前描述
# s = [位置, 速度, 周圍障礙]

# 動作 (Action): 智慧體可以執行的操作
# a = [前進, 後退, 左轉, 右轉, 加速, 減速]

# 獎賞 (Reward): 動作的立即回饋
# r = +1 得分, r = -1 失敗, r = 0 無變化

# 策略 (Policy): 狀態到動作的映射
# π(a|s) = P(a|s)  # 在狀態 s 執行動作 a 的機率

# 累積獎賞 (Return): 長期回報
# G = r₁ + γr₂ + γ²r₃ + ...  # γ 是折扣因子

# 價值函數 (Value Function): 狀態的長期價值
# Vπ(s) = E[G | s, π]  # 從狀態 s 開始的期望累積獎賞

# Q 函數 (Action-Value Function): 狀態-動作對的價值
# Qπ(s, a) = E[G | s, a, π]  # 在狀態 s 執行動作 a 的期望獎賞
```

## 學習流程

強化學習的訓練過程是一個持續互動和更新的過程：

### 互動-更新循環

```python
# 強化學習訓練迴圈

for episode in range(num_episodes):
    state = env.reset()  # 初始化環境
    
    for step in range(max_steps):
        # 1. 根據策略選擇動作
        action = agent.select_action(state)
        
        # 2. 執行動作，獲得獎賞和新狀態
        next_state, reward, done, info = env.step(action)
        
        # 3. 儲存經驗
        agent.store_transition(state, action, reward, next_state, done)
        
        # 4. 更新策略
        agent.update()
        
        # 5. 更新狀態
        state = next_state
        
        # 6. 檢查是否結束
        if done:
            break
    
    # 衰减探索率
    agent.decay_epsilon()
```

### 探索與利用

強化學習面臨的核心挑戰是如何平衡探索新動作和利用已知的最佳動作：

```
┌─────────────────────────────────────────────────────────────────┐
│                      探索 vs 利用                               │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  探索 (Exploration):                                          │
│  • 嘗試新的動作                                               │
│  • 發現潛在更好的策略                                         │
│  • 需要犧牲短期獎賞                                            │
│                                                              │
│  利用 (Exploitation):                                         │
│  • 選擇已知最好的動作                                         │
│  • 最大化當前預期獎賞                                         │
│  • 可能錯過更好的選項                                          │
│                                                              │
│  常見策略：                                                   │
│  • ε-greedy: 以 ε 機率隨機選擇，1-ε 選擇最佳                  │
│  • UCB: 平衡獎賞與不確定性                                    │
│  • Thompson Sampling: 機率抽樣                               │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 常見演算法

### 基於價值的演算法

基於價值的演算法學習估算每個狀態或狀態-動作對的價值：

#### Q-Learning

Q-Learning 是最經典的基於價值的演算法：

```python
import numpy as np

class QLearningAgent:
    def __init__(self, n_states, n_actions, learning_rate=0.1, gamma=0.95):
        self.n_states = n_states
        self.n_actions = n_actions
        self.lr = learning_rate
        self.gamma = gamma
        self.q_table = np.zeros((n_states, n_actions))
    
    def select_action(self, state, epsilon=0.1):
        if np.random.random() < epsilon:
            return np.random.randint(self.n_actions)
        else:
            return np.argmax(self.q_table[state])
    
    def update(self, state, action, reward, next_state):
        # Q-Learning 更新規則
        current_q = self.q_table[state, action]
        max_next_q = np.max(self.q_table[next_state])
        target_q = reward + self.gamma * max_next_q
        self.q_table[state, action] += self.lr * (target_q - current_q)
```

### 基於策略的演算法

基於策略的演算法直接學習最優策略：

```python
class PolicyNetwork(nn.Module):

            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim)
        )
    
    def forward(self, x):
        return F.softmax(self.net(x), dim=-1)
```

#### REINFORCE

REINFORCE 是基礎的策略梯度演算法：

```python
# REINFORCE 演算法
for episode in range(num_episodes):
    trajectory = []
    state = env.reset()
    
    # 收集軌跡
    while not done:
        probs = policy(state)
        action = torch.multinomial(probs, 1)
        next_state, reward, done = env.step(action)
        trajectory.append((state, action, reward))
        state = next_state
    
    # 計算回報
    G = 0
    for t in reversed(trajectory):
        state, action, reward = t
        G = gamma * G + reward
        
        # 更新策略
        log_prob = torch.log(policy(state)[action])
        loss = -log_prob * G
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

#### Proximal Policy Optimization (PPO)

PPO 是一種穩定的策略梯度演算法：

```python
# PPO 目標函數
def ppo_loss(old_log_probs, new_log_probs, advantages, clip_eps=0.2):
    ratio = torch.exp(new_log_probs - old_log_probs)
    clipped = torch.clamp(ratio, 1 - clip_eps, 1 + clip_eps)
    loss = -torch.min(ratio * advantages, clipped * advantages)
    return loss.mean()

# PPO 訓練循環
```

### 基於模型的演算法

基於模型的演算法學習環境的模型，然後規劃：

#### Monte Carlo Tree Search (MCTS)

MCTS 用於離散動作空間的規劃：

```python
class MCTS:
    def __init__(self, game):
        self.game = game
        self.root = None
    
    def search(self, state, num_iterations):
        root = Node(state)
        
        for _ in range(num_iterations):
            node = root
            
            # 1. 選擇（UCB）
            while node.is_expanded():
                node = node.best_child()
            
            # 2. 展開
            if not node.is_terminal():
                node = node.expand()
            
            # 3. 模擬
            reward = node.simulate()
            
            # 4. 回傳
            node.backpropagate(reward)
        
        return root.best_action()
```

## 獎賞設計

獎賞函數的設計對強化學習的成功至關重要：

### 獎賞類型

```python
# 稀疏獎賞 (Sparse Reward)
# 只在完成目標時給予獎賞
if goal_reached:
    reward = 100
else:
    reward = 0

# 密集獎賞 (Dense Reward)
# 每一步都給予獎賞
reward = -0.01  # 鼓勵快速完成
reward += distance_reward  # 靠近目標
reward += time_penalty  # 時間 penalty

# 形狀獎賞 (Shaped Reward)
# 提供逐步改進的信號
reward = current_distance - previous_distance  # 距離進步

# 問題：
# • 獎賞腐化：智慧體找到捷徑
# • 局部最佳：無法發現更好的解決方案
```

### 獎賞塑造

```
好的獎賞塑造原則：
• 與最終目標一致
• 提供有意義的梯度
• 避免不必要的約束
• 保持簡單
```

## 挑戰與解決方案

### 主要挑戰

```
強化學習面臨的挑戰：

1. 樣本效率低
   • 需要大量環境互動
   • 解決：經驗回放、離策略學習

2. 不穩定訓練
   • 高方差更新
   • 解決：目標網路、梯度裁剪

3. 稀疏獎賞
   • 難以學習長期規劃
   • 解決：層次 RL、獎賞塑造

4. 探索不足
   • 陷入局部最佳
   • 解決：熵正則化、UCB

5. 轉移學習
   • 環境變化需要重新訓練
   • 解決：多任務學習、元學習
```

### 解決方案

```python
# 經驗回放
```
