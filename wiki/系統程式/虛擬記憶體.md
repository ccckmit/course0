# 虛擬記憶體

虛擬記憶體（Virtual Memory）是作業系統提供的一種記憶體管理技術，它為每個程式提供獨立的、遠大於實體記憶體的位址空間。虛擬記憶體通過硬體和軟體的緊密配合，將程式的虛擬位址翻譯為實體記憶體（或磁碟）的實際位址，實現了記憶體的抽象化與保護。

## 虛擬記憶體的由來

早期電腦沒有虛擬記憶體時，程式直接訪問實體記憶體位址。這帶來諸多問題：

**位址空間限制**：程式能使用的記憶體受限於實際安裝的記憶體大小。若記憶體為 64KB，則即使硬碟有數百 GB 空間，程式也無法使用。

**程式間衝突**：若兩個程式都使用絕對位址（如 0x1000），則無法同時執行。必須精心安排各程式的記憶體布局，繁瑣且容易出錯。

**安全性問題**：一個程式可以訪問另一程式的記憶體，無任何隔離。

1960 年代，Manchester University 的 Atlas 電腦首次實現了分頁式虛擬記憶體的概念，此後這項技術成為現代作業系統的標配。

## 虛擬位址空間

在 32 位元系統中，每個程式擁有 4GB（2³² 位元組）的虛擬位址空間：

```
0x00000000                    0xFFFFFFFF
├────────────┴───────────────────────────┤
│           使用者空間 (0-3GB)            │
│  ├────────────────┤                   │
│  │     Text       │ 程式碼段             │
│  ├────────────────┤                   │
│  │     Data      │ 已初始化資料段        │
│  ├────────────────┤                   │
│  │     BSS       │ 未初始化資料段        │
│  ├────────────────┤                   │
│  │     Heap      │ 動態配置 (向上生長)  │
│     ...                                  │
│     ...                                  │
│  ├────────────────┤                   │
│  │     Stack     │ 函式呼叫 (向下生長)  │
│  └────────────────┘                   │
├────────────────────────────────────────┤
│           核心空間 (3-4GB)              │
│  核心映像、核心資料結構、系統呼叫介面     │
└────────────────────────────────────────┘
```

在 64 位元系統中，理論上位址空間為 2⁶⁴，但實際實現通常只使用其中一部分（如 Linux 使用 48 位元虛擬位址，支援 256TB 位址空間）。

## 分頁機制

虛擬記憶體透過分頁（Paging）機制將虛擬位址空間劃分為固定大小的頁（Page），將實體記憶體劃分為相同大小的頁框（Page Frame）。

### 虛擬位址翻譯

虛擬位址由兩部分組成：
- **虛擬頁號（Virtual Page Number, VPN）**：高位部分，用於索引頁表
- **頁內偏移（Offset）**：低位部分，直接對應實體頁框內的位元組位置

```
┌────────────────────┬───────────────────┐
│ 虛擬頁號 (VPN)      │ 頁內偏移 (Offset)  │
└────────────────────┴───────────────────┘
         │                    │
         ↓                    │
    ┌─────────┐               │
    │  頁表    │               │
    │         │               │
    │ VPN→PFN │               │
    │         │               │
    └────┬────┘               │
         │                    │
         ↓                    ↓
┌────────────────────┬───────────────────┐
│ 實體頁框號 (PFN)    │ 頁內偏移 (Offset)  │
└────────────────────┴───────────────────┘
```

### 頁表結構

頁表（Page Table）是核心資料結構，儲存虛擬頁到實體頁框的映射：

```c
// 簡化的頁表項結構
struct PageTableEntry {
    unsigned int present     : 1;   // 頁是否在記憶體
    unsigned int writable   : 1;   // 是否可寫
    unsigned int user       : 1;   // 使用者空間可訪問
    unsigned int accessed   : 1;   // 已被訪問
    unsigned int dirty      : 1;   // 已被修改
    unsigned int frame      : 20;  // 實體頁框號
};
```

### 多級頁表

為節省記憶體，現代系統使用多級頁表：

```
      虛擬位址 (虛擬頁號 + 偏移)
                │
                ↓
    ┌─────────────────────┐
    │  一級頁表 (頁目錄)    │
    │  (核心空間，恆駐記憶體) │
    └──────────┬──────────┘
               │ 讀取一級頁表項
               ↓
    ┌─────────────────────┐
    │  二級頁表             │
    │  (按需分配，可在磁碟)   │
    └──────────┬──────────┘
               │ 讀取二級頁表項
               ↓
         頁框號 + 偏移 → 實體位址
```

Linux 使用四級頁表（PGD → PUD → PMD → PTE），支援從 32 位元到 64 位元的各種架構。

## 分頁硬體：MMU

記憶體管理單元（Memory Management Unit, MMU）是專門負責位址翻譯的硬體模組：

```
CPU → 虛擬位址 → MMU → 實體位址 → 主記憶體
                    │
                    ├→ TLB (Translation Lookaside Buffer)
                    │     快取最近的翻譯結果
                    │
                    └→ 頁表走查
                          │
                          ├→ 頁面命中
                          │
                          └→ 頁面缺失 (Page Fault)
                                │
                                └→ OS 處理
```

### TLB 快取

TLB 是硬體快取，儲存最近使用的虛擬頁到實體頁框的映射：

- **命中率（Hit Rate）**：通常 95-99%
- **Miss Penalty**：TLB miss 需要查頁表，數十到數百個時鐘週期
- **關聯式快取**：使用內容定址，支援並行比對

## 分頁替換演算法

當需要將頁面載入記憶體但記憶體已滿時，必須選擇一個頁面換出到磁碟。這稱為分頁替換（Page Replacement）。

### 最佳替換（OPT）

理論最佳策略，選擇最長時間不被使用的頁面替換。但需要預知未來，無法實際實現，用作評估基準。

### 先進先出（FIFO）

最簡單的策略，置換最早進入記憶體的頁面：

```python
def fifo_page_replacement(pages, frames):
    frame_list = []
    page_faults = 0
    
    for page in pages:
        if page not in frame_list:
            if len(frame_list) < frames:
                frame_list.append(page)
            else:
                frame_list.pop(0)  # 移除最舊的
                frame_list.append(page)
            page_faults += 1
    
    return page_faults
```

問題：Belady 異常（增加頁框數反而增加缺頁率）。

### 最近最少使用（LRU）

選擇最長時間未被使用的頁面：

```python
def lru_page_replacement(pages, frames):
    frame_list = []
    page_faults = 0
    
    for page in pages:
        if page in frame_list:
            frame_list.remove(page)  # 移出並重新加入
            frame_list.append(page)   # 表示最近使用
        else:
            if len(frame_list) >= frames:
                frame_list.pop(0)  # 移除最久未使用的
            frame_list.append(page)
            page_faults += 1
    
    return page_faults
```

問題：需要記錄每次頁面訪問時間，開銷較大。

### 時鐘演算法（Clock）

LRU 的近似實現，使用參考位（Reference Bit）：

- 每頁有一個使用位（used bit）
- 時鐘指標循環檢查頁面
- 若 used bit = 0，替換此頁；否則設為 0，繼續檢查

```c
// 時鐘演算法偽代碼
while (true) {
    Page *p = clock_pointer;
    
    if (p->used == 0) {
        replace(p);  // 替換此頁
        clock_pointer = p->next;
        break;
    } else {
        p->used = 0;  // 重置使用位
        clock_pointer = p->next;
    }
}
```

## 交換空間

當實體記憶體不足時，作業系統將不活躍的頁面換出到磁碟的交換空間（Swap Space）：

```
┌─────────────────┐     ┌─────────────────┐
│  實體記憶體      │ ←→  │   交換空間       │
│                 │     │   (磁碟)         │
│  活躍頁面       │     │  不活躍頁面      │
└─────────────────┘     └─────────────────┘
```

Linux 中使用 `swap` 命令建立交換空間：
```bash
sudo fallocate -l 8G /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

## 核心空間與使用者空間

虛擬記憶體的另一關鍵功能是隔離核心空間和使用者空間：

**使用者空間（User Space）**：
- 每個行程獨立位址空間
- 應用程式執行區域
- 嘗試訪問核心空間會導致段錯誤（Segmentation Fault）

**核心空間（Kernel Space）**：
- 所有行程共用
- 核心程式碼和資料結構
- 處理系統呼叫、例外、中斷

### 系統呼叫的記憶體切換

當使用者程式執行系統呼叫時：

```
使用者空間 → 系統呼叫介面 → 核心空間處理 → 返回使用者空間
```

這涉及位址空間的切換，需要刷新 TLB（部分或全部）。

## 記憶體映射檔案

記憶體映射檔案（Memory-Mapped Files）將檔案內容直接映射到虛擬記憶體：

```c
#include <sys/mman.h>

int fd = open("data.bin", O_RDONLY);
void *addr = mmap(NULL, 4096, PROT_READ, MAP_PRIVATE, fd, 0);
// 現在可以像訪問指標一樣讀取檔案
// addr[0], addr[1], ...
munmap(addr, 4096);
```

用途：
- 高效讀寫大檔案
- 共享記憶體（多行程通訊）
- 動態連結（共享庫）

## 分頁的優勢與代價

### 優勢

1. **透明記憶體擴展**：程式無需理會實際記憶體大小
2. **記憶體隔離**：行程間互不影響
3. **簡化記憶體分配**：只需配置虛擬位址，OS 處理實體分配
4. **分享記憶體**：共享程式碼段、共享庫
5. **虛擬連續空間**：對程式而言記憶體是連續的

### 代價

1. **位址翻譯開銷**：每次記憶體訪問需額外查詢
2. **頁表記憶體開銷**：多級頁表本身佔用記憶體
3. **分頁交換延遲**：頁面換入/換出時間可觀
4. **TLB miss 懲罰**：快取失效導致效能下降

## 現代虛擬記憶體技術

### 大頁面（Huge Pages）

傳統頁面大小為 4KB，對記憶體密集型應用造成額外開銷。Linux支援大頁面（如 2MB、1GB）：

```bash
# 配置大頁面
echo 10 > /proc/sys/vm/nr_hugepages
```

### 記憶體壓縮（zswap/zram）

當 swap 即將被使用時，先壓縮記憶體內容而非換出到磁碟：

```
記憶體不足 → 壓縮頁面 → 儲存到記憶體中的壓縮池
```

### KSM（Kernel Samepage Merging）

核心掃描記憶體，合並相同內容的頁面（CoW），節省記憶體：

```bash
echo 1 > /sys/kernel/mm/ksm/run
```
