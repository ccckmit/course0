# 檔案系統

檔案系統（File System）是作業系統中負責管理和持久化儲存資料的核心子系統。它提供了從物理儲存設備到邏輯檔案組織的抽象層，使得使用者與應用程式可以透過一致性的介面存取資料，無需關心硬體細節。檔案系統的設計直接影響系統的效能、可靠性與可擴展性。

## 檔案系統的發展歷程

檔案系統經過數十年的演進，從簡單的順序存取發展到現代的高度優化與容錯機制：

- **早期檔案系統（1970s-1980s）**：UNIX 的原始檔案系統（UFS）、DOS 的 FAT12/16，僅支援基本讀寫
- **日誌檔案系統（1990s）**：引入日誌機制防止崩潰損失，Ext3、XFS、NTFS
- **現代檔案系統（2000s-至今）**：支援快照、壓縮、去重、彈性擴展，ZFS、Btrfs、APFS

## 檔案系統結構

### 經典 UNIX 檔案系統架構

```
┌─────────────────────────────────────────────┐
│            檔案系統                       │
├─────────────────────────────────────────────┤
│  超級區塊 (Super Block)                     │
│  - 檔案系統資訊                           │
│  - i-node 表位置                          │
│  - 資料區塊位置                         │
│  - 空閒空間管理                         │
├─────────────────────────────────────────────┤
│  i-node 表 (Index Node)                   │
│  - inode[0]: 閒置                        │
│  - inode[1]: 根目錄                     │
│  - inode[2]: /boot                      │
│  - inode[3]: /dev                       │
│  - ...                                   │
├─────────────────────────────────────────────┤
│  資料區塊 (Data Blocks)                │
│  - 區塊 0                                │
│  - 區塊 1                                │
│  - 區塊 2                                │
│  - ...                                   │
└─────────────────────────────────────────────┘
```

### 超級區塊 (Super Block)

超級區塊是檔案系統的元資料，包含整個檔案系統的關鍵資訊：

```c
struct super_block {
    uint32_t s_inodes_count;     // 總 i-node 數
    uint32_t s_blocks_count;     // 總區塊數
    uint32_t s_free_blocks_count; // 空閒區塊數
    uint32_t s_free_inodes_count; // 空閒 i-node 數
    uint32_t s_first_data_block; // 第一資料區塊
    uint32_t s_log_block_size;   // 區塊大小對數
    uint32_t s_blocks_per_group; // 每組區塊數
    uint32_t s_inodes_per_group; // 每組 i-node 數
    time_t s_lastwrite;        // 最後寫入時間
    uint16_t s_magic;         // 魔數 (0xEF53)
    uint16_t s_state;       // 狀態
    // ... 更多欄位
};
```

### i-node 結構

i-node 是檔案系統的核心資料結構，每個檔案對應一個 i-node：

```c
struct inode {
    uint16_t i_mode;          // 檔案類型與權限
    uint16_t i_uid;           // 所有者 UID
    uint32_t i_size;         // 檔案大小 (bytes)
    uint32_t i_atime;         // 最後存取時間
    uint32_t i_ctime;         // 最後修改時間
    uint32_t i_mtime;         // 最後更改時間
    uint32_t i_dtime;         // 刪除時間
    uint16_t i_gid;           // 所有者 GID
    uint16_t i_links_count;    // 連結數
    uint32_t i_blocks;         // 區塊數
    uint32_t i_block[15];      // 區塊指標 (直接 + 間接)
    uint32_t i_flags;         // 標誌
    // ... 更多欄位
};
```

### 區塊指標與間接尋址

i-node 中最多 15 個區塊指標的分配策略：

```
i_block[0-11]  直接區塊    12 × 4KB = 48KB
i_block[12]    一級間接    1024 × 4KB = 4MB
i_block[13]    二級間接    1024² × 4KB = 4GB
i_block[14]    三級間接    1024³ × 4KB = 4TB
```

這就是 UNIX 檔案系統單一檔案大小上限的由來。

## 常見檔案系統

### Ext4 (Fourth Extended Filesystem)

Linux 最廣泛使用的檔案系統，Ext3 的後繼者：

```bash
# 建立 Ext4 檔案系統
mkfs.ext4 /dev/sdb1

# 掛載
mount /dev/sdb1 /mnt/data

# 特色
# - 日誌 (Journal)
# - 延展分配 (Extents)
# - 區塊對齊
# - 支援最大 1EB 檔案系統、16TB 檔案
```

特色：
- **延展分配**：連續分配多個區塊，減少碎片
- **延遲分配**：寫入時最佳化區塊位置
- **校驗和日誌**：提升可靠性
- **無障礙調整大小**：線上擴展

### XFS

SGI 開發的高效能檔案系統，適合大型檔案與並行 I/O：

```bash
mkfs.xfs /dev/sdb1
mount /dev/sdb1 /mnt/data
```

特色：
- **元資料日誌**：高效恢復
- **分配組**：支援並行操作
- **Extent B+ 樹**：快速搜尋
- **配額管理**：使用者/目錄配額

### Btrfs (B-Tree Filesystem)

新一代 Linux 檔案系統，支援現代功能：

```bash
mkfs.btrfs /dev/sdb1
mount /dev/sdb1 /mnt/data

# 特色功能
btrfs subvolume create /mnt/data/snap    # 建立子卷
btrfs subvolume list /mnt/data         # 列表子卷
btrfs snapshot /mnt/data /mnt/snap1     # 建立快照
```

特色：
- **寫時複製 (COW)**：每次寫入新區塊，歷史可回溯
- **子卷**: 隔離的檔案系統視圖
- **快照**：唯讀/可寫快照
- **線上去重**：重複資料省空間
- **壓縮**：線上壓縮（zstd、lz4）
- ** RAID**：軟體 RAID 支援

### ZFS (Zettabyte File System)

Solaris/OpenZFS 開發的企業級檔案系統：

```bash
# 建立池
zpool create tank /dev/sda /dev/sdb

# 建立檔案系統
zfs create tank/home

# 特色
zfs set compression=on tank/home
zfs set dedup=on tank/home
zfs snapshot tank/home@monday
```

特色：
- **Pooled Storage**：虛擬化儲存池
- **資料完整性**：端到端校驗
- **快照/克隆**：高效複製
- **去重**：內建去重
- **RAID-Z**：軟體 RAID

### FAT / NTFS / APFS

| 檔案系統 | 作業系統 | 特色 |
|----------|----------|------|
| FAT32 | Windows | 跨平臺、簡單、4GB 限制 |
| NTFS | Windows | 日誌、壓縮、配額、加密 |
| exFAT | Windows | 大檔案、跨平臺 |
| APFS | macOS | 加密、快照、空間共享 |
| HFS+ | macOS (舊) | 日誌、相容性 |

## 檔案類型與權限

### Linux 檔案類型

| 前綴 | 類型 | 說明 |
|------|------|------|
| `-` | 普通檔案 | 文字、執行檔、圖像 |
| `d` | 目錄 | 資料夾 |
| `l` | 符號連結 | 捷徑 |
| `b` | 區塊裝置 | 硬碟、SSD |
| `c` | 字元裝置 | 終端機、序列埠 |
| `p` | 具名管道 | 程序間通訊 |
| `s` | socket | 網路通訊 |

### 權限模型

```bash
# 權限格式: drwxr-xr-x
# [類型] [所有者] [羣組] [其他]
# d     rwx       r-x     r-x

# 數值表示
# r=4, w=2, x=1
# 755 = rwxr-xr-x

# 設定權限
chmod 755 file.sh
chmod +x script.sh    # 新增執行權
chmod -w file       # 移除寫入權
```

```c
struct stat {
    st_mode;          // 類型 | 權限
    st_uid;           // 所有者
    st_gid;          // 羣組
    st_size;         // 大小
    st_atime;        // 存取時間
    st_mtime;        // 修改時間
    st_ctime;        // 更改時間
};
```

## 目錄結構

### UNIX 目錄層次

```
/
├── bin/           # 系統執行檔
├── sbin/          # 系統管理執行檔
├── lib/           # 共享庫
├── usr/           # 使用者工具
│   ├── bin/      # 使用者執行檔
│   ├── lib/      # 使用者庫
│   └── share/    # 共享資料
├── home/          # 使用者目錄
│   └── username/ # 使用者檔案
├── etc/           # 設定檔
├── dev/           # 裝置檔案
├── proc/          # 程序資訊 (虛擬)
├── sys/           # 系統資訊 (虛擬)
├── var/           # 變動資料
│   └── log/       # 日誌
└── tmp/           # 暫存檔案
```

### Proc 檔案系統

/proc 是虛擬檔案系統，提供程序與系統資訊：

```bash
# 程序資訊
ls /proc/              # 所有程序
ls /proc/$$/           # 當前程序
cat /proc/cpuinfo      # CPU 資訊
cat /proc/meminfo      # 記憶體
cat /proc/loadavg     # 負載

# 程序目錄結構
/proc/[pid]/
├── cmdline         # 啟動命令
├── environ        # 環境變數
├── status         # 狀態資訊
├── maps           # 記憶體對應
├── fd/            # 開啟的檔案
└── cwd/           # 目前目錄
```

### Sys 檔案系統

/sys 是虛擬檔案系統，暴露核心資料結構：

```bash
# 核心物件
ls /sys/
├── block/          # 區塊裝置
├── bus/           # 匯流排
├── class/         # 裝置類別
├── dev/           # 裝置節點
├── devices/       # 裝置資訊
├── kernel/        # 核心參數
└── module/       # 核心模組
```

## 一切皆檔案

Linux 的核心哲學：所有 I/O 資源都抽象為檔案介面。

### 標準輸入輸出

```c
#include <fcntl.h>

// 標準檔案描述符
// 0: stdin  標準輸入
// 1: stdout 標準輸出
// 2: stderr 標準錯誤
```

### 檔案操作

```c
#include <fcntl.h>
#include <unistd.h>

// 開檔
int fd = open("/path/to/file", O_RDONLY);

// 讀檔
char buf[1024];
read(fd, buf, sizeof(buf));

// 寫檔
write(fd, "hello", 5);

// 搜尋
lseek(fd, 0, SEEK_SET);

// 關檔
close(fd);
```

### 檔案描述符共享

```
程序 A          檔案描述符表
  0  --------> stdin (終端)
  1  --------> stdout (終端)
  2  --------> stderr (終端)
  3  --------> /dev/tty
  4  --------> /var/log/app.log
```

### 裝置檔案

```bash
# 區塊裝置
ls -l /dev/sda*
brw-rw---- 1 root disk 8, 0 /dev/sda

# 字元裝置
ls -l /dev/tty
crw-rw-rw- 1 root tty 5, 0 /dev/tty

# 映像檔
mkdir /mnt/iso
mount -o loop cd.iso /mnt/iso
```

### 程序間通訊

```bash
# 具名管道
mkfifo /tmp/myfifo

# 程序 A: 寫入
echo "data" > /tmp/myfifo

# 程序 B: 讀取
cat /tmp/myfifo
```

## 日誌檔案系統

### 日誌機制

日誌檔案系統透過預寫日誌（WAL）確保一致性：

```
1. 寫入日誌          日誌區塊
2. 複製到目標        資料區塊
3. 標記完成         日誌狀態
```

### 日誌模式

```c
// writeback: 僅日誌元資料，最快但不最安全
// ordered:  日誌元資料，寫入資料先於日誌
// data:     日誌元資料 + 資料，最慢但最安全
mount -o data=ordered /dev/sdb1 /mnt
```

### 恢復過程

```c
// 崩潰後恢復
1. 讀取日誌
2. 識別未完成交易
3. 重做已提交交易
4. 撤銷未提交交易
```

## 虛擬檔案系統 (VFS)

VFS 提供統一的檔案介面，支援多種檔案系統：

```
┌───────────────────────────────���─���───────────┐
│        應用程式                             │
├─────────────────────────────────────────────┤
│        VFS API (open, read, write, ...)      │
├─────────────────────────────────────────────┤
│  Ext4  │  XFS  │  Btrfs  │  NTFS  │  FAT  │
└─────────────────────────────────────────────┘
```

### VFS 結構

```c
struct super_operations {
    struct inode *(*alloc_inode)(struct super_block *);
    void (*destroy_inode)(struct inode *);
    void (*put_super)(struct super_block *);
    int (*write_inode)(struct inode *, int);
    // ...
};

struct inode_operations {
    int (*create)(struct inode *, struct dentry *, int);
    struct dentry *(*lookup)(struct inode *, struct dentry *);
    int (*link)(struct dentry *, struct inode *, struct dentry *);
    int (*unlink)(struct inode *, struct dentry *);
    // ...
};

struct file_operations {
    int (*open)(struct inode *, struct file *);
    int (*read)(struct file *, char __user *, int, loff_t *);
    int (*write)(struct file *, const char __user *, int, loff_t *);
    int (*mmap)(struct file *, struct vm_area_struct *);
    // ...
};
```
