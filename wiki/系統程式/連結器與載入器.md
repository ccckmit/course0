# 連結器與載入器

連結器（Linker）和載入器（Loader）是程式編譯流程的關鍵階段。連結器將多個目標檔案合併為單一可執行檔，載入器則在程式執行時將其載入記憶體。

## 編譯流程

```
原始碼 (.c)     目標檔 (.o)     可執行檔      記憶體
     │                │              │            │
     ▼                ▼              ▼            │
┌─────────┐      ┌─────────┐   ┌─────────┐   │
│  前端    │      │         │   │         │   │
│ (詞法/   │─────→│  編譯器  │──→│ 連結器  │──→│ 載入器 │
│  語法/   │      │         │   │         │   │
│  語意)   │      └─────────┘   └─────────┘   │
│         │                           │            │
│         │    ┌─────────┐          │            │
└─────────┘    │ 符號表  │          │            │
               └─────────┘          │            │
               ┌─────────┐          │            │
               │ 除錯資訊 │          │            │
               └─────────┘          │            │
                                   │            ▼
                               ┌─────────┐  ┌─────────┐
                               │ 符號表   │  │         │
                               │ 重定位表  │  │  載入    │
                               └─────────┘  │         │
                                            └─────────┘
```

## 目標檔格式

### ELF（Executable and Linkable Format）

Linux 和許多 Unix 系統使用的標準格式：

```bash
# 查看 ELF 檔案結構
readelf -h program

# 輸出範例：
# Magic: 7f 45 4c 46 02 01 01 00 00 00 00 00 00 00 00 00
# Class: ELF64
# Data: 2's complement, little endian
# Type: EXEC (Executable file)
# Machine: Advanced Micro Devices X86-64
```

### ELF 段（Sections）

```bash
# 列出所有段
readelf -S program.o

# 常見段：
# .text    - 可執行指令
# .data    - 已初始化全域/靜態變數
# .bss     - 未初始化全域/靜態變數
# .rodata  - 唯讀資料（常數）
# .symtab  - 符號表
# .strtab  - 字串表
# .reloc   - 重定位資訊
```

### ELF 程式頭（Program Headers）

```bash
# 列出程式頭
readelf -l program

# 輸出範例：
# Type           Offset   VirtAddr           PhysAddr
#                 FileSiz  MemSiz   Flg  Align
# PHDR           0x000040 0x0000000000400040 ...
# INTERP         0x0002a8 0x00000000004002a8 ...
# LOAD           0x000000 0x0000000000400000 ...
# DYNAMIC        0x000dd8 0x0000000000400dd8 ...
```

## 符號管理

### 符號種類

| 種類 | 說明 |
|------|------|
| 全域符號 | 定義在模組，可被其他模組引用 |
| 外部符號 | 被引用但未定義，需要連結其他模組 |
| 靜態符號 | 只在定義模組可見 |
| 區域符號 | 函式內的區域變數（通常不在符號表中） |

### 符號表

```bash
# 查看符號表
nm program.o

# 輸出範例：
#                  U puts                    # U = 未定義（外部）
# 0000000000000000 T main                    # T = 全域文字（已定義）
# 0000000000000000 D global_var             # D = 全域資料
# 0000000000000000 b .bss                   # b = 區域 BSS
```

### 弱符號與強符號

```c
// 強符號：已初始化的全域變數、函式定義
int global = 1;        // 強符號
void func() {}          // 強符號

// 弱符號：未初始化或標記為 __attribute__((weak))
__attribute__((weak)) int weak_var;  // 弱符號

// 連結規則：多個強符號是錯誤
// 強符號覆蓋弱符號
// 多個弱符號時任選其一
```

## 連結過程

### 符號解析

```c
// main.c
extern int shared_var;    // 外部參照（需要解析）
void helper();           // 外部函式參照

int main() {
    helper();
    return shared_var;
}

// helper.c
int shared_var = 42;     // 定義
void helper() {          // 定義
    shared_var++;
}

// 連結器將 main.o 中的 shared_var 和 helper 
// 與 helper.o 中的定義關聯
```

### 重定位

連結器更新目標檔中的相對位址為最終位址：

```bash
# 查看重定位表
readelf -r program.o

# 輸出範例：
# Relocation section '.rel.text' at offset 0x40:
#    Offset          Info           Type           Sym.Value  Sym.Name
# 000000000008  000500000011 R_X86_64_PC32   0000000000000000 shared_var
# 00000000000e  000a0000000a R_X86_64_PLT32 0000000000000000 helper
```

### 符號解析順序

```bash
# 連結器從左到右解析符號
# 定義來自最後出現的目標檔

# 錯誤：找不到符號
ld a.o b.o      # b.o 引用 a.o 的符號，但順序錯誤
ld b.o a.o      # 正確：引用方在定義方之前

# 解決：多次列出或使用 --start-group
ld a.o b.o a.o  # 重複 a.o
```

## 函式庫

### 靜態函式庫

```bash
# 建立靜態函式庫
gcc -c math.c -o math.o
gcc -c utils.c -o utils.o
ar rcs libmylib.a math.o utils.o

# 列出函式庫內容
ar t libmylib.a

# 使用靜態函式庫
gcc main.o -L. -lmylib -o program

# 從函式庫提取單一成員
ar x libmylib.a math.o
```

### 動態函式庫

```bash
# 建立動態函式庫
gcc -fPIC -c math.c -o math.o
gcc -fPIC -c utils.c -o utils.o
gcc -shared math.o utils.o -o libmylib.so

# -fPIC 生成位置無關碼（Position Independent Code）
```

### 動態連結

```bash
# 編譯時連結動態函式庫
gcc main.o -L. -lmylib -o program

# 執行時需要讓連結器找到函式庫
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH
./program

# 或使用 rpath
gcc main.o -L. -lmylib -Wl,-rpath,'$ORIGIN/lib' -o program
```

## 載入器

### 載入過程

```
1. 讀取程式頭，確定需要映射的段
2. 檢查是否有足夠虛擬記憶體
3. 建立虛擬位址空間
4. 映射段的檔案內容到記憶體
5. 處理重定位
6. 設定堆疊和環境變數
7. 啟動執行緒，傳遞控制權
```

### 延遲載入（Lazy Binding）

```c
// 動態符號在第一次呼叫時才解析
// PLT (Procedure Linkage Table) 實現

// 第一次呼叫：
# 1. 呼叫 PLT 條目
# 2. PLT 跳轉到動態連結器
# 3. 動態連結器解析符號
# 4. 更新 GOT（Global Offset Table）
# 5. 跳轉到實際函式

// 後續呼叫：
# 直接從 GOT 取得已解析的位址並跳轉
```

## 符號衝突與名稱改變

### 名稱改變（Name Mangling）

C++ 支援函式過載，需要名稱改變來區分：

```bash
# C++ 名稱改變範例
c++filt _Z4funcRKiP6vectorliE
# 輸出：func(int const&, std::vector<long, std::allocator<long> >)

// C 連結
extern "C" {
    void foo();  // 不改名稱
}
```

### static 和匿名命名空間

```c
// static 函式只有內部連結
static void internal_func() {}

// 匿名命名空間（C++）
namespace {
    void helper() {}  // 只有內部連結
}
```

## 控制連結器

### 指令稿

```ld
/* linker.ld */

ENTRY(_start)

SECTIONS
{
    . = 0x400000;  /* 基址 */
    
    .text : {
        *(.text)
        *(.text.*)
    }
    
    .rodata : {
        *(.rodata)
        *(.rodata.*)
    }
    
    .data : {
        *(.data)
        *(.data.*)
    }
    
    .bss : {
        *(.bss)
        *(.bss.*)
        *(COMMON)
    }
}

/* 連結時使用 */
ld -T linker.ld a.o b.o -o program
```

### 常用連結器選項

```bash
# 指定輸出格式
ld -o output a.o b.o           # 輸出檔名
ld -e entry_point a.o b.o       # 指定入口點
ld -s                         # 剝離符號表
ld -r                         # 部分連結（不重定位）
ld -Map=mapfile               # 輸出映射檔案

# 控制符號解析
ld --whole-archive a.o --no-whole-archive b.o
# --whole-archive 強制包含所有成員
```

## 共享物件與位置無關碼

### PIC 原理

```asm
# 位置無關碼的呼叫方式
# call get_func@PLT  # 透過 PLT 呼叫
# jmp *func@gotpcrel(%rip)  # 從 GOT 取位址
```

## 常見問題與除錯

### 未定義符號

```bash
# 錯誤訊息
undefined reference to `foo'

# 解決方法：
# 1. 確認函式庫已連結
# 2. 確認函式庫順序正確
# 3. 確認使用了正確的函式庫（32/64 位）
```

### 多重定義

```bash
# 錯誤訊息
multiple definition of `global_var'

# 解決方法：
# 1. 在其中一個檔案宣告為 static
# 2. 使用 extern
# 3. 使用匿名命名空間
```

### 符號版本

```bash
# 查看動態符號
nm -D program

# 版本腳本
{
    global:
        foo;
    local:
        *;
} Version1;
```
