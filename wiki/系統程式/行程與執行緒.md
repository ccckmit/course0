# 行程與執行緒

行程與執行緒是作業系統中最核心的概念，它們是系統資源分配和調度的基本單位。理解這兩個概念對於掌握作業系統的運作原理、開發高效能應用程式、解決並發問題至關重要。本篇將深入探討行程與執行緒的定義、特點、創建方式、同步機制以及它們之間的關係。

## 行程（Process）

行程是作業系統進行資源分配和調度的基本單位，代表了一個正在執行的程式。每個行程都有自己獨立的位址空間和系統資源，行程之間相互隔離，這種隔離確保了系統的穩定性和安全性。

### 行程的生命週期

```
┌────────┐      ┌────────┐      ┌────────┐      ┌────────┐      ┌────────┐
│  新建   │─────→│  就緒   │─────→│  執行   │─────→│  等待   │─────→│  終止   │
│ (New)   │      │ (Ready) │      │ (Running)│     │ (Blocked)│     │(Terminated)│
└────────┘      └────────┘      └────────┘      └────────┘      └────────┘
     │              │              │              │
     │              │              │              │
     │              │              │              │ ┌────────┐
     └──────────────┴──────────────┴──────────────┘→│  就緒   │
                              CPU 排程器              └────────┘
```

- **新建（New）**：行程正在被創建，尚未進入就緒隊列
- **就緒（Ready）**：等待獲得 CPU 執行權
- **執行（Running）**：正在 CPU 上執行指令
- **等待（Waiting）**：等待某個事件發生（如 I/O 完成、信號）
- **終止（Terminated）**：執行完畢，資源即將被回收

### 行程的創建

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();
    
    if (pid < 0) {
        perror("fork failed");
        exit(1);
    }
    
    if (pid == 0) {
        // 子行程
        printf("Child process: PID = %d, PPID = %d\n", 
               getpid(), getppid());
        
        // 執行新程式
        // execl("/bin/ls", "ls", "-l", NULL);
        
        exit(0);
    } else {
        // 父行程
        printf("Parent process: PID = %d, Child PID = %d\n", 
               getpid(), pid);
        
        int status;
        wait(&status);  // 等待子行程結束
        
        if (WIFEXITED(status)) {
            printf("Child exited with status: %d\n", 
                   WEXITSTATUS(status));
        }
    }
    
    return 0;
}
```

## 執行緒（Thread）

執行緒是行程內的輕量級執行單位，是 CPU 調度的最小單位。一個行程可以包含多個執行緒，這些執行緒共享行程的位址空間和資源，但各自有自己的執行上下文。

### 執行緒的優勢

```
┌─────────────────────────────────────────────────────────────┐
│                  執行緒的優勢                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 資源共享                                               │
│      - 共享行程的虛擬位址空間                               │
│      - 共享開啟的檔案                                       │
│      - 共享全域變數                                         │
│      - 共享動態配置的記憶體                                  │
│                                                             │
│   2. 創建成本低                                             │
│      - 行程建立需要複製整個位址空間                          │
│      - 執行緒只建立執行上下文                               │
│      - 建立時間：執行緒比行程快 10-100 倍                   │
│                                                             │
│   3. 通訊容易                                               │
│      - 直接透過共享記憶體通訊                               │
│      - 無需複雜的 IPC 機制                                  │
│      - 開發和維護成本低                                     │
│                                                             │
│   4. 充分利用多核                                           │
│      - 多執行緒可在多核 CPU 上並行執行                      │
│      - 充分利用計算資源                                     │
│      - 提高系統吞吐量                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 執行緒 vs 行程

| 特性 | 行程 | 執行緒 |
|------|------|--------|
| 資源 | 獨立的位址空間、檔案描述符、環境變數 | 共享行程資源 |
| 建立成本 | 高（需複製資源） | 低（僅建立執行上下文） |
| 通訊方式 | 需要 IPC | 直接共享記憶體 |
| 隔離性 | 完全隔離 | 共享記憶體 |
| 系統開銷 | 大 | 小 |
| 建立速度 | 慢 | 快 |
| 切換速度 | 慢 | 快 |
| 可靠性 | 高（一個崩潰不影響其他） | 低（一個崩潰導致行程崩潰） |

### POSIX 執行緒（pthread）

```c
#include <pthread.h>
#include <stdio.h>
#include <stdlib.h>

void* thread_function(void* arg) {
    int thread_num = *(int*)arg;
    printf("Thread %d is running\n", thread_num);
    
    // 執行緒本地變數
    int local_var = thread_num * 10;
    printf("Thread %d: local_var = %d\n", thread_num, local_var);
    
    return NULL;
}

int main() {
    pthread_t threads[3];
    int thread_args[3] = {1, 2, 3};
    
    // 創建執行緒
    for (int i = 0; i < 3; i++) {
        if (pthread_create(&threads[i], NULL, 
                          thread_function, &thread_args[i]) != 0) {
            perror("pthread_create failed");
            exit(1);
        }
    }
    
    // 等待執行緒結束
    for (int i = 0; i < 3; i++) {
        pthread_join(threads[i], NULL);
    }
    
    printf("All threads completed\n");
    
    return 0;
}
```

### 執行緒屬性

```c
#include <pthread.h>

pthread_attr_t attr;

// 初始化
pthread_attr_init(&attr);

// 設置分離狀態
pthread_attr_setdetachstate(&attr, PTHREAD_CREATE_JOINABLE);
// 或 PTHREAD_CREATE_DETACHED

// 設置排程策略
pthread_attr_setschedpolicy(&attr, SCHED_FIFO);

// 設置優先級

param.sched_priority = 10;
pthread_attr_setschedparam(&attr, &param);

// 設置堆疊大小
pthread_attr_setstacksize(&attr, 1024 * 1024);  // 1MB

// 使用屬性創建執行緒
pthread_t thread;
pthread_create(&thread, &attr, function, arg);

// 銷毀
pthread_attr_destroy(&attr);
```

### C++11 std::thread

```cpp
#include <thread>
#include <iostream>
#include <vector>
#include <mutex>

std::mutex mtx;

void print_thread_id(int id) {
    std::lock_guard<std::mutex> lock(mtx);
    std::cout << "Thread " << id << " is running\n";
}

int main() {
    std::vector<std::thread> threads;
    
    for (int i = 0; i < 5; i++) {
        threads.emplace_back(print_thread_id, i);
    }
    
    for (auto& t : threads) {
        t.join();
    }
    
    return 0;
}

// Lambda 表達式

    std::cout << "Lambda thread\n";
});

lambda_thread.join();
```

### Java 執行緒

```java
public class ThreadExample {
    // 繼承 Thread
    static class MyThread extends Thread {
        @Override
        public void run() {
            System.out.println("MyThread running");
        }
    }
    
    // 實現 Runnable
    static class MyRunnable implements Runnable {
        @Override
        public void run() {
            System.out.println("MyRunnable running");
        }
    }
    
    public static void main(String[] args) throws InterruptedException {
        // 方式1：繼承 Thread
        Thread t1 = new MyThread();
        t1.start();
        
        // 方式2：實現 Runnable
        Runnable r = new MyRunnable();
        Thread t2 = new Thread(r);
        t2.start();
        
        // 方式3：Lambda
        Thread t3 = new Thread(() -> {
            System.out.println("Lambda thread");
        });
        t3.start();
        
        // 等待執行緒
        t1.join();
        t2.join();
        t3.join();
        
        // 執行緒池
        ExecutorService executor = Executors.newFixedThreadPool(4);
        executor.submit(() -> System.out.println("Task"));
        executor.shutdown();
    }
}
```

## 同步機制

當多個執行緒同時訪問共享資源時，需要同步機制來防止競爭條件和確保資料一致性。

### 互斥鎖（Mutex）

```c
#include <pthread.h>

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

void* thread_function(void* arg) {
    pthread_mutex_lock(&mutex);
    // 臨界區 - 保護共享資源
    shared_variable++;
    pthread_mutex_unlock(&mutex);
    return NULL;
}
```

```cpp
// C++ 使用 RAII
#include <mutex>

std::mutex mtx;

void function() {
    std::lock_guard<std::mutex> lock(mtx);
    // 臨界區
}  // 自動解鎖
```

### 讀寫鎖

```c
pthread_rwlock_t rwlock;

pthread_rwlock_rdlock(&rwlock);  // 讀鎖
pthread_rwlock_wrlock(&rwlock); // 寫鎖
pthread_rwlock_unlock(&rwlock);
```

### 條件變數

```c
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t cond = PTHREAD_COND_INITIALIZER;
int ready = 0;

void* producer(void* arg) {
    sleep(1);
    pthread_mutex_lock(&mutex);
    ready = 1;
    pthread_cond_signal(&cond);  // 通知消費者
    pthread_mutex_unlock(&mutex);
    return NULL;
}

void* consumer(void* arg) {
    pthread_mutex_lock(&mutex);
    while (!ready) {
        pthread_cond_wait(&cond, &mutex);  // 等待條件
    }
    pthread_mutex_unlock(&mutex);
    return NULL;
}
```

### 生產者-消費者問題

```c
#include <pthread.h>
#include <semaphore.h>
#include <queue.h>

#define BUFFER_SIZE 10

pthread_mutex_t mutex;
sem_t empty;
sem_t full;

std::queue<int> buffer;

void* producer(void* arg) {
    for (int i = 0; i < 20; i++) {
        sem_wait(&empty);
        pthread_mutex_lock(&mutex);
        
        buffer.push(i);
        printf("Produced: %d\n", i);
        
        pthread_mutex_unlock(&mutex);
        sem_post(&full);
    }
    return NULL;
}

void* consumer(void* arg) {
    for (int i = 0; i < 20; i++) {
        sem_wait(&full);
        pthread_mutex_lock(&mutex);
        
        int val = buffer.front();
        buffer.pop();
        printf("Consumed: %d\n", val);
        
        pthread_mutex_unlock(&mutex);
        sem_post(&empty);
    }
    return NULL;
}

int main() {
    pthread_mutex_init(&mutex, NULL);
    sem_init(&empty, 0, BUFFER_SIZE);
    sem_init(&full, 0, 0);
    
    pthread_t prod, cons;
    pthread_create(&prod, NULL, producer, NULL);
    pthread_create(&cons, NULL, consumer, NULL);
    
    pthread_join(prod, NULL);
    pthread_join(cons, NULL);
    
    return 0;
}
```

### 死結（Deadlock）

```c
// 死結的 四個必要條件
// 1. 互斥條件
// 2. 請求與保持條件
// 3. 不剝奪條件
// 4. 循環等待條件

// 避免死結的方法
// 1. 固定順序獲取鎖
pthread_mutex_lock(&mutex1);
pthread_mutex_lock(&mutex2);
// ... 臨界區
pthread_mutex_unlock(&mutex2);
pthread_mutex_unlock(&mutex1);

// 2. 使用 trylock
if (pthread_mutex_trylock(&mutex1) == 0) {
    if (pthread_mutex_trylock(&mutex2) == 0) {
        // 臨界區
        pthread_mutex_unlock(&mutex2);
        pthread_mutex_unlock(&mutex1);
    } else {
        pthread_mutex_unlock(&mutex1);
    }
}
```

## 執行緒池

```python
# Python concurrent.futures
from concurrent.futures import ThreadPoolExecutor

def task(n):
    return n * n

with ThreadPoolExecutor(max_workers=4) as executor:
    # 提交任務
    future = executor.submit(task, 10)
    result = future.result()
    
    # map
    results = executor.map(task, range(10))
```
