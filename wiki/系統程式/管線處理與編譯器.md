# 管線處理與編譯器

管線（Pipeline）是現代電腦系統中最重要效能優化技術之一。從 CPU 指令管線到編譯器優化管線，管線化思想滲透在系統的各個層面。本章探討管線處理的基本原理及其在編譯器中的應用。

## 指令管線基礎

### 指令執行階段

典型的 RISC 指令執行分為五個階段：

```
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  IF     │   ID    │   EX    │   MEM   │   WB    │
│ 取指    │ 解碼    │ 執行    │ 記憶體   │ 寫回   │
└─────────┴─────────┴─────────┴─────────┴─────────┘
    │         │         │         │         │
    ▼         ▼         ▼         ▼         ▼
時脈 1     時脈 2     時脈 3     時脈 4     時脈 5
```

### 管線加速

非管線處理：5 個時脈週期執行 1 條指令
管線處理：每時脈週期完成 1 條指令

```
非管線：
┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
│ IF │ ID │ EX │MEM│WB │ IF │ ID │ EX │MEM│WB │
└────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘

管線：
┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
│I1:IF│I1:ID│I1:EX│I1:M│I1:WB│    │    │    │    │    │
└────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘
│    │I2:IF│I2:ID│I2:EX│I2:M│I2:WB│    │    │    │
└────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘
│    │    │I3:IF│I3:ID│I3:EX│I3:M│I3:WB│    │    │
```

理論加速比 = 管線深度 × 時脈頻率提升

## 管線 hazard

### 結構 hazard

硬體資源衝突：

```
問題：記憶體和 ALU 同時需要存取記憶體

┌─────────────────────────────┐
│ Stage 4 (MEM/WB)           │
│ 如果只有一個記憶體埠：        │
│ Load 無法同時讀寫？          │
└─────────────────────────────┘

解決：複製硬體資源（如雙口 SRAM）
```

### 資料 hazard

指令間的資料依賴：

```
add $t0, $t1, $t2    # $t0 在 WB 階段產生
sub $t3, $t0, $t4    # $t3 需要 $t0，但 $t0 尚未寫回
and $t5, $t0, $t6    # 與上相同問題
```

### 控制 hazard

分支指令導致的不確定性：

```
beq $t0, $t1, label   # 分支結果未知
   │
   ├── 預測跳轉 → 錯誤預測代價高
   │
   └── 等待結果 → 流水線停頓
```

## 解決 Hazard 的技術

### 轉發（Forwarding）

將結果直接轉發到需要它的階段：

```
┌─────────────────────────────────────────┐
│ EX/MEM.RegWrite ──────────────────────→ │
│     │     MEM/WB.RegWrite ────────────→ │
│     │           │                       │
│     ▼           ▼                       │
│  ┌─────────────────────────────┐       │
│  │  ALU 的輸入可選項：           │       │
│  │  1. 暫存器檔案               │       │
│  │  2. EX/MEM 轉發              │       │
│  │  3. MEM/WB 轉發              │       │
│  └─────────────────────────────┘       │
└─────────────────────────────────────────┘
```

### 載入延遲槽

載入指令後的延遲：

```mips
# 載入後需要一個指令的延遲才能使用
lw $t0, 0($t1)       # 延遲槽：$t0 尚不可用
nop                  # 必需的延遲槽
add $t2, $t0, $t3    # 現在可以使用 $t0
```

## 動態分支預測

### 2 位元饱和计数器

```
┌────────────────────────────────────────┐
│         分支預測器狀態機               │
│                                        │
│    strongly not taken                  │
│           │                            │
│           ▼                            │
│  ┌────────┐  ┌────────┐              │
│  │ 00     │──→│ 01     │              │
│  │ (NT)   │←──│ (WNT)  │              │
│  └───┬────┘  └────┬───┘              │
│      │ taken      │ not taken        │
│      ▼            ▼                  │
│  ┌────────┐  ┌────────┐              │
│  │ 11     │←──│ 10     │              │
│  │ (WT)   │──→│ (T)    │              │
│  └────────┘  └────────┘              │
│       ▲            ▲                   │
│       │   taken   │                   │
│       └───────────┘                   │
│                                        │
│  strongly taken ─→ weakly taken ─→    │
│                                        │
│  預測失敗需要兩次連續失敗才能改變預測    │
└────────────────────────────────────────┘
```

### 分支目標緩衝（BTB）

```c
// 簡化的 BTB 查詢
struct BTBEntry {
    uint32_t pc;
    uint32_t target;
    uint8_t state;  // 2 位元預測器狀態
    uint8_t valid;
};

BTBEntry* lookup(uint32_t pc) {
    uint32_t index = hash(pc);
    if (btb[index].valid && btb[index].pc == pc) {
        return &btb[index];
    }
    return NULL;
}
```

## 超純量與 VLIW

### 超純量（Superscalar）

硬體動態決定同時發射多少條指令：

```
       ┌──────────────────────────────────────┐
       │           Pipeline Issue Logic        │
       │   動態調度：根據執行單元可用性        │
       └──────────────────┬───────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
    ┌───────┐          ┌───────┐          ┌───────┐
    │  ALU  │          │  ALU  │          │  LSU  │
    │(Int)  │          │(FP)   │          │(Load) │
    └───────┘          └───────┘          └───────┘
```

### VLIW（Very Long Instruction Word）

編譯器靜態決定指令調度：

```c
// 編譯器靜態調度
// 3-issue VLIW: ALU | ALU | MEM
// .L:
{
    add $t0, $t1, $t2;    // Issue slot 1
    sub $t3, $t4, $t5;    // Issue slot 2
    nop;                  // Issue slot 3
    ld $t6, 0($sp);       // Issue slot 1 (下一個 packet)
    add $t7, $t6, $t8;   // Issue slot 2
    nop;                  // Issue slot 3
}
```

## 編譯器優化 Pass

LLVM 等現代編譯器使用 pass 架構：

```cpp
// 自定義優化 Pass
class MyOptimizationPass : public FunctionPass {
public:
    static char ID;
    
    MyOptimizationPass() : FunctionPass(ID) {}
    
    bool runOnFunction(Function &F) override {
        bool modified = false;
        
        for (auto &BB : F) {
            for (auto &inst : BB) {
                // 識別模式
                if (isRedundantLoad(&inst)) {
                    // 消除冗餘載入
                    replaceWithCachedValue(&inst);
                    modified = true;
                }
            }
        }
        
        return modified;
    }
};
```

### 常見優化 Pass

| Pass 名稱 | 功能 |
|-----------|------|
| Constant folding | 常數折疊 |
| Dead code elimination | 死碼消除 |
| Loop unrolling | 迴圈展開 |
| Loop-invariant code motion | 迴圈不變代碼移動 |
| Global value numbering | 全域值編號 |
| Sparse conditional constant propagation | 稀疏條件常數傳播 |

## 迴圈展開（Loop Unrolling）

減少迴圈控制開銷：

```c
// 原始程式
for (int i = 0; i < n; i++) {
    a[i] = b[i] + c[i];
}

// 展開 4 次
for (int i = 0; i < n - 3; i += 4) {
    a[i]   = b[i]   + c[i];
    a[i+1] = b[i+1] + c[i+1];
    a[i+2] = b[i+2] + c[i+2];
    a[i+3] = b[i+3] + c[i+3];
}

// 處理尾部
for (int i = ((n/4)*4); i < n; i++) {
    a[i] = b[i] + c[i];
}
```

### 完全迴圈展開

小型、固定次數迴圈：

```c
// 編譯器可完全展開
#define N 4
double dot_product(double a[N], double b[N]) {
    double result = 0;
    #pragma unroll
    for (int i = 0; i < N; i++) {
        result += a[i] * b[i];
    }
    return result;
}
```

## 軟體管線化

隱藏記憶體延遲的迴圈優化：

```c
// 軟體管線化
// 迭代 i 和 i+1 的指令可並行執行

// 原始迴圈
for (i = 0; i < n; i++) {
    ld a[i];          // 載入
    mul a[i] * b[i];  // 乘法
    add result;        // 加法
    st result;         // 存儲
}

// 軟體管線化後
// Initiation Interval (II) = 1
// 每個週期啟動新迭代

// 迴圈 prolog
ld a[0]; ld b[0];
mul a[0] * b[0];

// 迴圈核心（每週期一條指令啟動）
ld a[1]; ld b[1];       // 迭代 2 啟動
mul a[0] * b[0];        // 迭代 1 執行
add result;             // 迭代 0 執行

// 迴圈 epilog
st result;
```

## LLVM 優化驅動

### O 等級

```bash
# O0：優化用於調試，幾乎無優化
clang -O0 -g program.c -o program

# O1：基本優化
clang -O1 program.c -o program

# O2：積極優化
clang -O2 program.c -o program

# O3：更多優化（可能增加代碼大小）
clang -O3 program.c -o program

# Os：優化代碼大小
clang -Os program.c -o program

# Oz：最小代碼大小
clang -Oz program.c -o program
```

### 個別優化 Pass

```bash
# 查看某個函數的 LLVM IR
clang -S -emit-llvm -O2 -Xclang -disable-O0-optnone program.c -o program.ll
opt -S -mem2reg -loop-unroll -loop-vectorize program.ll -o optimized.ll

# 查看優化後的程式碼
llc -O2 optimized.ll -o program.s
```

## 向量化

SIMD 指令的自動生成：

```c
// 原始：標量版本
void add_arrays(float *a, float *b, float *c, int n) {
    for (int i = 0; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}

// 編譯器自動向量化
// GCC: -O3 -ftree-vectorize
// LLVM: -O3 -vectorize-loops=false (或 true)

// 等效向量化代碼
void add_arrays_vectorized(float *a, float *b, float *c, int n) {
    int i = 0;
    
    // 向量化：一次處理 4 個 float
    for (; i + 4 <= n; i += 4) {
        __m128 va = _mm_loadu_ps(&a[i]);
        __m128 vb = _mm_loadu_ps(&b[i]);
        __m128 vc = _mm_add_ps(va, vb);
        _mm_storeu_ps(&c[i], vc);
    }
    
    // 尾部處理
    for (; i < n; i++) {
        c[i] = a[i] + b[i];
    }
}
```

### SIMD 編譯指示

```c
void matrix_multiply(float *C, float *A, float *B, int n) {
    #pragma omp simd
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            float sum = 0.0f;
            #pragma omp simd reduction(+:sum)
            for (int k = 0; k < n; k++) {
                sum += A[i*n + k] * B[k*n + j];
            }
            C[i*n + j] = sum;
        }
    }
}
```

## 超純量編譯器支援

### 指令排程

```c
// 原始程式（相依圖）
a = x + y;     // 依賴：x, y
b = a * 2;     // 依賴：a
c = b + 1;     // 依賴：b

// 2-issue 超純量排程
// Issue slot 1: ALU
// Issue slot 2: LOAD/STORE

Cycle 1:  a = x + y;     b = c * 2;      // 兩條並行
Cycle 2:  c = b + 1;     d = e + f;      // 兩條並行
```

### 寄存器分配

```c
// 原始：大量虛擬暫存器
t1 = x + y;     // t1
t2 = t1 * z;    // t1 -> t2
t3 = t2 / 2;    // t2 -> t3
// ...

// 寄存器分配後
// 使用有限暫存器 Rx
R1 = x + y;
R2 = R1 * z;
R3 = R2 / 2;
// ...
```
