# 計算理論

計算理論是電腦科學的理論基礎，探討計算的本質、能力與極限。這個領域回答了「什麼可以被計算？」、「計算需要多少資源？」以及「哪些問題是無法解決的？」等基本問題。計算理論的發展始於 1930 年代，當時數學家們試圖理解 Hilbert 提出的判定問題，最終形成了圖靈機、λ演算等計算模型，為現代電腦科學奠定了堅實的理論基礎。

## 計算模型層次

計算理論建立了不同層次的計算模型，每個模型都有其獨特的能力和限制：

```

│  • 最強大的計算模型                                           │

│  • Church-Turing 論點的基礎                                    │

```

## 圖靈機

圖靈機由 Alan Turing 於 1936 年提出，是計算理論的核心概念。它是一個理論模型，由無限期性的磁帶、讀寫頭和有限狀態控制器組成：

### 形式定義

```
圖靈機 M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

其中：
• Q: 有限狀態集合
• Σ: 輸入字母表（不含空白）
• Γ: 磁帶字母表（包含空白 □）
• δ: 轉移函數 Q × Γ → Q × Γ × {L, R}
• q₀: 起始狀態
• q_accept: 接受狀態
• q_reject: 拒絕狀態
```

### 運作方式

```python
# 圖靈機運作
# 1. 讀寫頭讀取目前磁帶格子上的符號
# 2. 根據目前狀態和讀到的符號，轉移函數決定：
#    - 下一個狀態
#    - 寫入的新符號
#    - 讀寫頭移動方向（左或右）
# 3. 重複直到進入接受或拒絕狀態

# 範例：識別簡單語言 {0ⁿ1ⁿ | n ≥ 1}
# 步驟：
# 1. 從最左邊開始，將 0 改為 X，右移
# 2. 找對應的 1，改為 Y，左移
# 3. 重複直到處理完所有 0
# 4. 若成功配對，則接受
```

### Church-Turing 論點

```
Church-Turing 論點聲稱：
任何可有效計算的函數都可以由圖靈機計算。

這不是一個定理，而是一個猜想：
• Church 從 λ演算角度提出
• Turing 從圖靈機角度提出
• 兩者是等價的

意義：
• 圖靈機定義了「可計算」的數學意義
• 之後沒有任何計算模型能超越圖靈機
```

## λ演算

λ演算由 Alonzo Church 在 1936 年提出，是函數式程式語言的理論基礎：

### 語法

```
λ calculus 語法：
<表達式> ::= <變數>
           | (λ<變數>.<表達式>)
           | (<表達式><表達式>)

範例：
• x                - 變數
• λx.M             - 函數（x 為參數，M 為函式體）
• (λx.M N)         - 函數應用（將 N 應用於 λx.M）
```

### 核心概念

```python
# 自由變數和束縛變數
# λx.M 中，x 是束縛變數，其他是自由變數

# α轉換（Alpha Conversion）
# λx.M = λy.M[x/y]  # 更名不改变语义

# β歸約（Beta Reduction）
# (λx.M) N → M[x/N]  # 用 N 替換 M 中的 x

# η轉換（Eta Conversion）
# λx.(M x) = M  # 若 x 不在 M 中自由出現
```

### Y Combinator

```python
# Y Combinator - 實現遞迴
# Y = λf.(λx.f (x x)) (λx.f (x x))

# 使用 Y 定義遞迴函數
# Y F = F (Y F)  # 讓 F 套用自身

# 範例：階層函數
# fact = Y (λf n. if n==0 then 1 else n * f(n-1))
```

## 形式文法

形式文法描述語言的語法結構，是編譯器理論的基礎：

### Chomsky 階層

```
┌─────────────────────────────────────────────────────────────────┐
│                      Chomsky 階層                              │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  Type 0: 無限制文法 (Phrase Structure)                       │
│  • 產生式：α → β，α, β 為任意字串                            │
│  • 對應：圖靈機                                                │
│  • 語言：遞迴可枚舉語言                                        │
│                                                              │
│  Type 1: 上下文敏感文法 (Context-Sensitive)                   │
│  • 產生式：αAβ → αγβ                                        │
│  • 對應：線性有界自動機                                        │
│  • 語言：上下文敏感語言                                        │
│                                                              │
│  Type 2: 上下文無關文法 (Context-Free)                        │
│  • 產生式：A → γ                                             │
│  • 對應：下推自動機                                            │
│  • 語言：大多數程式語言語法                                    │
│                                                              │
│  Type 3: 正規文法 (Regular)                                   │
│  • 產生式：A → aB 或 A → a                                   │
│  • 對應：有限狀態機                                            │
│  • 語言：正則表達式描述的語言                                   │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 語法分析

```python
# 上下文無關文法的例子
# E → T | E + T
# T → F | T * F
# F → ( E ) | id

# 導出範例
# E → T → F * F → id * F → id * id

# 分析樹
#        E
#       / \
#      T   + T
#     /       \
#    F         F
#    |         |
#   id        id
```

## 有限狀態機

有限狀態機是最簡單的計算模型，用於識別正則語言：

### 確定性 vs 非確定性

```python
# DFA (Deterministic Finite Automaton)
# 每個狀態對每個輸入只有一個轉移
# M = (Q, Σ, δ, q₀, F)

# 範例：識別以 01 結尾的二進位字串
# Q = {q0, q1, q2}
# Σ = {0, 1}
# q0 = 起始狀態
# F = {q2}
# δ:
#   δ(q0, 0) = q0, δ(q0, 1) = q1
#   δ(q1, 0) = q2, δ(q1, 1) = q1
#   δ(q2, 0) = q2, δ(q2, 1) = q1

# NFA (Nondeterministic Finite Automaton)
# 一個狀態對同一輸入可以有多個轉移
# 空轉移 (ε) 允許
# 等價於 DFA（可以轉換）
```

### 正則表達式

```python
# 正則表達式與 FSM 的對應
# • a  → a 從狀態 s 到 f
# • ε  → 空轉移
# • r|s → 並聯
# • rs  → 串聯
# • r*  → 迴圈
```

## 可計算性理論

### 停機問題

停機問題是計算理論中最著名的不可判定問題：

```
停機問題 (Halting Problem)：
不存在一個程式能判斷任意程式是否會停止。

證明思路（反證法）：
假設存在函式 Halt(P, I) 判斷程式 P 對輸入 I 是否會停止
構造函式：
  Paradox(P):
    if Halt(P, P) then
      無限迴圈
    else
      停止

呼叫 Paradox(Paradox) 導致矛盾。

意義：
• 存在無法用演算法解決的問題
• 電腦有本質限制
• 數學真理不等於可計算真理
```

### 不可判定問題

```
其他經典不可判定問題：
• 停機問題的變體
• Post 對應問題
• 一階邏輯有效性
• 上下文無關文法交集
• 等價性（DFA 語言）
• 萬有歸納

Rice 定理：
任何非平凡的語言性質都是不可判定的。
```

## 複雜度理論

### 計算複雜度

```python
# 時間複雜度類別
# O(1)       常數
# O(log n)   對數
# O(n)       線性
# O(n log n) 線性對數
# O(n²)      二次
# O(2ⁿ)      指數

# 空間複雜度類別
# O(1)    常數空間
# O(n)    線性空間
# PSPACE  多項式空間
# EXPTIME 指數時間
```

### P vs NP

```
P 類問題：
• 可以在多項式時間內解決
• 例：排序、搜尋、最短路徑

NP 類問題：
• 可以在多項式時間內驗證解答
• 例：質數分解（！？）、旅行推銷員

NP-Complete：
• 最難的 NP 問題
• 任何 NP 問題都可以歸約到它
• 例：SAT、3-SAT、頂點覆蓋

核心問題：
P = NP 是否成立？
（目前傾向於否，但不確定）
```
