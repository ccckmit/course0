# TCP-IP協定

TCP/IP 是網際網路的基礎協定套件，定義了資料如何在網路中傳輸。TCP/IP 不是單一協定，而是由多個協定組成的協定堆疊，它使得全球不同類型的電腦和網路能夠相互通信。從我們日常瀏覽網頁、收發電子郵件，到企業的數據傳輸，都依賴 TCP/IP 協定。

## TCP/IP 協定架構

TCP/IP 採用四層架構，與 OSI 七層模型對應：

```
┌─────────────────────────────────────────────────────────────┐
│                     TCP/IP 四層模型                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   OSI 模型              TCP/IP 模型         常見協定       │
│   ┌──────────┐         ┌──────────┐                       │
│   │  應用層   │  ──────  │  應用層   │  HTTP, DNS, FTP,    │
│   │          │         │          │  SMTP, SSH, DHCP     │
│   ├──────────┤         ├──────────┤                       │
│   │  表示層   │  ──────  │          │                       │
│   │          │         │          │                       │
│   ├──────────┤         ├──────────┤                       │
│   │  會議層   │  ──────  │          │                       │
│   │          │         │          │                       │
│   ├──────────┤         ├──────────┤                       │
│   │  傳輸層   │  ──────  │  傳輸層   │  TCP, UDP          │
│   │          │         │          │                       │
│   ├──────────┤         ├──────────┤                       │
│   │  網路層   │  ──────  │  網路層   │  IP, ICMP, ARP     │
│   │          │         │          │                       │
│   ├──────────┤         ├──────────┤                       │
│   │  資料連結 │  ──────  │  連結層   │  Ethernet, WiFi    │
│   │          │         │          │  PPP, VLAN          │
│   ├──────────┤         ├──────────┤                       │
│   │  實體層   │  ──────  │          │                       │
│   └──────────┘         └──────────┘                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 各層職責

**連結層（Link Layer）**
- 負責在本地網路中傳送資料
- 定義硬體位址（MAC 位址）
- 處理錯誤檢測和流量控制
- 常見協定：Ethernet、Wi-Fi、PPP

**網路層（Network Layer）**
- 負責在不同網路間傳送資料
- 使用 IP 位址進行路由
- 封裝和解封裝資料包
- 常見協定：IP、ICMP、ARP、RIP、OSPF、BGP

**傳輸層（Transport Layer）**
- 負責端到端的資料傳輸
- 區分不同的應用程式（使用連接埠）
- 確保資料可靠或高效傳輸
- 常見協定：TCP、UDP

**應用層（Application Layer）**
- 為使用者提供網路服務
- 定義應用程式間的通訊格式
- 常見協定：HTTP、HTTPS、DNS、FTP、SMTP、POP3、IMAP、SSH、Telnet

## IP 協定（Internet Protocol）

IP 是 TCP/IP 的核心，負責將資料從來源傳送到目的地。每個連接到網際網路的設備都有一個唯一的 IP 位址。

### IPv4 位址

IPv4 位址是 32 位元（4 位元組）的位址，通常用點分十進製表示：

```
192.168.1.1

┌─────────┬─────────┬─────────┬─────────┐
│  192    │  168    │   1     │   1     │
│ 8 bits  │  8 bits │  8 bits │  8 bits │
└─────────┴─────────┴─────────┴─────────┘
```

**位址分類**：

| 類別 | 範圍 | 用途 | 私有範圍 |
|------|------|------|----------|
| A | 1-126 | 大型組織 | 10.0.0.0/8 |
| B | 128-191 | 中型組織 | 172.16.0.0/12 |
| C | 192-223 | 小型組織 | 192.168.0.0/16 |
| D | 224-239 | 多播 | - |
| E | 240-255 | 保留 | - |

**特殊位址**：
- 127.0.0.1：本地回環（localhost）
- 0.0.0.0：代表「任何網路」
- 255.255.255.255：廣播位址

### 子網路遮罩

子網路遮罩用於確定 IP 位址中網路部分和主機部分：

```
IP 位址:      192.168.1.100
子網路遮罩:   255.255.255.0 (/24)
             ────────────
             網路部分    主機部分

網路位址:     192.168.1.0
廣播位址:     192.168.1.255
可用主機數:   254 (2^8 - 2)
```

**常見子網路遮罩**：
- /8 (255.0.0.0) - 16M 主機
- /16 (255.255.0.0) - 65K 主機
- /24 (255.255.255.0) - 254 主機
- /30 (255.255.255.252) - 2 主機（點對點）

### IPv6 位址

IPv6 位址是 128 位元的位址，用冒號十六進製表示：

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
2001:db8:85a3::8a2e:370:7334  (簡化表示)
```

**IPv6 優點**：
- 位址空間巨大（2^128 個位址）
- 內建安全性（IPsec）
- 簡化標頭（更高效處理）
- 自動配置
- QoS 支援

## TCP 協定

TCP（Transmission Control Protocol）是面向連接的可靠傳輸協定，提供順序傳遞、錯誤控制、流量控制和壅塞控制。

### 三向交握（Three-Way Handshake）

TCP 在傳送資料前需要建立連接：

```
┌─────────────────────────────────────────────────────────────┐
│                    TCP 三向交握                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    客戶端                         伺服器                    │
│       │                              │                      │
│       │─────── SYN (seq=x) ──────────→│  客戶端發起連接      │
│       │                              │                      │
│       │←──── SYN-ACK (seq=y, ack=x+1) ─│  伺服器回應         │
│       │                              │                      │
│       │─────── ACK (ack=y+1) ─────────→│  客戶端確認          │
│       │                              │                      │
│       │══════════ 已建立連接 ══════════│                      │
│       │                              │                      │
└─────────────────────────────────────────────────────────────┘

步驟說明：
1. 客戶端發送 SYN 包， seq=x，表示請求建立連接
2. 伺服器回應 SYN-ACK 包，seq=y, ack=x+1，表示接受並確認
3. 客戶端發送 ACK 包，ack=y+1，確認完成
4. 連接建立，可以傳送資料
```

**為什麼需要三向交握？**
- 確保雙方都能收發資料
- 同步雙方的序列號
- 防止舊連接請求被錯誤接受

### 四向揮手（Four-Way Handshake）

TCP 連接的關閉過程：

```
┌─────────────────────────────────────────────────────────────┐
│                    TCP 四向揮手                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    客戶端                         伺服器                    │
│       │                              │                      │
│       │─────── FIN ─────────────────→│  客戶端關閉寫入        │
│       │                              │                      │
│       │←─────── ACK ────────────────│  伺服器確認            │
│       │                              │                      │
│       │←─────── FIN ────────────────│  伺服器也關閉          │
│       │                              │                      │
│       │─────── ACK ─────────────────→│  客戶端確認            │
│       │                              │                      │
│       │══════════ 連接關閉 ══════════│                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### TCP 可靠傳輸機制

**序列號和確認**：
- 每個位元組都有序列號
- 接收方確認已收到的資料
- 確認丟失會導致重傳

**超時重傳**：
- 發送後啟動計時器
- 超時未確認則重傳
- 動態調整超時值（根據 RTT）

**錯誤檢測**：
- 使用 CRC 校驗
- 損壞的資料會被丟棄
- 重傳丟失的資料

### 流量控制（Flow Control）

滑動視窗機制防止發送方壓垮接收方：

```
發送方視窗：
┌───┬───┬───┬───┬───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │ 9 │
└───┴───┴───┴───┴───┴───┴───┴───┴───┘
  已確認  │    已發送    │   可發送
          │              │
        視窗大小 = 5
```

**接收方的窗口**：
- 通過 TCP 標頭中的 Window Size 告知發送方
- 接收方可以控制發送方的發送速率

### 壅塞控制（Congestion Control）

防止網路過載的機制：

**1. 慢啟動（Slow Start）**：
- 初始擁塞窗口（cwnd）很小
- 指數增長直到達到閾值（ssthresh）
- 每收到一個 ACK，cwnd 增加一個 MSS

**2. 壅塞避免（Congestion Avoidance）**：
- 達到閾值後，改爲線性增長
- 每個 RTT 增加一個 MSS

**3. 快速重傳（Fast Retransmit）**：
- 收到 3 個重複 ACK
- 立即重傳丟失的片段

**4. 快速恢復（Fast Recovery）**：
- 快速重傳後進入
- 視窗減半，繼續傳輸

```
壅塞窗口變化：
cwnd
  ↑
  │       ╱╲        ╱╲
  │      ╱  ╲      ╱  ╲
  │     ╱    ╲    ╱    ╲──────
  │────╱      ╲──╱
  │              ╲
  └──────────────────────────────→ 時間
      慢啟動    壅塞避免   事件
```

## UDP 協定

UDP（User Datagram Protocol）是無連接的不可靠傳輸協定：

```
┌──────────────────────┐         ┌──────────────────────┐
│       TCP             │         │       UDP            │
├──────────────────────┤         ├──────────────────────┤
│ 面向連接              │         │ 無連接              │
│ 可靠                  │         │ 不可靠              │
│ 順序                  │         │ 無順序              │
│ 流量控制              │         │ 無流量控制          │
│ 壅塞控制              │         │ 無壅塞控制          │
│ 較大標頭 (20-60B)    │         │ 較小標頭 (8B)       │
│ 慢                    │         │ 快                  │
└──────────────────────┘         └──────────────────────┘
```

### UDP 應用場景

- 線上遊戲（低延遲優先）
- 影片串流（可容忍少量丟包）
- DNS 查詢（快速失敗重試）
- VoIP（實時性要求高）
- DHCP（簡單請求-響應）

## 常見網路層協定

### ICMP（Internet Control Message Protocol）

用於錯誤報告和診斷：

```bash
# ping 使用 ICMP
ping www.google.com

# traceroute 使用 ICMP
traceroute www.google.com
```

**常見 ICMP 類型**：
- 0: Echo Reply（回應）
- 3: Destination Unreachable（目的地不可達）
- 8: Echo Request（回應請求）
- 11: Time Exceeded（超時）

### ARP（Address Resolution Protocol）

IP 位址到 MAC 位址的映射：

```bash
# 查看 ARP 表
arp -a
```

**ARP 流程**：
1. 主機 A 要發送資料給 IP B
2. 查詢 ARP 表，有無 B 的 MAC
3. 若無，廣播 ARP 請求
4. B 回應自己的 MAC
5. A 快取 B 的 MAC

## 常見應用層協定

### HTTP/HTTPS

HTTP（HyperText Transfer Protocol）是 Web 的基礎協定：

```
HTTP 請求格式：
┌────────────────┬────────────────┐
│ Request Line   │ GET /index.html HTTP/1.1
├────────────────┼────────────────┤
│ Headers        │ Host: example.com
│                │ User-Agent: curl
│                │ Accept: */*
├────────────────┼────────────────┤
│ Body           │ (通常為空用於GET)
└────────────────┴────────────────┘

HTTP 回應格式：
┌────────────────┬────────────────┐
│ Status Line    │ HTTP/1.1 200 OK
├────────────────┼────────────────┤
│ Headers        │ Content-Type: text/html
│                │ Content-Length: 1234
├────────────────┼────────────────┤
│ Body           │ <html>...</html>
└────────────────┴────────────────┘
```

**HTTP 方法**：
- GET：請求資源
- POST：提交資料
- PUT：更新資源
- DELETE：刪除資源
- PATCH：部分更新

**狀態碼**：
- 1xx: 資訊性
- 2xx: 成功（200 OK, 201 Created）
- 3xx: 重定向（301 Moved, 304 Not Modified）
- 4xx: 客戶端錯誤（400 Bad Request, 404 Not Found）
- 5xx: 伺服器錯誤（500 Internal Server Error, 503 Service Unavailable）

### DNS（Domain Name System）

將網域轉換為 IP 位址：

```bash
# 查詢 DNS
nslookup www.google.com
dig www.google.com
host www.google.com
```

**DNS 查詢流程**：
1. 查詢本地 DNS 快取
2. 詢問根伺服器 (.com)
3. 詢問 TLD 伺服器 (google.com)
4. 詢問授權 DNS 伺服器
5. 獲得 IP 位址

### DHCP（Dynamic Host Configuration Protocol）

自動分配 IP 位址：

```
DHCP 流程：
客戶端 ──→ DISCOVER (廣播)
客戶端 ←── OFFER (提供 IP)
客戶端 ──→ REQUEST (請求)
客戶端 ←── ACK (確認)
```

## IP 路由

### 路由表

```bash
# 查看路由表 (Linux)
ip route

# 查看路由表 (Windows)
route print

範例輸出：
default via 192.168.1.1 dev eth0  預設閘道
192.168.1.0/24 dev eth0           本地網路
10.0.0.0/8 via 192.168.1.2       特定路由
```

### 路由演算法

- **靜態路由**：管理員手動配置
- **動態路由**：使用路由協定自動學習
  - RIP：適合小型網路（最大 15 跳）
  - OSPF：適閤中大型網路
  - BGP：跨自治系統（網際網路骨幹）

## NAT（Network Address Translation）

將私有 IP 轉換為公有 IP：

```
┌─────────────────────────────────────────────────────────────┐
│                        NAT 示意                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   內網                          外網                        │
│  ┌─────────┐                   ┌─────────┐                 │
│  │ 客戶端  │                   │ 網際    │                 │
│  │        │─── 192.168.1.100 ──│ 網路    │─── 203.0.113.10 │
│  └─────────┘                   └─────────┘                 │
│        │                              │                    │
│        │  NAT 閘道器 (Router)         │                    │
│        │  192.168.1.1 ────────────── 203.0.113.1          │
│                                                             │
│  轉換表：                                                  │
│  ┌────────────┬────────────────┐                           │
│  │ 內部IP     │ 外部IP:Port    │                           │
│  ├────────────┼────────────────┤                           │
│  │ 192.168.1.100│ 203.0.113.1:50001│                       │
│  └────────────┴────────────────┘                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
