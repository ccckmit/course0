# 虛擬機

虛擬機（Virtual Machine）是一種軟體模擬技術，通過軟體模擬硬體或特定指令集，在單一硬體平臺上執行多個獨立的作業系統與應用環境。虛擬化技術是雲端運算的基礎，實現了硬體資源的有效利用與隔離。

## 虛擬化的類型

### 系統虛擬化

系統虛擬化模擬完整硬體平臺，讓客端作業系統能在隔離環境中執行：

```
┌─────────────────────────────────────────────────────┐
│                  主機硬體                           │
├─────────────────────────────────────────────────────┤
│              虛擬機器監視器 (VMM/Hypervisor)         │
├─────────────┬─────────────┬─────────────┬─────────────┤
│   VM 1      │   VM 2      │   VM 3      │   VM N      │
│  Guest OS   │  Guest OS   │  Guest OS   │  Guest OS  │
│   App       │   App       │   App       │   App      │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

代表系統：
- **QEMU** - 開源的完整系統模擬器
- **VMware Workstation/Fusion** - 商業虛擬化
- **VirtualBox** - 開源 Oracle 虛擬化
- **Hyper-V** - Microsoft 虛擬化

### 程序虛擬化

程序虛擬化模擬特定指令集，讓同一平臺上執行不同架構的程式：

```
┌─────────────────────────────────────────────────────┐
│                  主機作業系統                         │
├─────────────────────────────────────────────────────┤
│              程序虛擬機                             │
├─────────────┬─────────────┬─────────────┬─────────────┤
│  Java App   │  .NET App  │  WebAssembly │ Python    │
│  (JVM)     │  (CLR)    │  (WASM)    │  (Python) │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

代表系統：
- **JVM** - Java 虛擬機器
- **CLR** - .NET 公共語言執行時
- **WebAssembly Runtime** - WASM 執行環境
- **Python VM** - Python 解譯器

### 作業系統虛擬化

作業系統虛擬化共享主機核心，隔離程序與資源：

```
┌─────────────────────────────────────────────────────┐
│                  主機核心 (Kernel)                   │
├─────────────────────────────────────────────────────┤
│     容器引擎 (Docker / containerd)                 │
├─────────────┬─────────────┬─────────────┬─────────────┤
│  Container  │  Container  │  Container  │  Container │
│    A       │    B       │    C       │    D       │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

代表系統：
- **Docker** - 容器平臺
- **containerd** - 容器執行時
- **runc** - 容器執行器

## Hypervisor 類型

### Type 1 (Bare Metal)

Hypervisor 直接在硬體上執行，無需主機作業系統效能最高：

```
┌─────────────────────────────────────────────┐
│              Bare Metal                    │
├─────────────────────────────────────────────┤
│  Hypervisor (直接管理硬體)                  │
├─────────────┬─────────────┬─────────────┤
│   VM 1     │   VM 2     │   VM 3     │
│ Guest OS   │ Guest OS   │ Guest OS   │
└─────────────┴─────────────┴─────────────┘
```

代表：
- **VMware ESXi** - 企業級虛擬化平臺
- **Microsoft Hyper-V** - Windows 伺服器虛擬化
- **Xen** - 開源虛擬化
- **KVM** - Linux 核心虛擬化模組

### Type 2 (Hosted)

Hypervisor 像應用程式一樣在主機 OS 上執行：

```
┌─────────────────────────────────────────────┐
│            Host OS (Linux/Windows)          │
├─────────────────────────────────────────────┤
│          Hosted Hypervisor                   │
├─────────────┬─────────────┬─────────────┤
│   VM 1     │   VM 2     │   VM 3     │
│ Guest OS   │ Guest OS   │ Guest OS   │
└─────────────┴─────────────┴─────────────┘
```

代表：
- **VMware Workstation** - 桌面虛擬化
- **VirtualBox** - 開源虛擬化
- **QEMU** - 模擬器

## CPU 虛擬化

### 硬體輔助虛擬化

現代 CPU 提供硬體輔助虛擬化指令：

```c
// Intel VT-x / AMD-V
// VMX root mode / SVM mode
// 指令: VMXON, VMXOFF, VMCALL, etc.

// 功能
// - 二階分頁表 (EPT / NPT)
// - I/O 虛擬化
// - 中斷虛擬化
```

### 模擬與翻譯

```c
// 完整模擬 (Slow)
// 每次指令都翻譯執行

// 二進位翻譯 (BT)
// 翻譯 Guest 代碼為 Host 代碼
// 緩存翻譯結果

// 半虛擬化 (Paravirtualization)
// 修改 Guest OS 配合 Hypervisor
// 例: Xen, VMware Tools
```

## 記憶體虛擬化

### 分頁虛擬化

```
Guest 虛擬位址 (GVA)
       ↓
Guest 物理位址 (GPA)
       ↓
Host 虛擬位址 (HVA)
       ↓
Host 物理位址 (HPA)
```

### 兩種模式

| 模式 | 說明 | 效能 |
|------|------|------|
| Shadow Paging | Hypervisor 管理影子頁表 | 較慢 |
| Nested Paging | EPT/NPT 硬體支援 | 快 |

### 記憶體超量配置

```c
// 記憶體氣球
// 從 Guest 回收閒置記憶體

// 記憶體壓縮
// 壓縮不活躍頁面

// 置換
// 交換到磁碟 (不推薦)
```

## 網路虛擬化

### 虛擬網路類型

```bash
# 橋接模式
# VM <-> Bridge <-> Physical NIC <-> Network

# NAT 模式
# VM <-> vNIC <-> NAT <-> Physical NIC <-> Network
# (VM 可存取網路，但外部無法主動連線)

# Host-only
# VM <-> vNIC <-> vSwitch <-> Host
# (僅與 Host 通訊)
```

### 軟體定義網路

```c
// Open vSwitch
// - Flow Table
// - VLAN
// - QoS
// - Tunnel (VXLAN, GRE)

// CNI (Container Network Interface)
// - Flannel
// - Calico
// - Weave
```

## Docker 容器

### 容器 vs VM

| 特性 | VM | 容器 |
|------|-----|------|
| 隔離 | 完整隔離 | 共享核心 |
| 開機 | 秒級 | 毫秒級 |
| 映像 | GB 級 | MB 級 |
| 效能 | 無損 | 近無損 |

### Docker 架構

```dockerfile
# Dockerfile
FROM ubuntu:22.04

RUN apt-get update && apt-get install -y nginx

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

```bash
# 建構映像
docker build -t myapp .

# 執行容器
docker run -d -p 8080:80 myapp

# 列出容器
docker ps
```

## QEMU 模擬

### 基本使用

```bash
# 模擬 ARM
qemu-system-arm \
    -M vexpress-a9 \
    -kernel zImage \
    -dtb vexpress-v2p-ca9.dtb \
    -nographic

# 模擬 RISC-V
qemu-system-riscv64 \
    -M virt \
    -kernel vmlinuz \
    -initrd initrd.img \
    -append "root=/dev/vda"

# 模擬 x86
qemu-system-x86_64 \
    -m 2G \
    -cdrom centos.iso \
    -hda disk.qcow2
```

### KVM 加速

```bash
# 啓用 KVM
qemu-system-x86_64 \
    -enable-kvm \
    -m 4G \
    -hda disk.qcow2
```
