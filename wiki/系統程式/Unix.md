# Unix 概述

## 發展歷程

Unix 的發展可以追溯到 1969 年，當時 Ken Thompson 在貝爾實驗室的 PDP-7 電腦上開發了一個簡單的作業系統。這個當初只用了匯編語言編寫、總共約 50 頁代碼的系統，奠定了後來最具影響力作業系統家族的基礎。1973 年，Dennis Ritchie 用 C 語言重寫了 Unix，這使得 Unix 成為首個真正可移植的作業系統，從此開啟了 Unix 稱霸電腦世界的時代。

Unix 的發展經歷了幾個重要的分支。貝爾實驗室先後發布了 Version 1 到 Version 7，其中 Version 6 於 1975 年向大學分發，Version 7 是最後一個貝爾實驗室官方版本。同時，AT&T 將 Unix 授權給學術機構，加州大學伯克利分校基於 Version 6 和 Version 7 發展出了 BSD（Berkeley Software Distribution），成為 Unix 的另一重要分支。1980 年代，AT&T 成立 Unix Systems Laboratories，推出了 System V，成為商業 Unix 的主流。1988 年，AT&T 與多家 Unix 供應商成立 POSIX 標準委員會，奠定了現代 Unix 相容性的基礎。

## 核心設計原則

Unix 的設計哲學深刻影響了整個軟體產業。Ken Thompson 與 Dennis Ritchie 在設計 Unix 時遵循了幾個核心原則，這些原則至今仍是軟體設計的經典指導。

第一個原則是「小即是美」（Small is Beautiful）。Unix 的每個工具只做好一件事，透過組合多個簡單工具來完成複雜任務。這種設計使得每個程式都可以獨立運作、獨立測試，也讓系統更容易理解和維護。Unix 的核心只有約 10,000 行代碼，卻提供了完整的功能，這種簡潔性是 Unix 最重要的特徵之一。

第二個原則是「一切皆檔案」（Everything is a File）。Unix 把所有資源都抽象為檔案，包括普通檔案、目錄、設備、管道、網路 sockets 都以檔案介面呈現。這種統一性讓使用者可以用相同的方式操作不同的資源，也讓程式設計師可以用一致的 API 處理各種 I/O 操作。更重要的是，這種設計極大地簡化了系統的實現，因為所有資源都可以用相同的底層機制來管理。

第三個原則是「組合優先於擴展」（Compose over Extend）。Unix 設計了管道（Pipe）機制，讓一個程式的輸出可以直接作為另一個程式的輸入。這種組合方式讓使用者可以像搭積木一樣組合工具，創造出無限的可能性。一個簡單的 `ls | grep .txt | sort` 命令就展示了管道如何讓三個各自獨立的工具協同工作。

第四個原則是「保持開放」（Keep Open）。Unix 從一開始就允許研究者和大學自由取得原始碼，這種開放策略讓 Unix 得以廣泛傳播和改進。原始碼的可獲得性不僅讓使用者可以根據自己的需求修改系統，也讓無數人從中學習到了作業系統的設計精髓。這種開放精神最終催生了 Linux 和整個開源運動。

## 系統架構

Unix 系統採用分層架構，從硬體到應用程式形成了清晰的抽象層次。這種分層設計使得各層之間可以相對獨立地發展，同時也讓系統更容易理解和使用。

在最底層是硬體，包括 CPU、記憶體、磁碟、網路卡等物理設備。這些硬體由核心（Kernel）直接管理。核心是 Unix 作業系統的心臟，負責處理所有底層操作，包括記憶體管理、行程排程、檔案系統、設備驅動等。核心運行在特權模式（Kernel Mode），拥有访问所有硬件资源的权限，而普通应用程序运行在用户模式（User Mode），只能通过系统调用请求核心提供服务。

在核心之上是系統呼叫介面，這是使用者程式與核心溝通的唯一橋樑。系統呼叫提供了一組受保護的操作，包括檔案操作（open、read、write、close）、行程管理（fork、exec、wait、exit）、記憶體管理（brk、mmap）、網路通信（socket、connect、send、recv）等。POSIX 標準定義了這些系統呼叫的介面，確保了不同 Unix 系統之間的相容性。

再往上是用戶空間的各種服務和工具。Shell 是使用者與系統互動的主要介面，Bourne Shell（sh）、C Shell（csh）、Bourne Again Shell（bash）等提供了命令解釋和腳本執行功能。工具層包含了無數的實用程式，如 ls、cat、grep、sed、awk、find、xargs 等，這些工具遵循 Unix 哲學，每個只完成一個特定任務，但可以透過管道組合實現強大的功能。

## 檔案系統

Unix 的檔案系統是整個系統設計的核心體現。Unix 採用單一樹狀目錄結構，所有檔案都從根目錄（/）開始延伸，形成一個統一的命名空間。這種設計與早期的Multics等多重根目錄設計形成了鮮明對比，簡化了使用者對檔案的認知和管理。

Unix 檔案系統的核心概念是 inode。每個檔案都對應一個 inode 結構，記錄了檔案的元數據，包括檔案類型、權限、所有者、大小、時間戳以及指向資料塊的指標。這種設計將檔案的屬性與名稱分離，名稱存放在目錄中，指向對應的 inode。這種間接引用的方式使得 Unix 可以輕實現硬連結（Hard Link），讓多個檔名指向同一個 inode。

Unix 的設備也以檔案的形式呈現，稱為設備檔案。字符設備（Character Device）以字節為單位進行 I/O，如終端、印表機；區塊設備（Block Device）以固定大小的區塊進行 I/O，如硬碟、光碟。設備檔案位於 /dev 目錄下，使用者可以像操作普通檔案一樣操作設備，這是「一切皆檔案」原則的典型體現。

管 道（Pipe）是 Unix 獨有的創新，它是行程間通信的重要機制。管道是臨時的緩衝區，一端連接寫入端，另一端連接讀取端，實現了單向的位元組流傳遞。Shell 中的豎線（|）就是創建管道的語法。另一種重要的機制是 FIFO（Named Pipe），它是具名的管道，可以讓無親屬關係的行程通過檔案路徑進行通信。

## 行程管理

行程（Process）是 Unix 系統中資源分配和調度的基本單位。每個行程都有獨立的位址空間，包括程式碼段、資料段、堆疊段等。行程之間相互隔離，一個行程的錯誤不會影響其他行程，這保證了系統的穩定性。

Unix 使用 fork 系統呼叫創建新行程。fork 會複製當前行程，創建一個幾乎完全相同的子行程。子行程會獲得父行程位址空間的副本，包括所有打開的檔案描述符。fork 之後，父子行程各自執行不同的程式碼，通常通過 exec 系統呼叫來執行新的程式。這種 fork-exec 模型是 Unix 行程創建和程式執行的核心機制。

Unix 的行程調度採用多級反饋隊列算法，兼顧了響應速度和系統吞吐量。每個行程有一個動態優先級，CPU 使用時間越長優先級越低，這保證了交互式程式能夠獲得更好的響應。Unix 還支持即時行程，通過設定較高的優先級，確保即時任務能夠準時完成。

行程間的同步和通信是 Unix 的重要課題。信號（Signal）是 Unix 提供的進程間通信機制，用於通知行程發生了某個事件。管道和 FIFO 提供了父子行程或兄弟行程之間的數據傳遞。共享記憶體（Shared Memory）允許多個行程共享同一段物理記憶體，實現高效的大數據共享。訊息佇列（Message Queue）提供了非同步的消息傳遞機制。訊號量（Semaphore）用於同步對共享資源的訪問。

## 設備驅動

Unix 的設備驅動程式是核心的一部分，但遵循統一的架構。每個設備都有一個對應的設備驅動，負責處理設備的 I/O 操作。設備驅動分為字符設備驅動和區塊設備驅動兩大類，分別實現不同的介面。

字符設備以字節為單位進行 I/O 操作，讀寫操作是順序的，常見的字符設備包括終端（TTY）、序列埠、印表機等。區塊設備以固定大小的區塊進行 I/O，支持隨機訪問，常見的包括硬碟、光碟、USB 隨身碟等。核心使用緩衝區（Buffer Cache）來快取區塊設備的資料，提高 I/O 效率。

設備驅動與核心的其他部分通過一系列標準介面交互。設備驅動需要實現打開、關閉、讀取、寫入等操作，這些操作透過設備切換表（Device Switch Table）與系統呼叫關聯。設備驅動還需要處理中斷（Interrupt），當設備完成 I/O 操作或發生錯誤時，設備會產生中斷信號，驅動程式需要註冊中斷處理例程來響應這些事件。

## Shell 與指令

Shell 是使用者與 Unix 系統互動的主要介面，它既是命令解釋器也是腳本語言解釋器。Unix 發展出了多種 Shell，每種都有其特點和適用場景。

Bourne Shell（sh）是最早的 Unix Shell，由 Stephen Bourne 開發，是所有 Unix Shell 的鼻祖。它的語法簡潔高效，腳本可移植性好，至今仍是最重要的 Shell 之一。幾乎所有 Unix 系統都提供 sh，POSIX 標準也基於 Bourne Shell 語法制定。

Bourne Again Shell（bash）是 GNU 專案開發的 Shell，是 Linux 系統的預設 Shell。bash 完全相容於 sh，並增加了許多有用的功能，包括命令補全、歷史記錄、陣列支援、算術展開等。bash 是目前最流行的 Shell，幾乎所有 Linux 发行版都預設使用 bash。

C Shell（csh）由 Bill Joy 開發，其語法類似 C 語言因而得名。C Shell 提供了命令列編輯、歷史機制等交互功能，但腳本語法與 sh 不相容。C Shell 的後續改進版本是 tcsh，增加了更多互動功能。

Z Shell（zsh）結合了 bash、tcsh、ksh 等多種 Shell 的特性，提供了強大的補全系統和主題支援。macOS 從 10.15 版本開始將 zsh 作為預設 Shell。

## 網路功能

Unix 從 4.2 BSD 開始引入網路功能，實現了 TCP/IP 協議棧，從此 Unix 成為網路計算的核心平台。BSD Unix 的網路實現被稱為 Berkeley Sockets，成為網路編程的事實標準。

Socket 是 Unix 網路編程的核心抽象。它提供了一種通用的進程間通信機制，不僅可以用於本地進程，還可以用於跨主機的網路通信。Socket API 提供了創建、綁定、監聽、連接、發送、接收等操作，幾乎所有網路應用都基於 Socket 構建。

Unix 網路功能的一個重要特點是將網路連接也視為檔案。網路連接可以使用標準的 read/write 系統呼叫進行 I/O，也可以使用 select、poll、epoll 等機制實現高效的 I/O 多路複用。這種設計讓網路程式可以使用與本地檔案相同的程式設計模型，簡化了開發。

## 權限與安全

Unix 採用基於使用者（User）和群組（Group）的權限模型。每個檔案和目錄都有三組權限：所有者（Owner）、所屬群組（Group）和其他使用者（Others）。每組權限包含讀（r）、寫（w）、執行（x）三種權限，共九個權限位。這種簡潔而強大的權限模型是 Unix 系統安全的基礎。

Unix 使用使用者 ID（UID）和群組 ID（GID）來標識使用者和群組。root 使用者（UID=0）拥有系統的最高權限，可以執行任何操作。普通使用者的權限受限，只能訪問自己擁有或被授權的資源。這種設計確保了系統的安全性和穩定性。

Unix 還提供了更精細的權限控制機制。檔案系統ACL（Access Control List）可以對單個使用者或群組設定更細緻的權限。SELinux（Security-Enhanced Linux）和 AppArmor 等強制訪問控制框架提供了更強大的安全機制，可以控制程式能夠訪問的資源和執行的操作。

## 主要分支

Unix 經過數十年的發展，形成了多個重要的分支，每個分支都有其特點和應用場景。

System V 是 AT&T 推出的商業 Unix 版本，曾是商業 Unix 市場的主流。System V 引入了很多重要特性，包括 System V IPC（訊號量、共享記憶體、訊息佇列）、檔案系統擴展等。Oracle Solaris、IBM AIX、HP-UX 等商業 Unix 都基於 System V。

BSD（Berkeley Software Distribution）是加州大學伯克利分校基於 AT&T Unix 開發的分支，是開源 Unix 的重要源頭。BSD 發展出了多個重要版本，包括 FreeBSD、NetBSD、OpenBSD 等。BSD 的程式碼也貢獻給了 macOS 和 iOS 的核心（Darwin），以及 Sony PlayStation 的系統軟體。

Linux 是最成功的 Unix-like 系統，由 Linus Torvalds 於 1991 年發布。Linux 完全相容於 POSIX 標準，提供了與 Unix 幾乎相同的功能和介面，但原始碼完全重新編寫。Linux 現在是伺服器、雲端運算、嵌入式系統的主流選擇，Android 智慧手機也基於 Linux 核心。

macOS 是 Apple 的桌面作業系統，基於 Darwin（基於 BSD）和 NextSTEP。macOS 提供了圖形化介面，與 Unix 命令列完美結合，是開發者喜愛的平台。iOS 是 macOS 的移動版本，兩者共享 Darwin 核心。

## 影響與遺產

Unix 對計算機科學的影響是深遠的。它不僅是一個作業系統，更是一種設計哲學和文化的載體，塑造了整個軟體產業的發展方向。

在作業系統領域，Unix 的設計思想影響了幾乎所有後續的作業系統。Linux、BSD、macOS 都直接繼承了 Unix 的設計理念。Windows NT 虽然不是 Unix 的分支，但也吸收了很多 Unix 的概念。POSIX 標準的制定確保了 Unix 相容系統之間的互操作性。

在程式語言領域，C 語言因 Unix 而誕生，並且成為系統程式設計的標準語言。C 語言的簡潔性、可移植性和高效性使其成為開發作業系統、編譯器、資料庫等系統軟體的首選。幾乎所有主流程式語言的編譯器都是用 C 語言實現的。

在軟體工程領域，Unix 哲學影響了無數軟體的設計。模組化、可組合、可移植成為軟體設計的基本原則。開源軟體運動也深受 Unix 開放策略的影響，Linux、Apache、MySQL 等開源軟體都是 Unix 文化的延續。

在網路計算領域，Unix 是互聯網的基礎。TCP/IP 協議棧最初在 BSD Unix 上實現，互聯網上大多數伺服器運行在 Linux 或 BSD 上。Apache、Nginx 等 Web 伺服器，Postfix、Sendmail 等郵件伺服器都是 Unix 環境下的經典軟體。

Unix 還催生了眾多重要的工具和技術。版本控制系統 Git、資料庫系統 MySQL/PostgreSQL、網頁伺服器 Apache/Nginx、程式語言 Python/Perl/Ruby 等，都與 Unix 文化和環境密不可分。

## 總結

Unix 的發展歷程展現了一個簡潔而強大的系統如何影響整個計算機世界。從 1969 年的 PDP-7 上的小型系統，到如今無處不在的 Linux 和移動設備上的 Android，Unix 及其衍生系統已經成為現代資訊科技的基石。Unix 的設計哲學——簡潔、優雅、可組合、可移植——不僅是作業系統設計的經典範例，也是整個軟體工程領域的重要指導原則。

Unix 的成功在於它找到了一個完美的平衡點：既足夠簡單讓人能够理解，又足夠強大滿足各種需求。它允許用户根據自己的需求自由地组合和扩展，這種靈活性使得 Unix 能夠適應從嵌入式系統到超級電腦的各種環境。今天，當我們使用智慧手機、瀏覽網頁、雲端計算的時候， Unix 的遺產無處不在。
