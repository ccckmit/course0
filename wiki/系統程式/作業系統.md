# 作業系統

作業系統（Operating System, OS）是電腦系統中最基礎也是最重要的系統軟體，它扮演著硬體與應用程式之間的橋樑角色。作業系統的核心目標是有效管理計算機資源，並提供使用者與程式一個簡潔、抽象的介面來使用這些資源。從個人電腦到超級電腦，從智慧手機到嵌入式系統，作業系統無處不在，是現代資訊科技的基石。

## 作業系統的發展歷史

作業系統的發展經歷了漫長的演進過程：

### 第一代（1950s-1960s）：早期批處理系統

早期的電腦沒有作業系統，程式設計師直接操作硬體。隨後出現了簡單的批處理系統，自動順序執行多個工作，無需人工介入。

### 第二代（1960s-1970s）：分時系統與多重程式設計

- **Multics**：麻省理工學院、貝爾實驗室、通用電氣合作的專案，開創了分時概念
- **UNIX**：由 Ken Thompson 和 Dennis Ritchie 在貝爾實驗室開發，成為最具影響力的作業系統
- **VMS**：DEC 公司的作業系統

### 第三代（1980s-1990s）：個人電腦與網路

- **MS-DOS**：IBM PC 的標準作業系統
- **Mac OS**：Apple 的圖形化作業系統
- **Windows**：Microsoft 的視窗系統
- **Linux**：Linus Torvalds 開發的開源作業系統

### 第四代（2000s-至今）：行動與雲端運算

- **iOS/Android**：智慧手機作業系統
- **雲端作業系統**：虛擬化技術的成熟

## 作業系統的核心功能

### 1. 資源管理

作業系統的首要職責是有效管理計算機的各類資源：

#### 1.1 處理器管理

- **CPU 排程**：決定哪個行程在何時獲得 CPU 執行權
- **上下文切換**：保存和恢復行程的執行狀態
- **多核負載平衡**：在多處理器系統中分配工作負載

#### 1.2 記憶體管理

- **虛擬記憶體**：提供比實際物理記憶體更大的位址空間
- **分頁/分段**：將記憶體劃分為固定或可變大小的單位
- **記憶體分配**：動態分配和回收記憶體空間

#### 1.3 I/O 設備管理

- **設備驅動程式**：與硬體設備溝通的軟體模組
- **緩衝區管理**：減少 I/O 操作的等待時間
- **設備排程**：優化多個 I/O 請求的處理順序

### 2. 程序管理

程序（Process）是作業系統進行資源分配和調度的基本單位：

#### 2.1 程序的生命週期

```
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│  新建   │───→│  就緒   │───→│  執行   │───→│  等待   │───→│  終止   │
└────────┘    └────────┘    └────────┘    └────────┘    └────────┘
     │              │              │              │
     └──────────────┴──────────────┴──────────────┘
                          就緒/執行
```

- **新建（New）**：程序正在被創建
- **就緒（Ready）**：等待獲得 CPU 執行權
- **執行（Running）**：正在 CPU 上執行指令
- **等待（Waiting）**：等待某個事件（如 I/O 完成）
- **終止（Terminated）**：程序執行完畢

#### 2.2 程序控制塊（PCB）

每個程序都有一個稱為程序控制塊（Process Control Block, PCB）的資料結構，儲存：

- 程序ID（PID）
- 程序狀態
- 程式計數器（PC）
- CPU 暫存器
- 記憶體管理資訊
- I/O 狀態資訊
- 帳務資訊

### 3. 執行緒管理

執行緒（Thread）是程序內的輕量級執行單位，同一程序內的多個執行緒共享：

- **共享資源**：記憶體位址空間、全域變數、打開的檔案
- **獨立資源**：堆疊、程式計數器、暫存器集合

執行緒相比程序的優勢：

- **創建成本低**：創建執行緒比創建程序快 10-100 倍
- **通訊容易**：執行緒間可以直接共享資料，無需 IPC
- **並行性**：可充分利用多核處理器

執行緒實現方式：

- **使用者層執行緒**：由用戶空間的執行緒庫管理
- **核心層執行緒**：由作業系統核心直接管理
- **混合式**：結合兩者優點（如 Linux 的 NPTL）

### 4. 記憶體管理

記憶體管理是作業系統最複雜的功能之一：

#### 4.1 連續記憶體分配

- **首次適應（First Fit）**：選擇第一個足夠大的空閒區塊
- **最佳適應（Best Fit）**：選擇最小的足夠大的空閒區塊
- **最差適應（Worst Fit）**：選擇最大的空閒區塊

#### 4.2 分頁（Paging）

將記憶體劃分為固定大小的頁框（Page Frame），虛擬記憶體分為頁（Page）：

- **頁表**：映射虛擬頁到物理頁框
- **TLB**：Translation Lookaside Buffer，硬體加速的頁表查找
- **換頁（Page Replacement）**：當物理記憶體不足時，將頁面置換到磁碟

常見的換頁演算法：

- **FIFO**：最早進入記憶體的頁面最先被置換
- **LRU（Least Recently Used）**：最近最少使用的頁面被置換
- **Clock（Second Chance）**：改善的 FIFO

#### 4.3 分段（Segmentation）

將記憶體劃分為邏輯上的段（Segment），如程式碼段、資料段、堆疊段。每個段有自己的名稱和大小，實現更自然的記憶體保護。

### 5. 檔案系統

檔案系統負責在二級儲存（硬碟、SSD）上組織和存取資料：

#### 5.1 檔案系統結構

- **目錄結構**：樹狀階層結構
- **檔案屬性**：名稱、類型、大小、創建時間、權限
- **空間分配**：連續分配、鏈接分配、索引分配

#### 5.2 常見檔案系統

| 檔案系統 | 作業系統 | 特點 |
|----------|----------|------|
| ext4 | Linux | 日誌檔案系統，支援大檔案 |
| NTFS | Windows | 支援壓縮、加密、許可權 |
| APFS | macOS | 針對 SSD 優化 |
| FAT32 | 跨平臺 | 兼容性高，但單檔案受限 |

#### 5.3 虛擬檔案系統（VFS）

提供統一的介面支援多種底層檔案系統，隱藏具體實現差異。

### 6. 網路功能

現代作業系統提供完整的網路功能：

- **TCP/IP 協定堆疊**：傳輸控制協定、網際協定
- **Socket API**：網路程式設計的標準介面
- **路由**：決定資料包的最佳路徑
- **防火牆**：網路安全控制

### 7. 安全與保護

作業系統必須保護系統資源不被未授權訪問：

- **使用者身份驗證**：密碼、指紋、智慧卡
- **權限控制**：讀、寫、執行權限
- **隔離**：程序間的記憶體隔離
- **Audit**：記錄系統活動

## 系統呼叫

系統呼叫（System Call）是使用者程式與作業系統核心溝通的介面。它們提供了一組受保護的操作，讓使用者程式能夠請求作業系統的服務。

### 主要系統呼叫類別

#### 程序管理

| 系統呼叫 | 功能 |
|----------|------|
| fork() | 創建新程序 |
| exec() | 執行新程式 |
| wait() | 等待子程序結束 |
| exit() | 終止目前程序 |
| getpid() | 取得程序ID |

#### 檔案操作

| 系統呼叫 | 功能 |
|----------|------|
| open() | 打開檔案 |
| read() | 讀取資料 |
| write() | 寫入資料 |
| close() | 關閉檔案 |
| seek() | 移動檔案指標 |

#### 記憶體管理

| 系統呼叫 | 功能 |
|----------|------|
| mmap() | 記憶體映射 |
| brk() | 修改堆疊大小 |
| mprotect() | 修改記憶體權限 |

#### 程序通訊

| 系統呼叫 | 功能 |
|----------|------|
| pipe() | 創建管道 |
| socket() | 創建socket |
| bind() | 綁定位址 |
| connect() | 連線 |
| send()/recv() | 傳送/接收資料 |

## 核心概念深入理解

### 核心（Kernel）vs 使用者空間（User Space）

作業系統的核心（Kernel）是系統最底層的軟體，擁有硬體的最高控制權：

- **核心態（Kernel Mode）**：可以執行任何硬體指令，訪問任何記憶體位址
- **使用者態（User Mode）**：只能執行受限的指令集，訪問受限的記憶體

現代作業系統採用微核心（Microkernel）設計，將核心功能最小化，更多功能在使用者空間實現，提高系統可靠性和可維護性。

### 排程演算法

CPU 排程決定哪個就緒程序應該獲得 CPU 資源：

#### 先進先出（FIFO/FCFS）

按到達順序排程，簡單但可能導致短程序等待長程序。

#### 最短工作優先（SJF）

選擇估計執行時間最短的程序，理論上最優但需要預先知道執行時間。

#### 時間片輪轉（RR）

每個程序分配固定的時間片，輪流執行，適合分時系統。

#### 優先級排程

根據程序優先級決定執行順序，可結合搶佔式或非搶佔式策略。

### 同步與互斥

多個執行緒同時訪問共享資源時需要同步：

#### 臨界區（Critical Section）

一次只能有一個執行緒執行的程式碼區域。

#### 互斥鎖（Mutex）

確保一次只有一個執行緒可以進入臨界區。

#### 訊號量（Semaphore）

更通用的同步原語，可控制同時訪問資源的執行緒數量。

#### 監視器（Monitor）

高層次的同步機制，封裝共享資料和操作。

## 主要作業系統介紹

### Linux

由 Linus Torvalds 於 1991 年開始開發的開源作業系統：

- **核心**：Monolithic kernel，支援模組動態載入
- **Shell**：Bash、Zsh 等
- **套件管理**：APT、YUM、DNF
- **發行版**：Ubuntu、Debian、Fedora、Arch Linux

### Windows

Microsoft 開發的商業作業系統：

- **核心設計**：Hybrid kernel
- **圖形介面**：Windows Shell
- **API**：Win32 API、WinRT
- **版本**：Windows 10/11、Windows Server

### macOS

Apple 基於 Darwin（Unix）和 NextSTEP 開發的作業系統：

- **核心**：XNU（混合核心）
- **檔案系統**：APFS
- **圖形介面**：Aqua
- **硬體**：專為 Apple 硬體設計

### Android

基於 Linux 核心的智慧手機作業系統：

- **框架**：Application framework
- **應用程式**：Java/Kotlin
- **虛擬機器**：Dalvik/ART
