# 系統程式的歷史

系統程式（System Programming）是電腦科學中開創性的領域，從作業系統到程式語言，從編譯器到網路協定，系統程式的演進塑造了現代資訊科技的基石。這段歷史始於 1940 年代的電子計算機，經過近八十年的發展，構成了當今數位世界的底層基礎。

## 早期計算機（1940-1950 年代）

### ENIAC 與 EDVAC

1945 年，賓州大學的 ENIAC（Electronic Numerical Integrator and Computer）問世，這是第一台通用電子數值計算機。然而 ENIAC 需要手動重新接線來改變程式，效率極低。

1945 年，John von Neumann 提出了「儲存程式概念」（Stored-Program Concept），指出程式和資料應儲存在同一記憶體中。1951 年的 EDVAC（Electronic Discrete Variable Automatic Computer）首次實現了這一概念，成為現代計算機的原型。

### 組合語言的誕生

早期程式設計直接使用機器碼（純數字），繁瑣且容易出錯。1949 年，Short Code 成為第一個高階程式語言，但仍需要解釋執行。

1950 年代初期，組合語言（Assembly Language）出現，以助記符號替代二進制碼：
- 1953 年，IBM 701 的 SOAP（Symbolic Optimizing Assembly Program）
- 1955 年，IBM 704 的 FAP（Fortran Assembly Program）

組合語言提高了程式設計效率，但仍是低階的、與特定硬體綁定的。

## 作業系統的黎明（1950-1960 年代）

### 批處理系統

早期計算機沒有作業系統，程式設計師直接操作硬體。1950 年代中期，出現了簡單的批處理系統（Batch Processing），將一批 jobs 自動順序執行，無需人工介入。

**GM-NAA I/O**（1956）是第一個廣泛使用的批處理系統，由通用汽車和北美航空開發，運行於 IBM 701。

### 監控程式（Monitor）

監控程式是作業系統的雛形，提供：
- 自動載入下一個程式
- 基本的輸入/輸出控制
- 錯誤處理

### Atlas 與分頁系統

1962 年，Manchester University 的 Atlas 電腦實現了分頁式虛擬記憶體，成為現代作業系統記憶體管理的先驅。Atlas 還引入了中斷機制和通道 I/O。

### CTSS 與分時系統

1961 年，MIT 的 CTSS（Compatible Time-Sharing System）問世，這是第一個成功實現分時（Time-Sharing）的系統。多個使用者可以同時使用計算機，互動式計算開始普及。

## UNIX 的誕生（1969-1970 年代）

### Multics 計畫

1964 年，MIT、貝爾實驗室和通用電氣啟動了 Multics 計畫，目標是建立一個大型、共時、多使用者的作業系統。

Multics 失敗了——它過於複雜和昂貴。但它留下了豐富的遺產：
- 分層檔案系統
- 動態連結
- 驚人的安全性架構

### Ken Thompson 與 UNIX

1969 年，貝爾實驗室的 Ken Thompson 在 DEC PDP-7 上開發了一個簡化的單使用者作業系統，命名為 UNICS（UNiplexed Information and Computing Service），後來簡稱為 UNIX。

Thompson 用彙編語言撰寫了這個系統。隨後，他開發了 B 語言（基於 BCPL），以及第一個文字編輯器 `ed`。

### Dennis Ritchie 與 C 語言

1971 年，Dennis Ritchie 開始在 UNIX 上開發 C 語言。C 結合了高階語言的可讀性和低階語言的效率，極大增強了 UNIX 的可移植性。

1973 年，Thompson 和 Ritchie 用 C 重寫了 UNIX。這是里程碑意義的成就：
- 作業系統第一次用高階語言重寫
- UNIX 可以輕鬆移植到新硬體
- 奠定了「軟體向下相容」的文化

### UNIX 哲學

UNIX 及其文化催生了一系列深刻的程式設計哲學：

> "Write programs that do one thing and do it well."
> "Write programs to work together."
> "Write programs to handle text streams, because that is a universal interface."
> — Doug McIlroy

這些原則影響了數十年的軟體設計。

## C 語言的標準化（1970-1990 年代）

### K&R C

1978 年，Brian Kernighan 和 Dennis Ritchie 出版了《The C Programming Language》（俗稱 K&R），成為 C 語言事實上的標準。

1983 年，美國國家標準協會（ANSI）成立 X3J11 委員會，制定 C 語言的正式標準。

### ANSI C (C89)

1989 年，ANSI 發布了 ANSI C（C89），這是第一個正式的 C 標準。1990 年，ISO 採納為 ISO C90。

標準定義了：
- C 標準函式庫
- 語法和語意的正式規格
- 移植性的保證

### C99 與 C11

- **C99（1999）**：引入 `inline` 函數、`//` 註釋、變長陣列、布尔型 `_Bool`
- **C11（2011）**：多執行緒支援、泛型選擇（`_Generic`）、匿名結構/聯合
- **C17/C23**：持續改進與修正

## GCC 與自由軟體運動（1980-1990 年代）

### GNU 計畫

1983 年，Richard Stallman 發起了 GNU 計畫（GNU's Not Unix），目標是建立完全自由（free as in freedom）的作業系統：

- **GNU Manifesto（1985）**：闡述自由軟體理念
- **GNU General Public License（GPL, 1989）**：保證軟體自由的許可協議
- **GNU Hurd**：GNU 的核心（但延遲多年未完成）

### GCC 的誕生

1987 年，Richard Stallman 以 Pastel 語言的編譯器為基礎，開發了 GNU C Compiler（gcc）。後來擴展支持多語言，成為 GNU Compiler Collection。

GCC 的貢獻：
- 開源的、生產級品質的編譯器
- 成為 Linux 系統默認編譯器
- 推動了 GCC 家族的繁榮（g++, gfortran, Java, Go 前端）

### Linux 的誕生

1991 年，芬蘭學生 Linus Torvalds 開始開發一個類 UNIX 核心：

> "Hello everybody out there using minix -
> I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones."
> — Linus Torvalds, 1991-08-25

Linux 結合了 GNU 工具鏈（gcc, glibc, binutils）和 MINIX 的部分設計理念，迅速發展成為最重要的伺服器、作業系統。

## 現代系統程式（2000 年代至今）

### LLVM 與現代編譯器

2000 年，Chris Lattner 在伊利諾大學開始開發 LLVM（Low Level Virtual Machine）。LLVM 的革命性設計：

- **模組化架構**：前端/中端/後端分離
- **LLVM IR**：統一的、中立的 Intermediate Representation
- **Clang**：高性能的 C/C++ 前端
- **MLIR**：多層次中級表示，支援硬體加速器

LLVM 被 Apple（Xcode）、Google（Android）、Microsoft（Visual Studio）等廣泛採用。

### Rust 的崛起

2006 年，Mozilla 的 Graydon Hoare 開始開發 Rust。Rust 的核心理念是：
- 記憶體安全（無 GC，無空指標，無資料競爭）
- 零成本抽象
- 適合系統程式設計

2015 年 Rust 1.0 發布，此後在作業系統（Linux、RISC-V）、網路服務、區塊鏈等領域快速普及。

Rust 的所有權系統（ownership）和借用檢查器（borrow checker）在編譯時消滅了大量傳統 C/C++ 難以避免的錯誤類別。

### Go 語言

2009 年，Google 發布了 Go 語言，由 Robert Griesemer、Rob Pike 和 Ken Thompson 設計。

Go 的特點：
- 簡潔的語法（刻意限制語言複雜度）
- 內建並發（goroutine + channel）
- 快速的編譯
- 垃圾回收

Go 迅速成為雲端服務和網路服務的主流語言，Docker、Kubernetes、Terraform 等基礎設施軟體都使用 Go 撰寫。

### 容器與虛�擬化

2013 年，Docker 發布，徹底改變了應用程式部署。Docker 建立在 Linux Container（lxc）技術之上：

- **隔離**：每個容器有獨立的檔案系統、網路、程序空間
- **輕量**：比虛擬機更快啟動、更少資源消耗
- **可移植**：一次構建，處處執行

2014 年，Kubernetes（K8s）發布，成為容器編排的事實標準。

## 網路與協定（1960 年代至今）

### ARPANET 與 TCP/IP

1969 年，ARPANET 連接了第一批四台電腦，標誌著網際網路的誕生。

1974 年，Vint Cerf 和 Bob Kahn 發表了 TCP 協定論文。1983 年 1 月 1 日，ARPANET 正式採用 TCP/IP，此後 TCP/IP 成為網路標準。

### UNIX 網路程式設計

UNIX 發明了 Socket API（1971），成為網路程式設計的標準介面：

```c
// 經典的 TCP Socket 伺服器
int server_fd = socket(AF_INET, SOCK_STREAM, 0);
bind(server_fd, (struct sockaddr *)&addr, sizeof(addr));
listen(server_fd, 128);
int client_fd = accept(server_fd, NULL, NULL);
read(client_fd, buffer, sizeof(buffer));
write(client_fd, response, strlen(response));
close(client_fd);
```

### HTTP 與 Web

1989 年，Tim Berners-Lee 在 CERN 發明了 World Wide Web。HTTP 協定從 0.9 到 1.0、1.1，再到 HTTP/2 和 HTTP/3，持續演化。

## 系統程式的文化遺產

### 開放原始碼

從 GNU/Linux 到 LLVM，從 Apache 到 Kubernetes，開放原始碼已成為系統程式的主流模式。開放的系統軟體：
- 加速了創新
- 提高了安全性（審計 vs. 隱晦）
- 建立了全球協作文化

### Unix 設計原則

1. **簡潔**：每個程式做一件事
2. **可組合**：程式的輸出可作為另一程式的輸入
3. **文本優先**：文字流是通用的介面
4. **小而美**：工具越小，越容易理解和組合

### 程式設計教育

從 Harvey Mudd 的 C 語言課程到 MIT 的「作業系統」（xv6），系統程式設計一直是電腦科學教育的核心。

## 未來趨勢

### 記憶體安全語言的普及

C/C++ 的安全問題促使業界轉向 Rust、Go、Carbon 等記憶體安全語言。Google、Microsoft、Apple 等大公司宣布將在系統軟體中使用 Rust。

### 形式化驗證

seL4、F*、Coq 等形式化方法開始應用於作業系統和編譯器的驗證。Rust 的 unsafe 程式碼驗證也是活躍的研究領域。

### 新興架構

RISC-V、Arm64 等開放架構的興起，催生新的系統軟體需求。WebAssembly 作為通用的位元組碼格式，正在成為新的系統層。
