# 程式語言

程式語言是電腦科學中用於表達計算過程的人工語言。從機器碼到高階語言，從編譯型到直譯型，程式語言的演化反映了電腦硬體的發展和人類對計算抽象的不斷追求。理解程式語言的設計原理、實作機制及其與底層系統的關係，是掌握系統程式的關鍵基礎。

## 程式語言的發展歷程

### 機器語言與組合語言

最早的程式直接以二進制機器碼撰寫，每條指令對應 CPU 能直接執行的操作。這種方式雖然能完全控制硬體，但極度繁瑣且容易出錯。組合語言的出現解決了這一問題——它以助記符號（mnemonics）替代二進制碼，如 `MOV AX, 5` 代表將值 5 移入暫存器 AX，同時引入標籤、巨集等機制提升可讀性和可維護性。

組合語言與特定 CPU 架構緊密耦合。x86 組合語言與 ARM 組合語言有著截然不同的語法和指令集，這使得跨平臺程式設計變得困難。這也是高階語言產生的根本驅動力。

### 高階語言的誕生

1950 年代末期，高階語言的概念開始成形：

- **FORTRAN（1957）**：由 John Backus 在 IBM 開發的科學計算語言，被視為第一個成功的高階語言
- **LISP（1958）**：由 John McCarthy 發明的函數式語言，引入了閉包、垃圾回收等概念
- **COBOL（1959）**：面向商業資料處理的語言，強調可讀性和標準化

這些語言的共同特點是：語法更接近人類自然語言，程式設計師無需直接處理暫存器配置和記憶體位址等硬體細節。

### 結構化與物件導向時代

1970 年代至 1990 年代見證了程式設計範式的革命性演進：

- **C 語言（1972）**：由 Dennis Ritchie 在貝爾實驗室開發，結合了高階語言的可移植性和組合語言的效率，成為系統程式設計的基石
- **Pascal（1970s）**：由 Niklaus Wirth 設計的教學語言，推動了結構化程式設計
- **C++（1985）**：由 Bjarne Stroustrup 開發，在 C 的基礎上引入物件導向特性
- **Java（1995）**：Sun Microsystems 推出的跨平臺語言，以虛擬機器實現「寫一次，處處執行」
- **Python（1991）**：Guido van Rossum 設計的腳本語言，以簡潔易學著稱

## 程式語言的分類

### 依執行方式分類

#### 編譯型語言

編譯型語言在程式執行前需要經過編譯器完整翻譯，將原始碼轉換為機器碼或中介表示式：

| 語言 | 編譯器 | 特點 |
|------|--------|------|
| C/C++ | GCC, Clang, MSVC | 直接編譯為機器碼，效能極佳 |
| Rust | rustc | 安全並發，記憶體 guarantees |
| Go | gc | 簡潔語法，並行支援 |
| Fortran | gfortran, ifort | 科學計算高效能 |

編譯型語言的優勢在於執行效率——轉換後的機器碼可以直接由 CPU 執行，無需額外的執行環境。這使其特別適合作業系統、驅動程式、遊戲引擎等效能關鍵場景。

#### 直譯型語言

直譯型語言在程式執行時由直譯器逐行翻譯並執行原始碼：

```python
# Python 直譯器逐行執行
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
```

直譯型語言的優勢是開發效率高、跨平臺性強、除錯方便，但執行效能通常低於編譯型語言。

#### 位元組碼語言

位元組碼語言採用折衷策略：先編譯為平臺無關的位元組碼，再由虛擬機器直譯執行：

```
原始碼 (.java) → Java 編譯器 → 位元組碼 (.class) → JVM 直譯執行
原始碼 (.py)   → Python 編譯器 → 位元組碼 (.pyc)  → Python VM
```

這種設計兼具編譯型的跨平臺能力和直譯型的開發彈性。JVM（Java Virtual Machine）的成功催生了 Kotlin、Scala、Clojure 等 JVM 上的新語言。

### 依型別系統分類

#### 靜態定型語言

靜態定型語言在編譯時確定每個變數的型別：

```rust
// Rust - 靜態定型，編譯時檢查型別錯誤
fn add(a: i32, b: i32) -> i32 {
    a + b
}

let result: i32 = add(5, 3);
let wrong: f64 = add(5, 3);  // 編譯錯誤：型別不匹配
```

優點：及早發現型別錯誤、編譯器可優化、IDE 支援好。代表語言包括 C、C++、Java、Rust、Go。

#### 動態定型語言

動態定型語言的型別在執行時才確定：

```python
# Python - 動態定型，型別由執行時值決定
def add(a, b):
    return a + b

print(add(5, 3))       # 輸出: 8
print(add("Hello, ", "World"))  # 輸出: Hello, World
print(add([1, 2], [3, 4]))     # 輸出: [1, 2, 3, 4]
```

優點：語法簡潔、彈性高、適合快速開發。代表語言包括 Python、JavaScript、Ruby、PHP。

### 依程式設計範式分類

#### 程序式程式設計

程序式語言以程序（procedure）或函數為基本組織單元，程式由一系列執行步驟組成：

```c
// C 語言 - 程序式程式設計
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 5, y = 10;
    swap(&x, &y);
    printf("x=%d, y=%d\n", x, y);
    return 0;
}
```

#### 物件導向程式設計

物件導向語言以物件為核心，結合封裝、繼承、多型三大特性：

```cpp
// C++ - 物件導向程式設計
class Stack {
private:
    vector<int> data;
public:
    void push(int val) { data.push_back(val); }
    int pop() {
        int top = data.back();
        data.pop_back();
        return top;
    }
};
```

#### 函數式程式設計

函數式語言強調不可變性、高階函數和閉包：

```haskell
-- Haskell - 函數式程式設計
fibonacci :: [Integer]
fibonacci = 0 : 1 : zipWith (+) fibonacci (tail fibonacci)

main = print $ take 20 fibonacci
```

函數式程式設計的優勢包括：並行化簡單、程式碼簡潔、便於推理和測試。

#### 邏輯式程式設計

邏輯式語言以邏輯推論為基礎，程式由事實和規則組成：

```prolog
% Prolog - 邏輯式程式設計
parent(tom, bob).
parent(bob, ann).

grandparent(X, Z) :- parent(X, Y), parent(Y, Z).

% 查詢: grandparent(tom, Who).
% 回答: Who = ann
```

## 程式語言的實作機制

### 語法與語意

程式語言由語法（syntax）和語意（semantics）兩部分定義：

- **語法**：決定哪些字元序列是合法的語言結構
- **語意**：決定這些結構的意義

形式文法（formal grammar）提供了一種嚴謹的方式描述語法。Chomsky 階層將文法分為四種類型，對應不同複雜度的語言描述能力。編譯器的前端（詞法分析、語法分析）負責將原始碼轉換為語法樹，後端（語意分析、碼生成）負責將語法樹轉換為可執行碼。

### 執行環境

#### 堆疊式虛擬機器

以 JVM 和 .NET CLR 為代表的堆疊式虛擬機器：
- 以堆疊為主要運算環境
- 位元組碼指令從堆疊取運算元，將結果推回堆疊
- 即時編譯器（JIT）將熱點位元組碼編譯為機器碼

#### 暫存器式虛擬機器

以 Lua VM 和 WebAssembly 為代表的暫存器式虛擬機器：
- 類似真實硬體，有限數量的虛擬暫存器
- 指令指定運算元位置，而非僅依賴堆疊
- 更高的指令密度和更少的記憶體訪問

### 記憶體管理

不同語言採用不同的記憶體管理策略：

| 管理方式 | 語言 | 優點 | 缺點 |
|----------|------|------|------|
| 手動管理 | C, C++ | 完全控制、高效 | 記憶體洩漏風險 |
| 垃圾回收 | Java, Python, Go | 安全、簡易 | 執行時開銷、暫停 |
| 區域所有權 | Rust | 無 GC 開銷、安全 | 學習曲線 |
| 參照計數 | Swift, Perl | 即時釋放 | 循環參照問題 |

## 主流程式語言介紹

### C 語言

C 語言是系統程式設計的基石，幾乎所有現代作業系統的核心都是用 C 撰寫的。C 的設計哲學是「相信程式設計師」——語言本身不做太多約束，讓程式設計師有最大程度的控制權。這既是 C 的力量，也是它的危險所在：緩衝區溢位、空指標解引用等錯誤往往導致安全漏洞。

C 的關鍵特性：
- 指標（pointer）和指標運算
- 手動記憶體管理（malloc/free）
- 原始型別直接對應硬體
- 預處理器（macros）
- 函數指標

### Rust 語言

Rust 是 Mozilla 開發的系統程式語言，其核心理念是在不犧牲效能的前提下，提供記憶體安全和並行安全。Rust 通過「所有權系統」（ownership）和「借用檢查器」（borrow checker）在編譯時排除記憶體錯誤，而不需要垃圾回收器。

Rust 的關鍵特性：
- 所有權（ownership）與借用（borrowing）
- 生命週期（lifetimes）
-  trait 系統（類似介面）
- 無包袱的並行（fearless concurrency）

### Python 語言

Python 以其簡潔優雅的語法著稱，是最受歡迎的教學語言和快速原型開發語言。Python 的「一種最佳做法」（one obvious way）哲學使得程式碼易讀易懂。Python 廣泛應用於網頁開發、資料科學、機器學習、自動化腳本等領域。

Python 的關鍵特性：
- 動態定型、解釋執行
- 縮排語法（indent-based syntax）
- 豐富的標準庫和第三方生態
- GIL 限制下的多執行緒

### Go 語言

Go 是 Google 開發的系統程式語言，以簡潔、快速、安全為設計目標。Go 的語法極度精簡，學習曲線平緩，特別適合雲端服務和網路服務開發。Goroutine 和 Channel 機制使得併發程式設計變得優雅而強大。

Go 的關鍵特性：
- Goroutine（輕量級執行緒）
- Channel（併發通訊）
- 記憶體回收（延遲低）
- 標準化工具鏈（gofmt, go test）

## 程式語言的未來趨勢

### 領域特定語言

越來越多的領域特定語言（DSL）出現，以更好地服務特定應用場景：

- **SQL**：資料庫查詢語言
- **LaTeX**：排版語言
- **Shader languages**：GPU 程式設計
- **TensorFlow/PyTorch**：深度學習框架

### 語言互操作性

現代程式設計越來越依賴多種語言的協作：

- **Foreign Function Interface (FFI)**：語言間函數呼叫
- **WebAssembly**：跨語言的執行環境
- **LLVM**：統一的編譯器基礎設施

### 形式化驗證

隨著軟體複雜度增加，形式化驗證語言開始受到關注。這類語言（如 Coq、Idris）允許數學證明來確保程式正確性，應用於安全關鍵系統。
