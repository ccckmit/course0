# 區塊鏈與比特幣

區塊鏈（Blockchain）是一種去中心化的分散式帳本技術，最初作為比特幣（Bitcoin）的底層技術被提出。2008 年，一位化名中本聰（Satoshi Nakamoto）的個人或團體發表了比特幣白皮書《Bitcoin: A Peer-to-Peer Electronic Cash System》，標誌著區塊鏈技術的誕生。

## 比特幣的誕生背景

### 中心化金融系統的問題

傳統金融系統存在諸多痛點：
- **第三方信任**：需要銀行等中介機構
- **交易可逆性**：信用卡可撤銷，商家可能被欺詐
- **去個性化**：必須經過身份驗證
- **機構停機**：系統維護時無法交易
- **門檻高**：偏遠地區難以獲得銀行服務

### 密碼朋克運動

1990 年代，密碼朋克（Cypherpunk）運動主張用密碼學保護個人隱私。比特幣匯聚了這一流派數十年的思想：

- Wei Dai 的 b-money
- Nick Szabo 的 Bit Gold
- Hal Finney 的 RPOW（可重複使用的工作量證明）

## 比特幣的基本概念

### 去中心化

比特幣網路沒有中央伺服器：
- **節點（Node）**：運行比特幣軟體的全節點
- **礦工（Miner）**：驗證並記錄交易的節點
- **錢包（Wallet）**：儲存私鑰的軟體

任何人可以下載比特幣客戶端（如 Bitcoin Core）加入網路。

### 數位貨幣的「雙花問題」

數位貨幣可以低成本複製，如何防止同一筆錢被花費兩次？

```
攻擊情境：
1. Alice 向 Bob 支付 1 BTC
2. Alice 同時向自己支付 1 BTC（隱藏交易）
3. 她廣播偽造的區塊包含自己的交易
4. 如果成功，Bob 收不到錢但 Alice 保留了 BTC
```

比特幣透過區塊鏈和共識機制解決了這一問題。

## 區塊鏈的結構

### 區塊結構

每個區塊包含：
- **區塊頭（Block Header）**：80 位元組
  - 版本號
  - 前一個區塊的雜湊值
  - Merkle 根
  - 時間戳
  - 難度目標
  - Nonce（隨機數）
- **交易列表（Transactions）**

```
┌─────────────────────────────────────┐
│           區塊 N                      │
├─────────────────────────────────────┤
│ Block Header (80 bytes)              │
│ ├─ version: 1                        │
│ ├─ prev_block: 0x7a3d...             │
│ ├─ merkle_root: 0x9f2e...           │
│ ├─ timestamp: 1700000000            │
│ ├─ bits: 0x1a05a8e4                  │
│ └─ nonce: 2083236893                │
├─────────────────────────────────────┤
│ Transactions                         │
│ ├─ TX 1: Coinbase (挖礦獎勵)         │
│ ├─ TX 2: Alice → Bob                │
│ ├─ TX 3: Carol → Dave               │
│ └─ ...                              │
└─────────────────────────────────────┘
```

### 區塊鏈

區塊按時間順序串聯，形成不可篡改的鏈：

```
Genesis ──── Block 1 ──── Block 2 ──── Block 3 ──── ...
   │           │           │           │
   │    上一個區塊的    上一個區塊的   上一個區塊的
   │    雜湊值=Gen    雜湊值=Blk1   雜湊值=Blk2
```

修改歷史區塊會改變其雜湊值，導致後續所有區塊失效。

### Merkle 樹

交易列表透過 Merkle 樹組織，只需下載區塊頭即可驗證交易是否存在：

```
          Root (Merkle Root)
         /                \
    Hash(0-1)          Hash(2-3)
    /     \             /     \
  Hash0  Hash1      Hash2    Hash3
   |       |         |        |
  TX0     TX1       TX2      TX3
```

## 工作量證明（Proof of Work）

比特幣使用工作量證明（PoW）作為共識機制。

### 挖礦過程

礦工競爭解決一個數學難題：找到一個 Nonce，使得區塊頭的 SHA-256 雙雜湊小於難度目標：

```python
import hashlib

def check_proof(block_header: bytes, difficulty: int) -> bool:
    """驗證工作量證明"""
    hash_result = hashlib.sha256(hashlib.sha256(block_header).digest()).hexdigest()
    # 轉換為整數，檢查是否小於目標
    target = 2 ** (256 - difficulty)
    return int(hash_result, 16) < target

def mine_block(prev_hash: str, merkle_root: str, difficulty: int) -> int:
    """簡化挖礦：找到有效的 nonce"""
    version = 1
    timestamp = 1700000000
    nonce = 0
    
    while True:
        header = f"{version}{prev_hash}{merkle_root}{timestamp}{difficulty}{nonce}"
        header_bytes = header.encode()
        
        if check_proof(header_bytes, difficulty):
            print(f"Found nonce: {nonce}")
            return nonce
        
        nonce += 1
        if nonce % 1000000 == 0:
            print(f"Checked {nonce} nonces...")
```

### 難度調整

比特幣每 2016 個區塊（約兩週）自動調整難度：

- **目標**：保持平均每區塊 10 分鐘
- **上調**：如果區塊產生太快
- **下調**：如果區塊產生太慢

### 51% 攻擊

若攻擊者控制超過 50% 的算力：
- 可逆已完成交易（雙花）
- 阻止新交易確認
- 壟斷新区块的產生

但攻擊成本極高，且會損害比特幣價值，攻擊者自身受損最大。

## 比特幣交易

### UTXO 模型

比特幣使用未花費交易輸出（Unspent Transaction Output, UTXO）模型：

```
UTXO Set:
┌────────────────────────────┐
│ TXID: abc123               │
│ Output Index: 0            │
│ Value: 0.5 BTC             │
│ Script: Alice can spend    │
└────────────────────────────┘

┌────────────────────────────┐
│ TXID: def456               │
│ Output Index: 0            │
│ Value: 0.3 BTC             │
│ Script: Bob can spend      │
└────────────────────────────┘
```

每筆交易消耗 UTXO，產生新的 UTXO。

### 比特幣腳本

比特幣交易由腳本控制：

```bitcoin
# P2PKH（Pay to Public Key Hash）標準交易
# 鎖定腳本：
OP_DUP OP_HASH160 <public_key_hash> OP_EQUALVERIFY OP_CHECKSIG

# 解鎖腳本：
<signature> <public_key>
```

### 交易費用

交易費用 = 輸入總和 - 輸出總和

費用激勵礦工打包交易，費用率以 sat/vB（每虛擬位元組的 satoshi）計算。

## 錢包與位址

### 階層式確定性錢包（BIP-32）

從單一種子派生所有金鑰：

```
Seed (128/256 bits)
    │
    └── Master Key (私鑰)
          │
          └── Chain Code
                │
                ├── 外部分支（接收地址）
                └── 內部分支（找零地址）
```

助記詞（BIP-39）讓用戶只需備份 12-24 個單詞：

```
原始熵 → 校验和 → 助记词
 128 bits  4 bits  12 words
 256 bits  8 bits  24 words
```

### 位址格式

- **P2PKH**（傳統）：`1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa`（比特幣主網）
- **P2SH**（Pay to Script Hash）：`3J98t1WpEZ73CNmQviecrnyiWrnqRhWNLy`
- **Bech32**（原生隔離見證）：`bc1qw508d6qejxtdg4y5r3zarvary0c5xw7kv8f3t4`

## 以太坊與智慧合約

2015 年，Vitalik Buterin 發布以太坊，將區塊鏈從單一貨幣擴展為智慧合約平臺。

### 以太坊虛擬機器（EVM）

EVM 是圖靈完整的虛擬機，可在區塊鏈上執行任意程式：

```
┌─────────────────────────────────────────┐
│           以太坊區塊                      │
├─────────────────────────────────────────┤
│ Header                                  │
│ ├─ difficulty: 0x1234                   │
│ ├─ gasLimit: 3000000                    │
│ └─ ...                                 │
├─────────────────────────────────────────┤
│ Transactions                            │
│ └─ deploy contract: 0x606060...         │
├─────────────────────────────────────────┤
│ Uncle Blocks (Ommer)                    │
└─────────────────────────────────────────┘
```

### Solidity 智慧合約

```solidity
// Solidity 簡單合約
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 private storedData;
    address public owner;
    
    event Stored(uint256 value, address indexed setter);
    
    constructor() {
        owner = msg.sender;
    }
    
    function set(uint256 x) public {
        require(msg.sender == owner, "Only owner");
        storedData = x;
        emit Stored(x, msg.sender);
    }
    
    function get() public view returns (uint256) {
        return storedData;
    }
}
```

### 帳戶模型 vs UTXO

以太坊使用帳戶模型（類似銀行帳戶）：

```
帳戶 State:

│ Nonce: 5  (交易計數)                   │
│ Balance: 1.5 ETH                      │
│ CodeHash: 0xabcd... (合約)            │
│ StorageRoot: 0xdefg...               │
└──────────────────────────────────────┘
```

## 區塊鏈的挑戰

### 可擴展性

區塊鏈的「不可能三角」：
- **去中心化**：網路節點數量
- **安全性**：對抗攻擊的能力
- **效能**：交易吞吐量

比特幣 TPS 約 7，以太坊約 15-30，遠低於 Visa 的數千筆/秒。

### Layer 2 解決方案

- **比特幣**：Lightning Network（支付通道）
- **以太坊**：Rollups（ZK-Rollup、Optimistic Rollup）

### 能源消耗

比特幣 PoW 消耗大量電力。環保倡議推動轉向 PoS（共識），以太坊已於 2022 年完成 Merge。

### 監管與合規

各國對加密貨幣的態度不一：
- 有些國家完全禁止
- 有些國家納入現有金融監管
- 央行數位貨幣（CBDC）興起

## 共識機制演進

### 工作量證明（PoW）

優點：
- 安全性經過時間驗證
- 抗審查

缺點：
- 能源消耗大
- 礦池中心化

### 權益證明（PoS）

以太坊 2.0 採用的共識機制：
- 質押 ETH 作為保證金
- 驗證者隨機選擇產生區塊
- 惡意行為者被罰沒質押

### 實用拜占庭容錯（PBFT）

適合聯盟鏈，容忍一定比例的惡意節點。

## 未來展望

### Web3

區塊鏈被視為 Web3（去中心化互聯網）的基礎：
- 用戶擁有數據
- 去中心化身份（DID）
- 抗審查

### 區塊鏈互通性

跨鏈橋解決不同區塊鏈之間的資產轉移問題。

### 機構採用

比特幣現貨 ETF 的批准標誌著主流金融機構開始接受加密貨幣。
