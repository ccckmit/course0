# 有限狀態機 (Finite State Machine, FSM)

**標籤**: #有限狀態機 #FSM #DFA #NFA #正則語言 #自動機理論 #計算理論
**日期**: 2024-01-15

有限狀態機（Finite State Machine，簡稱 FSM），又稱有限狀態自動機（Finite State Automaton，FSA），是計算理論中最基本也是最重要的計算模型之一。它用於描述一個系統在有限數量的狀態之間進行轉換的行為，根據輸入字元從一個狀態轉移到另一個狀態。有限狀態機在電腦科學的許多領域都有廣泛應用，包括編譯器的詞彙分析、協定設計、數位電路設計、遊戲 AI 等。

## 有限狀態機的發展歷程

### 理論起源 (1950s)

有限狀態機的概念起源於 1950 年代，當時主要應用於類神經網路和開關電路的研究。Mealy 和 Moore 分別在 1955 年和 1956 年提出了兩種著名的有限狀態機模型，至今仍是數位電路設計的基礎。

### 在編譯器中的應用 (1960s-1970s)

Ken Thompson 在 1968 年開發的 Unix 編輯器 ed 和隨後的 grep 工具，首次將有限狀態機與正則表達式結合，用於模式匹配。這一創新後來被應用於 lex 等詞彙分析器生成工具。

### 現代應用 (1980s-現在)

有限狀態機在軟體工程中持續發揮重要作用，特別是在協定實現、遊戲開發、和工作流程引擎等領域。

## 有限狀態機的形式定義

有限狀態機是一個五元組 M = (Q, Σ, δ, q₀, F)，各元件的意義如下：

```
Q  - 有限的非空狀態集合 (States)
Σ  - 有限的輸入字母表 (Alphabet)
δ  - 轉移函數 (Transition Function): Q × Σ → Q
q₀ - 起始狀態 (Initial State), q₀ ∈ Q
F  - 接受狀態集合 (Final/Accepting States), F ⊆ Q
```

### 轉移函數的類型

根據轉移函數的不同特性，有限狀態機可分為：

1. **確定型有限狀態機 (DFA)**：轉移函數 δ(q, a) 至多返回一個狀態
2. **非確定型有限狀態機 (NFA)**：轉移函數 δ(q, a) 可以返回多個狀態，甚至空轉移 (ε-轉移)

## DFA vs NFA

### 確定型有限狀態機 (DFA)

在 DFA 中，每個狀態對每個輸入符號只有唯一的下一個狀態。這意味著對於相同的輸入，總是產生相同的轉移路徑。

**DFA 的特點**：
- 每個狀態對每個輸入符號只有一個轉移
- 沒有 ε-轉移
- 任意時刻只有一個活躍狀態
- 空間需求較小

**DFA 形式化定義**：

```
DFA = (Q, Σ, δ, q₀, F)

其中：
- Q = {q0, q1, q2, ...}  # 狀態集合
- Σ = {a, b, c, ...}      # 輸入字母表
- δ : Q × Σ → Q           # 轉移函數
- q0 ∈ Q                  # 起始狀態
- F ⊆ Q                  # 接受狀態集合
```

### 非確定型有限狀態機 (NFA)

在 NFA 中，一個狀態對同一個輸入符號可能有多個可能的轉移，也可能沒有轉移。此外，NFA 允許 ε-轉移，即不需要消耗輸入字元就可以進行的轉移。

**NFA 的特點**：
- 一個狀態對同一個輸入可能有多个下一狀態
- 允許 ε-轉移（空字元轉移）
- 可能需要指數級的空間來模擬
- 設計相對直觀

**NFA 形式化定義**：

```
NFA = (Q, Σ, δ, q₀, F)

其中：
- Q = {q0, q1, q2, ...}         # 狀態集合
- Σ = {a, b, c, ...}             # 輸入字母表
- δ : Q × Σ ∪ {ε} → P(Q)         # 轉移函數（返回狀態的集合）
- q0 ∈ Q                         # 起始狀態
- F ⊆ Q                          # 接受狀態集合
```

### DFA 與 NFA 的比較

| 特性 | DFA | NFA |
|------|-----|-----|
| 轉移確定性 | 每個狀態每個輸入只有一個轉移 | 可能有多個轉移 |
| ε-轉移 | 無 | 有 |
| 記憶體需求 | O(1) | 可能需指數空間 |
| 設計難度 | 較高 | 較低 |
| 等價性 | 與 NFA 等價 | 與 DFA 等價 |
| 接受語言 | 正則語言 | 正則語言 |

### 等價性證明

DFA 和 NFA 接受的是同一類語言——正則語言。這個結論基於以下定理：

**定理**：對於任意 NFA，存在一個等價的 DFA 接受相同的語言。

**證明思路**（子集構造法）：
1. DFA 的每個狀態對應 NFA 狀態集合的一個子集
2. DFA 的起始狀態是包含 NFA 起始狀態的集合的 ε-閉包
3. DFA 的轉移：對於狀態集合 S 和輸入 a，轉移到所有從 S 中的狀態經過 a 可達的狀態的 ε-閉包
4. DFA 的接受狀態是包含 NFA 接受狀態的任何狀態

## 有限狀態機的 Python 實現

### DFA 實現

```python
class DFA:
    """確定型有限狀態機"""
    
    def __init__(self, states, alphabet, transition_function, start_state, accept_states):
        self.states = states
        self.alphabet = alphabet
        self.transition = transition_function
        self.start_state = start_state
        self.accept_states = accept_states
    
    def accept(self, input_string):
        """檢查輸入字串是否被接受"""
        state = self.start_state
        
        for symbol in input_string:
            if symbol not in self.alphabet:
                return False
            
            key = (state, symbol)
            if key not in self.transition:
                return False
            
            state = self.transition[key]
        
        return state in self.accept_states
    
    def reject(self, input_string):
        """檢查輸入字串是否被拒絕"""
        return not self.accept(input_string)
    
    def run(self, input_string, verbose=False):
        """執行並可選地輸出詳細資訊"""
        state = self.start_state
        steps = [(state, '')]
        
        for symbol in input_string:
            key = (state, symbol)
            if key not in self.transition:
                if verbose:
                    print(f"在狀態 {state} 遇到 '{symbol}' 無有效轉移")
                return False
            
            state = self.transition[key]
            steps.append((state, symbol))
        
        if verbose:
            print("執行步驟：")
            for i, (s, sym) in enumerate(steps):
                if i == 0:
                    print(f"  起始: {s}")
                else:
                    print(f"  讀取 '{sym}' -> {s}")
            
            if state in self.accept_states:
                print(f"  最終狀態: {state} (接受)")
            else:
                print(f"  最終狀態: {state} (拒絕)")
        
        return state in self.accept_states
```

### NFA 實現

```python
class NFA:
    """非確定型有限狀態機"""
    
    def __init__(self, states, alphabet, transition_function, start_state, accept_states):
        self.states = states
        self.alphabet = alphabet
        self.transition = transition_function  # δ : (state, symbol) -> set of states
        self.start_state = start_state
        self.accept_states = accept_states
    
    def epsilon_closure(self, states):
        """計算 ε-閉包：從給定狀態集合出發，經過任意數量 ε-轉移能到達的所有狀態"""
        closure = set(states)
        stack = list(states)
        
        while stack:
            state = stack.pop()
            epsilon_key = (state, 'ε')
            
            if epsilon_key in self.transition:
                for next_state in self.transition[epsilon_key]:
                    if next_state not in closure:
                        closure.add(next_state)
                        stack.append(next_state)
        
        return closure
    
    def move(self, states, symbol):
        """計算從給定狀態集合讀取符號後能到達的狀態集合"""
        result = set()
        
        for state in states:
            key = (state, symbol)
            if key in self.transition:
                result.update(self.transition[key])
        
        return result
    
    def accept(self, input_string):
        """檢查輸入字串是否被接受"""
        current_states = self.epsilon_closure({self.start_state})
        
        for symbol in input_string:
            if symbol not in self.alphabet:
                return False
            
            # 計算讀取 symbol 後的狀態集合
            next_states = self.move(current_states, symbol)
            # 計算 ε-閉包
            current_states = self.epsilon_closure(next_states)
        
        # 如果任何當前狀態是接受狀態，則接受
        return bool(current_states & self.accept_states)
    
    def to_dfa(self):
        """將 NFA 轉換為等價的 DFA"""
        # DFA 的每個狀態是 NFA 狀態的集合
        dfa_states = set()
        dfa_transition = {}
        dfa_accept_states = set()
        
        # 起始狀態：起始狀態的 ε-閉包
        start_closure = self.epsilon_closure({self.start_state})
        dfa_states.add(frozenset(start_closure))
        
        # 如果起始狀態集合包含接受狀態，則為接受狀態
        if start_closure & self.accept_states:
            dfa_accept_states.add(frozenset(start_closure))
        
        # 使用 BFS 探索所有 DFA 狀態
        queue = [frozenset(start_closure)]
        
        while queue:
            current_set = queue.pop(0)
            current_frozenset = frozenset(current_set)
            
            for symbol in self.alphabet:
                next_set = self.epsilon_closure(self.move(current_set, symbol))
                next_frozenset = frozenset(next_set)
                
                dfa_transition[(current_frozenset, symbol)] = next_frozenset
                
                if next_frozenset not in dfa_states:
                    dfa_states.add(next_frozenset)
                    
                    if next_set & self.accept_states:
                        dfa_accept_states.add(next_frozenset)
                    
                    queue.append(next_frozenset)
        
        return DFA(
            states=dfa_states,
            alphabet=self.alphabet,
            transition_function=dfa_transition,
            start_state=frozenset(start_closure),
            accept_states=dfa_accept_states
        )
```

### 範例 1：接受字串 "ab" 的語言

```python
# 設計：接受包含子字串 "ab" 的所有字串
# 狀態：
#   q0: 起始狀態，還沒看到 "ab"
#   q1: 剛讀到 'a'
#   q2: 已讀到 "ab"（接受狀態）

dfa = DFA(
    states={'q0', 'q1', 'q2'},
    alphabet={'a', 'b'},
    transition_function={
        ('q0', 'a'): 'q1',
        ('q0', 'b'): 'q0',
        ('q1', 'a'): 'q1',
        ('q1', 'b'): 'q2',
        ('q2', 'a'): 'q2',
        ('q2', 'b'): 'q2',
    },
    start_state='q0',
    accept_states={'q2'}
)

# 測試
test_cases = [
    ("ab", True),
    ("aab", True),
    ("bba", False),
    ("aba", True),
    ("ba", False),
    ("", False),
]

for input_str, expected in test_cases:
    result = dfa.accept(input_str)
    status = "✓" if result == expected else "✗"
    print(f"{status} '{input_str}': {result}")
```

## 有限狀態機的極小化

### 等價狀態

兩個狀態稱為等價，如果：
1. 從其中一個狀態出發，讀取任何字串到達的都是接受/拒絕狀態
2. 從另一個狀態出發也一樣

### Myhill-Nerode 定理

Myhill-Nerode 定理提供了判斷兩個狀態是否等價的框架：

**定義**：對於語言 L，狀態 p 和 q 是不可區分的，如果對於所有字串 w，pw ∈ L ⇔ qw ∈ L。

**定理**：語言 L 是正則的當且僅當 L 的 Myhill-Nerode 等價類的數量是有限的。

### 狀態極小化演算法

```python
def minimize_dfa(dfa):
    """使用 Myhill-Nerode 定理極小化 DFA"""
    
    # 步驟 1：區分接受狀態和非接受狀態
    partitions = [dfa.accept_states, dfa.states - dfa.accept_states]
    
    # 迭代細分直到穩定
    while True:
        new_partitions = []
        
        for partition in partitions:
            # 嘗試細分這個分區
            for symbol in dfa.alphabet:
                # 根據這個符號的轉移目標來分組
                groups = {}
                
                for state in partition:
                    target = dfa.transition.get((state, symbol))
                    key = tuple(frozenset(p) if p in dfa.accept_states 
                               else tuple() for p in partitions)
                    # 簡化的分組邏輯
                    groups.setdefault(target, []).append(state)
                
                if len(groups) > 1:
                    new_partitions.extend(groups.values())
                    break
            else:
                new_partitions.append(partition)
        
        if new_partitions == partitions:
            break
        partitions = new_partitions
    
    return partitions
```

## 有限狀態機的應用

### 1. 詞彙分析器 (Lexer)

詞彙分析器使用正則表達式 → NFA → DFA 的流程來識別 tokens。

```python
import re
from collections import namedtuple

Token = namedtuple('Token', ['type', 'value', 'position'])

# 正則表達式定義
token_definitions = [
    ('NUMBER', r'\d+'),
    ('STRING', r'"[^"]*"'),
    ('IDENT', r'[a-zA-Z_][a-zA-Z0-9_]*'),
    ('PLUS', r'\+'),
    ('MINUS', r'-'),
    ('MULT', r'\*'),
    ('DIV', r'/'),
    ('LPAREN', r'\('),
    ('RPAREN', r'\)'),
    ('WHITESPACE', r'\s+'),
]

def tokenize(code):
    tokens = []
    position = 0
    
    while position < len(code):
        match = None
        
        for token_type, pattern in token_definitions:
            regex = re.compile(pattern)
            match = regex.match(code, position)
            
            if match:
                value = match.group(0)
                
                if token_type != 'WHITESPACE':
                    tokens.append(Token(token_type, value, position))
                
                position = match.end(0)
                break
        
        if not match:
            raise ValueError(f"無法識別字元於位置 {position}: {code[position]}")
    
    return tokens
```

### 2. 協定狀態機

TCP 協定使用複雜的有限狀態機來管理連線狀態。

```python
class TCPStateMachine:
    """TCP 連線狀態機"""
    
    STATES = {
        'CLOSED', 'LISTEN', 'SYN_SENT', 'SYN_RECEIVED',
        'ESTABLISHED', 'FIN_WAIT_1', 'FIN_WAIT_2', 'CLOSING',
        'TIME_WAIT', 'CLOSE_WAIT', 'LAST_ACK'
    }
    
    def __init__(self):
        self.state = 'CLOSED'
        self.transitions = {
            ('CLOSED', 'OPEN'): 'LISTEN',
            ('CLOSED', 'CONNECT'): 'SYN_SENT',
            ('LISTEN', 'SYN'): 'SYN_RECEIVED',
            ('LISTEN', 'CONNECT'): 'SYN_SENT',
            ('SYN_SENT', 'SYN_ACK'): 'SYN_RECEIVED',
            ('SYN_SENT', 'SYN'): 'SYN_RECEIVED',
            ('SYN_RECEIVED', 'ACK'): 'ESTABLISHED',
            ('ESTABLISHED', 'CLOSE'): 'FIN_WAIT_1',
            ('FIN_WAIT_1', 'ACK'): 'FIN_WAIT_2',
            ('FIN_WAIT_2', 'CLOSE'): 'CLOSED',
            ('ESTABLISHED', 'FIN'): 'CLOSE_WAIT',
            ('CLOSE_WAIT', 'CLOSE'): 'LAST_ACK',
            ('LAST_ACK', 'ACK'): 'CLOSED',
        }
    
    def process_event(self, event):
        """處理事件並轉移狀態"""
        key = (self.state, event)
        
        if key in self.transitions:
            self.state = self.transitions[key]
            return True
        
        return False
    
    def can_send(self):
        """檢查是否可以發送資料"""
        return self.state == 'ESTABLISHED'
    
    def can_receive(self):
        """檢查是否可以接收資料"""
        return self.state == 'ESTABLISHED'
```

### 3. 遊戲 AI 狀態機

```python
class EnemyAI:
    """遊戲敵人 AI 狀態機"""
    
    class State:
        IDLE = 'idle'
        PATROL = 'patrol'
        CHASE = 'chase'
        ATTACK = 'attack'
        RETREAT = 'retreat'
        DEAD = 'dead'
    
    def __init__(self):
        self.state = self.State.IDLE
        self.health = 100
        self.player_detected = False
        self.in_attack_range = False
        self.low_health = False
    
    def update(self):
        """更新狀態"""
        if self.state == self.State.IDLE:
            if self.player_detected:
                self.state = self.State.CHASE
            else:
                self.state = self.State.PATROL
        
        elif self.state == self.State.PATROL:
            if self.player_detected:
                self.state = self.State.CHASE
        
        elif self.state == self.State.CHASE:
            if self.in_attack_range:
                self.state = self.State.ATTACK
            elif self.low_health:
                self.state = self.State.RETREAT
            elif not self.player_detected:
                self.state = self.State.IDLE
        
        elif self.state == self.State.ATTACK:
            if not self.in_attack_range:
                self.state = self.State.CHASE
            elif self.low_health:
                self.state = self.State.RETREAT
        
        elif self.state == self.State.RETREAT:
            if self.health > 70:
                self.state = self.State.CHASE
            elif not self.player_detected:
                self.state = self.State.IDLE
    
    def execute(self):
        """執行當前狀態的行為"""
        if self.state == self.State.IDLE:
            return "等待..."
        elif self.state == self.State.PATROL:
            return "巡邏中..."
        elif self.state == self.State.CHASE:
            return "追擊玩家！"
        elif self.state == self.State.ATTACK:
            return "攻擊！"
        elif self.state == self.State.RETREAT:
            return "撤退！"
        elif self.state == self.State.DEAD:
            return "死亡"
```

### 4. 正則表達式引擎

有限狀態機可以實現簡單的正則表達式引擎。

```python
import re

def regex_to_nfa(pattern):
    """將簡單的正則表達式轉換為 NFA"""
    # 這是一個簡化的實現
    # 完整的實現需要 Thompson 的構造演算法
    
    nfa = NFA(
        states=set(),
        alphabet=set(),
        transition_function={},
        start_state=None,
        accept_states=set()
    )
    
    # 處理字元類
    if pattern.isdigit():
        nfa.alphabet.add(pattern)
    elif pattern == '.':
        nfa.alphabet.update('abcdefghijklmnopqrstuvwxyz')
    
    return nfa

def match_with_fsm(pattern, text):
    """使用有限狀態機進行匹配"""
    # 構建 NFA
    nfa = regex_to_nfa(pattern)
    # 轉換為 DFA
    dfa = nfa.to_dfa()
    # 執行匹配
    return dfa.accept(text)
```

## 有限狀態機的限制

有限狀態機雖然強大，但有其無法解決的問題：

### 語言的階層

| 語言類型 | 例子 | FSM 能識別？ |
|----------|------|--------------|
| 正則語言 | a*b* | 是 |
| 無上下文語言 | aⁿbⁿ | 否 |
| 無上下文語言 | 匹配的括號 | 否 |
| 敏感語言 | aⁿbⁿcⁿ | 否 |

### FSM 無法識別的語言

1. **aⁿbⁿ** (n ≥ 1)：需要記憶讀取了多少個 'a'
2. **匹配的括號**：需要棧來追蹤括號深度
3. **質數個 'a'**：需要無限記憶體

這些語言需要圖靈機或更強大的計算模型才能識別。

## 進階主題

### 摩爾機 vs 米利機

**摩爾機 (Moore Machine)**：
- 輸出只與當前狀態有關
- 輸出在狀態轉移時產生
- 適合時序邏輯電路

**米利機 (Mealy Machine)**：
- 輸出與當前狀態和輸入有關
- 輸出在讀取輸入時產生
- 通常比摩爾機需要更少的狀態

### 萬年曆

使用 FSM 實現簡單的日期驗證。

```python
class DateValidator:
    """日期驗證 FSM"""
    
    def __init__(self):
        self.state = 'INIT'
        self.day = 0
        self.month = 0
        self.year = 0
    
    def is_valid_date(self, date_string):
        """驗證日期格式 MM/DD/YYYY"""
        parts = date_string.split('/')
        
        if len(parts) != 3:
            return False
        
        try:
            month = int(parts[0])
            day = int(parts[1])
            year = int(parts[2])
        except ValueError:
            return False
        
        if not (1 <= month <= 12):
            return False
        
        days_in_month = {
            1: 31, 2: 28, 3: 31, 4: 30,
            5: 31, 6: 30, 7: 31, 8: 31,
            9: 30, 10: 31, 11: 30, 12: 31
        }
        
        # 閏年處理
        if (year % 4 == 0 and year % 100 != 0) or (year % 400 == 0):
            days_in_month[2] = 29
        
        max_day = days_in_month[month]
        
        return 1 <= day <= max_day
```

## 總結

有限狀態機是計算理論的基石，它展現了：
1. **簡單性**：只有有限數量的狀態和簡單的轉移規則
2. **強大性**：足以識別所有正則語言
3. **實用性**：在編譯器、協定、遊戲等領域廣泛應用
4. **理論意義**：連接了形式語言、計算理論和實際應用

理解有限狀態機對於深入理解計算機科學的許多核心概念至關重要。
