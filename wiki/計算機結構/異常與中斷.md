# 異常與中斷 (Exceptions and Interrupts)

## 概述

異常（Exception）和中斷（Interrupt）是處理器回應事件機制，允許系統在正常執行流程中暫停，轉而處理緊急或重要的事件。

## 術語定義

| 術語 | 說明 | 範例 |
|------|------|------|
| Exception | 由 CPU 內部產生的異常事件 | 除零、非法指令、系統呼叫 |
| Interrupt | 由外部硬體觸發的事件 | 計時器、鍵盤、網路卡 |
| Trap | 刻意觸發的異常（用於系統呼叫） | syscall, break |
| Fault | 可修復的異常，處理後可恢復執行 | 分頁錯誤 |
| Abort | 不可恢復的異常 | 硬體錯誤 |

## 中斷類型

### 外部中斷 (Hardware Interrupt)
```
鍵盤 ──┐
滑鼠 ──┤
網卡 ──┼──→ 中斷控制器 ──→ CPU
計時器 ─┤           (IRQ)
硬碟 ──┘           (PIC/APIC)
```

### 例外中斷 (Exception)
| 類型 | 說明 | 可恢復 |
|------|------|--------|
| Reset | 系統重置 | 否 |
| Bus Error | 記憶體存取錯誤 | 否 |
| Address Error | 位址對齊錯誤 | 否 |
| Overflow | 算術溢位 | 可 |
| Divide by Zero | 除零錯誤 | 可 |
| Illegal Instruction | 無效指令 | 可 |
| System Call | 系統呼叫 | 可 |

## 中斷處理流程

```
正常執行 → 中斷發生 → 完成當前指令 → 保存現場 → 查詢中斷源
                ↓
         查詢中斷控制器
                ↓
         取得中斷向量號
                ↓
         查詢中斷向量表
                ↓
         跳轉到中斷處理常式
                ↓
         處理完成
                ↓
         恢復現場 → 返回執行
```

## RISC-V 中斷處理

### CSR 暫存器
```assembly
# 中斷相關 CSR
mstatus    # Machine status register
mie        # Machine interrupt enable
mip        # Machine interrupt pending
mtvec      # Machine trap vector
mepc       # Machine exception PC
mcause     # Machine cause register
mtval      # Machine trap value
```

### mtvec 格式
```c
// Direct 模式：所有中斷跳轉到同一地址
// Base 必須 4 位元組對齊
mtvec = base_addr  // 中斷向量地址

// Vectored 模式：根據中斷類型跳轉
// Base 必須 64 位元組對齊
// mtvec = base_addr + cause * 4
mtvec = base_addr | 1  // 最低位為 1 表示 vectored 模式
```

### 中斷處理範例
```c
// 中斷初始化
void interrupt_init() {
    // 設定中斷向量
    volatile uint32_t *mtvec = (uint32_t *)0x300;
    *mtvec = (uint32_t)trap_vector | 1;  // Vectored mode
    
    // 啟用全域中斷
    set_csr(mstatus, MSTATUS_MIE);
    
    // 啟用特定中斷
    set_csr(mie, MIE_MEIE);  // Machine external interrupt
    set_csr(mie, MIE_MTIE);  // Machine timer interrupt
    set_csr(mie, MIE_MSIE);  // Machine software interrupt
}

// 中斷處理常式
void trap_handler() {
    uint32_t cause = read_csr(mcause);
    uint32_t epc = read_csr(mepc);
    
    if (cause & 0x80000000) {
        // 中斷 (Interrupt)
        uint32_t irq = cause & 0x7FFFFFFF;
        switch (irq) {
            case 7:  // Timer interrupt
                handle_timer();
                break;
            case 11: // External interrupt
                handle_external();
                break;
        }
    } else {
        // 例外 (Exception)
        switch (cause) {
            case 0:  // Instruction address misaligned
            case 1:  // Instruction access fault
            case 2:  // Illegal instruction
            case 3:  // Breakpoint
            case 8:  // ECALL from U-mode
            case 11: // ECALL from M-mode
                handle_exception(cause, epc);
                break;
        }
    }
    
    // 更新 EPC（對於中斷，增加 4 位元組）
    if (cause & 0x80000000) {
        write_csr(mepc, epc + 4);
    }
}
```

## 中斷控制器

### 8259 PIC (Programmable Interrupt Controller)
```
         ┌─────────────────────────────────┐
         │            8259 PIC             │
         ├─────────────────────────────────┤
  IRQ0 ─→│                                 │
  IRQ1 ─→│  ┌─────────────────────────┐   │
  ...   →│  │   Interrupt Request     │   │
  IRQ7 ─→│  │   Register (IRR)       │   │
         │  └───────────┬─────────────┘   │
         │              │                  │
         │  ┌───────────▼─────────────┐   │
         │  │   In-Service           │   │
         │  │   Register (ISR)       │──→ INT ──→ CPU
         │  └─────────────────────────┘   │
         │                                 │
         │  ┌─────────────────────────┐   │
         │  │   Interrupt Mask        │   │
         │  │   Register (IMR)        │   │
         │  └─────────────────────────┘   │
         └─────────────────────────────────┘
```

### APIC (Advanced PIC)
```
        ┌──────────────────────────────────────────────┐
        │                  APIC 架構                     │
        ├──────────────────────────────────────────────┤
        │                                              │
        │  ┌─────────────┐      ┌─────────────────┐  │
        │  │   Local     │      │   I/O APIC      │  │
        │  │   APIC      │←─────│   (每個設備)     │  │
        │  │  (每顆 CPU) │      └─────────────────┘  │
        │  ├─────────────┤                            │
        │  │ LVT Table    │                            │
        │  │ - Timer      │                            │
        │  │ - Thermal    │                            │
        │  │ - Perf Mon  │     IRQ0 ──┐               │
        │  │ - Ext Int   │     IRQ1 ──┼──→ I/O APIC  │
        │  ├─────────────┤     ...    │               │
        │  │ Task Prio  │            │               │
        │  ├─────────────┤     IRQ N ──┘               │
        │  │ IRR/ISR    │                            │
        │  └─────────────┘                            │
        │            ↑                                 │
        │            │        APIC Bus                │
        │            └────────────────────────────────│
        │                   (或透過系統匯流排)          │
        └──────────────────────────────────────────────┘
```

## 向量中斷 (Vectored Interrupt)

```c
// 中斷向量表 (x86 範例)
void *interrupt_vector_table[] = {
    [0]  = divide_error,
    [1]  = debug_exception,
    [2]  = nmi_interrupt,
    [3]  = breakpoint,
    ...
    [32] = timer_interrupt,
    [33] = keyboard_interrupt,
    ...
};

// 中斷描述符表 (IDT) 設定
struct idt_entry {
    uint16_t offset_low;
    uint16_t selector;
    uint8_t  ist;
    uint8_t  type_attr;
    uint16_t offset_mid;
    uint32_t offset_high;
    uint32_t zero;
};

void idt_set_gate(int n, uint64_t handler) {
    idt[n].offset_low = handler & 0xFFFF;
    idt[n].offset_mid = (handler >> 16) & 0xFFFF;
    idt[n].offset_high = (handler >> 32) & 0xFFFFFFFF;
    idt[n].selector = KERNEL_CS;
    idt[n].type_attr = GATE_INTERRUPT;
}
```

## 軟體中斷 (Software Interrupt)

### 系統呼叫範例
```c
// RISC-V 系統呼叫
static inline long sys_call(long num, long arg0, long arg1, long arg2) {
    register long a0 __asm__("a0") = arg0;
    register long a1 __asm__("a1") = arg1;
    register long a2 __asm__("a2") = arg2;
    register long a7 __asm__("a7") = num;
    
    __asm__ __volatile__("ecall"
        : "+r"(a0)
        : "r"(a7), "r"(a0), "r"(a1), "r"(a2)
        : "memory");
    
    return a0;
}

// 使用
sys_call(SYS_write, 1, (long)buffer, 13);  // write to stdout
```

## 中斷延遲與即時性

| 類型 | 延遲特性 | 應用場景 |
|------|----------|----------|
| 向量中斷 | 低延遲 | 快速回應需求 |
| 輪詢 | 可預測延遲 | 即時系統 |
| 中斷 + DMA | 最小 CPU 幹預 | 高速資料傳輸 |
