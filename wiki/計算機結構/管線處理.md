# 管線處理

管線處理是現代 CPU 提高效能的核心技術，透過讓多條指令同時處於不同執行階段來提升指令吞吐量。與其等待一條指令完全執行完畢才開始下一條，管線將指令執行分為多個階段，讓不同指令的不同階段可以並行進行。這種技術類似工廠生產線的概念，每個工人（管線階段）專注處理特定任務，大幅提高整體產出效率。從早期的五級管線到現代的十幾級甚至二十級以上的深管線，管線處理一直是 CPU 架構演進的關鍵推手。

## 管線處理基礎

### 基本原理

管線處理將指令執行分為多個獨立的階段：

```
┌─────────────────────────────────────────────────────────────────┐
│                      五級管線                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  時脈:    T1   T2   T3   T4   T5   T6   T7   T8   T9  T10   │
│                                                              │
│  I1:     IF   ID   EX   MEM  WB                              │
│  I2:          IF   ID   EX   MEM  WB                        │
│  I3:              IF   ID   EX   MEM  WB                    │
│  I4:                  IF   ID   EX   MEM  WB                │
│  I5:                      IF   ID   EX   MEM  WB            │
│                                                              │
│  IF: 指令擷取 (Instruction Fetch)                            │
│  ID: 指令解碼 (Instruction Decode)                           │
│  EX: 執行 (Execute)                                          │
│  MEM: 記憶體存取 (Memory Access)                              │
│  WB: 寫回 (Write Back)                                       │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 指令週期詳細

```python
# 各階段詳細任務

# IF (Instruction Fetch)
# - 從記憶體讀取指令
# - 更新程式計數器 (PC)
# - 分支預測

# ID (Instruction Decode)
# - 指令解碼
# - 讀取暫存器
# - 產生控制訊號

# EX (Execute)
# - ALU 執行運算
# - 計算分支目標
# - 計算記憶體位址

# MEM (Memory Access)
# - 載入/儲存指令的資料傳輸
# - 延遲槽處理

# WB (Write Back)
# - 結果寫回暫存器檔案
# - 更新條件碼
```

### 管線效能

```python
# 管線效率計算
# 假設：
# - 非管線 CPU：每條指令 N 個時脈週期
# - 管線 CPU：每個階段 1 個時脈週期
# - 理想狀況：CPI ≈ 1

# 加速比
# Speedup = N / 1 = N
# (N 為管線階段數)

# 限制因素：
# • 啟動/排出時間
# • Hazards 導致停頓
# • 時脈頻率增加
```

## 管線危險

管線執行時會遇到三種主要危險，這些問題需要透過硬體設計或軟體技術來解決：

### 結構危險

當硬體資源不足以同時執行所有指令時發生：

```
┌─────────────────────────────────────────────────────────────────┐
│                      結構危險                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  範例：載入/儲存單元只有一個                                    │
│  在同一時脈週期不能同時執行 LOAD 和 STORE                     │
│                                                              │
│  解決方案：                                                   │
│  • 複製資源 (多重執行單元)                                     │
│  • 延遲執行 (排程)                                            │
│  • 衝突檢測和停頓                                              │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

```verilog
// 結構危險範例
// 如果記憶體只能同時處理一個存取
// LOAD 指令必須等 STORE 完成
```

### 資料危險

當指令之間存在資料依賴時發生：

```python
# RAW (Read After Write) - 真實依賴
# I1: add x1, x2, x3   # x1 = x2 + x3
# I2: add x4, x1, x5   # 讀取 I1 的結果 x1
# 解決：資料前饋、暫停、編譯器排程

# WAR (Write After Read) - 反向依賴
# I1: add x1, x2, x3   # 讀取 x2
# I2: add x2, x4, x5   # 寫入 x2 (原本讀取的)
# 解決：暫存器重新命名

# WAW (Write After Write) - 輸出依賴
# I1: add x1, x2, x3
# I2: add x1, x4, x5   # 寫入同一個暫存器
# 解決：暫存器重新命名
```

### 資料前饋

```verilog
// 資料前饋 (Forwarding) 範例
// I1 計算結果直接傳給 I2

// 硬體連線：
// ALU 輸出 → EX/MEM 暫存器 → ALU 輸入
//       → MEM/WB 暫存器 → ALU 輸入

// 時序：
// Cycle 1: I1 EX
// Cycle 2: I1 MEM, I2 EX (從 I1 前饋)
```

### 控制危險

分支指令導致的危險：

```python
# 分支問題
# 分支結果未知時，後續指令是否應該進入管線？

# 分支代價：
# • 正確預測：0 週期
# • 錯誤預測：需清空管線 (3-10+ 週期)

# 解決方案：
# • 靜態分支預測
# • 動態分支預測 (1-bit, 2-bit, 歷史)
# • 延遲分支 (編譯器處理)
# • 分支目標緩衝 (BTB)
```

```verilog
// 延遲分支
// 分支後的指令一定執行
// I1: bne loop
// I2: (延遲槽) add x1, x2, x3  # 一定執行
// I3: sub x4, x5, x6           # 分支目標
```

## 管線實現

### 五級管線 CPU

```verilog
// 簡化的五級管線 CPU
module pipeline_cpu (
    input  wire        clk,
    input  wire        rst_n,
    input  wire [31:0] instruction,
    output wire [31:0] result
);

    // IF 階段
    reg [31:0] pc;
    reg [31:0] if_id;
    
    // ID 階段
    reg [31:0] id_ex;
    
    // EX 階段
    reg [31:0] ex_mem;
    
    // MEM 階段
    reg [31:0] mem_wb;
    
    // IF
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            pc <= 0;
        else
            pc <= pc + 4;
    end
    
    // 簡易管線暫存器傳遞
    // 實際需要更複雜的控制邏輯
    
endmodule
```

## 亂序執行

亂序執行提高硬體利用率：

```python
# 亂序執行 (Out-of-Order Execution)
# • 指令按序抵達
# • 準備好就執行
# • 按序寫回

# 硬體元件：
# • 預取/解碼單元
# • 暫存器重新命名
# • 保留站 (Reservation Station)

```

## 分支預測

### 靜態預測

```python
# 靜態分支預測
# • 向後分支 (loop): 預測執行

```

### 動態預測

```python

```
