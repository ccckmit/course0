# 快取

快取是位於 CPU 與主記憶體之間的高速記憶體，用於減少 CPU 存取記憶體的延遲。現代 CPU 的運算速度遠快於記憶體存取速度，如果每次存取都需要等待記憶體回應，CPU 將花費大量時間在等待上，嚴重影響效能。快取利用了程式的局部性原理（Locality），包括時間局部性和空間局部性，將最近使用和即將使用的資料儲存在高速記憶體中，大幅提升資料存取效率。快取是現代 CPU 架構中不可或缺的組成部分。

## 快取層次結構

現代 CPU 通常採用多層快取結構，越接近 CPU 的層級速度越快、容量越小：

```
┌─────────────────────────────────────────────────────────────────┐
│                      CPU 快取層次                              │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  CPU Core                                                    │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │  L1 I-Cache (指令)    ~32KB    1-2 周期                   │  │
│  │  L1 D-Cache (資料)    ~32KB    1-2 周期                   │  │
│  ├─────────────────────────────────────────────────────────┤  │
│  │  L2 Cache (指令+資料)  ~256KB   10-20 周期                │  │
│  ├─────────────────────────────────────────────────────────┤  │
│  │  L3 Cache (共享)      ~8MB     30-50 周期                 │  │
│  └─────────────────────────────────────────────────────────┘  │
│                         ↓                                      │
│                    主記憶體                                    │
│                    DDR4/DDR5  100+ 周期                       │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 各層特性

| 層次 | 大小 | 延遲 | 關聯性 | 共享方式 |
|------|------|------|--------|----------|
| L1 | 32-64KB | 1-2 周期 | 高 (8-way) | 核心專用 |
| L2 | 256KB-1MB | 10-20 周期 | 中 (8-way) | 核心專用 |
| L3 | 4-32MB | 30-50 周期 | 低 (16-way) | 多核共享 |

## 快取映射方式

### 直接映射

每個記憶體區塊只能放在一個固定的快取槽：

```python
# 直接映射
# 快取槽 = 記憶體位址 % 快取槽數

# 範例：
# 快取大小：8 槽
# 記憶體位址：0, 8, 16, 24 -> 槽 0
# 記憶體位址：1, 9, 17, 25 -> 槽 1

# 優點：簡單、快速
# 缺點：衝突頻繁
```

### 完全關聯

任何記憶體區塊可以放在任何快取槽：

```python
# 完全關聯
# 需要比較所有槽的標籤

# 範例：
# 快取槽 0: 標籤 0x1000, 資料
# 快取槽 1: 標籤 0x2345, 資料
# 快取槽 7: 標籤 0xABCD, 資料

# 優點：彈性高、命中率最高
# 缺點：硬體複雜、延遲高
```

### 集合關聯

結合直接映射和完全關聯的優點：

```python
# 集合關聯 (N-way)
# 快取分為多個集合，每個集合有多個槽

# 範例：4-way 集合關聯
# 集合數 = 快取槽數 / 4
# 記憶體位址 % 集合數 -> 集合
# 標籤比對 -> 槽

# 優點：平衡效能與複雜度
# 缺點：需要 N 個比較器
```

## 快取行結構

快取行（Cache Line）是快取的基本單位：

```
┌─────────────────────────────────────────────────────────────────┐
│                      快取行結構                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  Tag (標籤)  │ Index │ Offset │ Data                         │
│  ───────────┼───────┼────────┼──────────────                 │
│  高位址位元  │ 集合  │ 位元組  │ 64-128 位元組                 │
│              │       │        │  (通常 64 位元組)             │
│                                                              │
│  欄位說明：                                                 │
│  • Tag: 識別記憶體區塊                                        │
│  • Index: 選擇集合                                           │
│  • Offset: 選擇位元組                                        │
│  • Valid bit: 是否有效                                       │
│  • Dirty bit: 是否修改                                       │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 寫入策略

### Write Through

每次寫入同時更新快取和主記憶體：

```python
# Write Through 流程
def write(address, data):
    cache.write(address, data)
    memory.write(address, data)  # 同時寫入記憶體
```

```
優點：
• 一致性簡單
• 可靠（資料不會丟失）

缺點：
• 寫入延遲高
• 頻寬消耗大
```

### Write Back

只寫入快取，直到該行被驅逐才寫回記憶體：

```python
# Write Back 流程
def write(address, data):
    if cache.contains(address):
        cache.write(address, data)
        set_dirty(address)
    else:
        # 先載入再修改
        load_to_cache(address)
        cache.write(address, data)
        set_dirty(address)
```

```
優點：
• 寫入速度快
• 減少記憶體頻寬

缺點：
• 一致性複雜
• 資料可能丟失
```

## 置換策略

當快取已滿，需要選擇移除哪個快取行：

### LRU (Least Recently Used)

最近最少使用的快取行被移除：

```python
# 近似 LRU (Pseud-LRU)
# 用樹狀結構追蹤使用順序

# 每次存取更新樹節點
# 驅逐時選擇最近的葉節點
```

### FIFO (First In First Out)

最先進入的快取行被移除：

```python
# 實現簡單
queue = []

def evict():
    line = queue.pop(0)
    return line
```

### Random

隨機選擇快取行移除：

```python
# 最簡單的實現
def evict():
    return random.choice(cache_lines)
```

## 快取一致性

多核系統中，保持各核心快取一致性的問題：

### MESI 協定

```python
# MESI 狀態
# Modified (修改): 已修改，獨有
# Exclusive (獨有): 未修改，獨有
# Shared (共享): 未修改，多個副本
# Invalid (無效): 無效

# 狀態轉換
# Read Miss -> M/E
# Read Hit -> S
# Write Hit -> M
# 收到 BusRd -> S/I
# 收到 BusRdX -> I
```

### 快取一致性問題

```python
# 問題範例
# Core 1: read a -> 0
# Core 2: read a -> 0
# Core 1: write a = 1  # 需要通知 Core 2
# Core 2: read a -> ?
```

## 快取效能優化

### 快取命中率優化

```python
# 優化技巧：
# 1. 資料對齊
#    - 結構體對齊到快取行邊界
#    - 避免跨快取行的結構
# 2. 迴圈最佳化
#    - 改變迴圈順序 (row vs column)
#    - 合併迴圈減少存取
# 3. 資料配置
#    - 優先使用本核 L1
#    - 避免 false sharing
```

### False Sharing

```python
# False sharing 範例
# Thread 1: var.x = 1
# Thread 2: var.y = 2
# 如果 x, y 在同一快取行
# -> 每次寫入都觸發快取一致性流量

# 解決：
# - 填充對齊
# - 使用 thread-local 緩衝
```

## 快取效能分析

### 快取未命中率類型

| 類型 | 原因 | 解決方式 |
|------|------|----------|
| Compulsory | 第一次存取 | 預取 |
| Capacity | 快取太小 | 增加容量 |
| Conflict | 映射衝突 | 增加關聯性 |
| Coherence | 一致性無效 | 減少共享 |

### 效能指標

```python
# 平均存取時間
# AMAT = Hit Time + Miss Rate × Miss Penalty

# 範例：
# L1: Hit Time = 1, Miss Rate = 5%
# L2: Miss Penalty = 10
# AMAT = 1 + 0.05 × 10 = 1.5 周期
```
