# 暫存器

暫存器是 CPU 內部的高速儲存單元，用於暫時儲存資料、指令和控制資訊。暫存器是電腦系統中存取速度最快的記憶體，因為它們位於 CPU 晶片內部，直接與 ALU 和控制單元相連。相比主記憶體，暫存器的存取時間幾乎為零，這使得暫存器成為影響 CPU 效能的關鍵因素。現代 CPU 通常配備數十到數百個暫存器，每個暫存器可以儲存一個或多個機器字（通常是 32 或 64 位元）。暫存器的設計和使用方式直接影響指令集架構的效率和程式設計的便利性。

## 暫存器的基本概念

### 暫存器的特性

```
┌─────────────────────────────────────────────────────────────────┐
│                      暫存器特性                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  • 位置：CPU 內部                                             │
│  • 速度：最快（零存取延遲）                                   │
│  • 大小：通常 32/64 位元                                      │
│  • 數量：數十到數百個                                          │
│  • 用途：資料儲存、位址儲存、控制資訊                          │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 暫存器 vs 快取 vs 主記憶體

| 層級 | 位置 | 大小 | 延遲 | 頻寬 |
|------|------|------|------|------|
| 暫存器 | CPU 核心 | 數 KB | 0 周期 | 最高 |
| L1 快取 | CPU 核心 | ~32KB | 1-4 周期 | 高 |
| L2 快取 | CPU 封裝 | ~256KB | 10-20 周期 | 中 |
| L3 快取 | CPU 封裝 | ~8MB | 30-50 周期 | 中 |
| 主記憶體 | 主機板 | GB | 100+ 周期 | 低 |

## 通用暫存器

通用暫存器用於暫時存放資料和位址，可以執行各種運算：

### RISC-V 暫存器

```assembly
# RISC-V 通用暫存器 (64 位元)
# x0 - 永遠為 0 (zero)
# x1 - ra (返回位址)
# x2 - sp (堆疊指標)
# x3 - gp (全域指標)
# x4 - tp (執行緒指標)
# x5-x7 - t0-t2 (臨時暫存器)
# x8-x9 - s0-s1 (保存暫存器)
# x10-x17 - a0-a7 (函式引數/返回值)
# x18-x27 - s2-s11 (保存暫存器)
# x28-x31 - t3-t6 (臨時暫存器)

# 常用別名
# a0-a7: 函式引數 (x10-x17)
# t0-t6: 臨時暫存器 (x5-x7, x28-x31)
# s0-s11: 保存暫存器 (x8-x9, x18-x27)
# sp: 堆疊指標 (x2)
# ra: 返回位址 (x1)
# zero: 永遠為 0 (x0)
```

### x86 暫存器

```assembly
# x86-64 通用暫存器
# RAX - 累積器 (返回值、運算)
# RBX - 基址暫存器
# RCX - 計數器 (迴圈計數)
# RDX - 資料暫存器 (I/O、乘法/除法)
# RSI - 來源索引
# RDI - 目的索引
# RBP - 基底指標 (堆疊框架)
# RSP - 堆疊指標
# R8-R15 - 新增暫存器

# 32 位元子集
# EAX, EBX, ECX, EDX, ESI, EDI, EBP, ESP

# 16 位元子集
# AX, BX, CX, DX, SI, DI, BP, SP

# 8 位元子集
# AL, AH, BL, BH, CL, CH, DL, DH
# R8B-R15B (新)
```

### 暫存器使用約定

```assembly
# RISC-V 函式呼叫約定
# 引數: a0-a7 (最多 8 個)
# 返回值: a0-a1
# 臨時: t0-t6 (呼叫者保存)
# 保存: s0-s11 (被呼叫者保存)
# 特殊: sp, ra, gp, tp

# x86-64 System V ABI
# 引數: RDI, RSI, RDX, RCX, R8, R9
# 返回值: RAX, RDX
# 臨時: RAX, RCX, RDX, RSI, RDI, R8-R11
# 保存: RBX, RBP, R12-R15
```

## 暫存器檔案

暫存器檔案是 CPU 內部的暫存器集合：

```verilog
// 暫存器檔案結構
module RegisterFile (
    input clk,
    input rst,
    input [4:0] read_addr1,  // 讀取位址 1
    input [4:0] read_addr2,  // 讀取位址 2
    input [4:0] write_addr,  // 寫入位址
    input [63:0] write_data, // 寫入資料
    input write_enable,       // 寫入使能
    output [63:0] read_data1, // 讀取資料 1
    output [63:0] read_data2  // 讀取資料 2
);

    reg [63:0] registers [0:31];
    integer i;
    
    // 初始化
    always @(posedge rst) begin
        for (i = 0; i < 32; i = i + 1)
            registers[i] = 0;
    end
    
    // 讀取 (組合邏輯)
    assign read_data1 = registers[read_addr1];
    assign read_data2 = registers[read_addr2];
    
    // 寫入 (時序邏輯)
    always @(posedge clk) begin
        if (write_enable && write_addr != 0)
            registers[write_addr] <= write_data;
    end
endmodule
```

## 暫存器重命名

現代 CPU 使用暫存器重命名來支援超出暫存器數量的並行指令：

```python
# 暫存器重命名概念
# 問題：依賴關係導致 stall
# 解決：動態重命名

# 範例：
# 指令 1: R1 = R2 + R3
# 指令 2: R4 = R1 + R5
# 指令 3: R1 = R6 - R7  # 與指令 1 競爭 R1

# 重命名後：
# 指令 1: R1_0 = R2 + R3    -> T1 = R2 + R3
# 指令 2: R4 = R1_0 + R5   -> T2 = T1 + R5
# 指令 3: R1_1 = R6 - R7   -> T3 = R6 - R7
# R1_0 和 R1_1 不衝突
```

## 呼叫框架

函式呼叫使用特定暫存器管理堆疊框架：

```assembly
# RISC-V 堆疊框架
# 進入函式
addi sp, sp, -16      # 分配空間
sd ra, 8(sp)          # 保存返回位址
sd s0, 0(sp)          # 保存框架指標

# 建立框架
addi s0, sp, 16       # s0 = 框架指標

# 函式本體
# 使用 a0-a7, t0-t6

# 返回
ld ra, 8(sp)          # 恢復返回位址
ld s0, 0(sp)          # 恢復框架指標
addi sp, sp, 16       # 釋放空間
ret                   # 返回
```
