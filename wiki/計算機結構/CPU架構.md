# CPU架構

CPU（中央處理器，Central Processing Unit）是計算機的核心元件，負責執行指令和處理資料。從最早的單一功能處理器到現代的多核心、高管線、超純量處理器，CPU 架構經歷了數十年的演進。CPU 的設計涉及多個複雜的子系統，包括控制單元、運算單元、暫存器檔案、快取記憶體和時脈管理，這些元件共同協作實現高效的計算。理解 CPU 架構是掌握電腦系統運作原理的關鍵。

## CPU 基本結構

CPU 的硬體架構由多個功能模組組成，每個模組負責特定任務：

```

│  │  • 指令擷取        • 指令解碼                            │  │

│  └─────────────────────────────────────────────────────────┘  │
```

### 控制單元

控制單元是 CPU 的大腦，負責協調各部件運作：

```python
# 控制單元功能
# 1. 指令擷取 (Fetch)
#    - 從記憶體讀取下一條指令
#    - 更新程式計數器

# 2. 指令解碼 (Decode)
#    - 解釋指令操作碼
#    - 讀取暫存器操作數

# 3. 控制訊號產生
#    - 產生控制 ALU、暫存器、匯流排的訊號
#    - 決定資料流向

# 4. 分支處理
#    - 計算分支目標
#    - 處理分支預測失敗
```

### 運算單元

ALU（Arithmetic Logic Unit）執行所有算術和邏輯運算：

```verilog
// ALU 基本結構
module alu (
    input  wire [31:0] a, b,
    input  wire [3:0]  op,
    output reg  [31:0] result,
    output wire        zero, negative, carry, overflow
);

    always @(*) begin
        case (op)
            4'b0000: result = a + b;        // 加法
            4'b0001: result = a - b;        // 減法
            4'b0010: result = a & b;        // AND
            4'b0011: result = a | b;        // OR
            4'b0100: result = a ^ b;        // XOR
            4'b0101: result = a << b[4:0];  // 邏輯左移
            4'b0110: result = a >> b[4:0];  // 邏輯右移
            4'b0111: result = $signed(a) >>> b[4:0];  // 算術右移
            default: result = 0;
        endcase
    end

    assign zero = (result == 0);
    assign negative = result[31];
    // 進位和溢位需要更複雜的邏輯

endmodule
```

### 暫存器檔案

暫存器是 CPU 內部最快的記憶體：

```verilog
// 32 個 32 位元暫存器的檔案
module register_file (
    input  wire        clk,
    input  wire        rst,
    input  wire [4:0] read_addr1,
    input  wire [4:0] read_addr2,
    input  wire [4:0] write_addr,
    input  wire [31:0] write_data,
    input  wire        write_enable,
    output wire [31:0] read_data1,
    output wire [31:0] read_data2
);

    reg [31:0] registers [0:31];
    integer i;

    // x0 永遠為 0
    always @(*) registers[0] = 32'b0;

    // 寫入
    always @(posedge clk) begin
        if (write_enable && write_addr != 0)
            registers[write_addr] <= write_data;
    end

    // 讀取 (組合邏輯)
    assign read_data1 = registers[read_addr1];
    assign read_data2 = registers[read_addr2];

endmodule
```

## 指令週期

指令週期描述 CPU 執行一條指令的完整過程：

```
┌─────────────────────────────────────────────────────────────────┐
│                      指令週期                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 擷取 (Fetch)                                             │
│     PC → MAR → 記憶體 → IR                                    │
│     PC + 4 → PC                                              │
│                                                              │
│  2. 解碼 (Decode)                                             │
│     IR → 控制單元 → 解讀操作碼                                │
│     讀取暫存器                                                 │
│                                                              │
│  3. 執行 (Execute)                                            │
│     ALU 執行運算                                              │
│     計算分支目標                                              │
│                                                              │
│  4. 記憶體存取 (Memory Access)                                │
│     載入/儲存指令的資料傳輸                                    │
│                                                              │
│  5. 寫回 (Write Back)                                         │
│     結果寫回暫存器                                             │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 微操作

每個階段由多個微操作組成：

```python
# 範例：LOAD R1, (R2)
# 擷取階段:
#   MAR = PC
#   PC = PC + 4
#   MDR = Memory[MAR]
#   IR = MDR

# 解碼階段:
#   A = R1
#   B = R2

# 執行階段:
#   MAR = A + B

# 記憶體存取:
#   MDR = Memory[MAR]

# 寫回階段:
#   R1 = MDR
```

## 指令集架構

指令集架構（ISA）定義 CPU 支援的指令和行為：

### CISC vs RISC

```
┌─────────────────────────────────────────────────────────────────┐
│                      CISC vs RISC                              │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  CISC (Complex Instruction Set Computer)                      │
│  • 指令數量多 (數百條)                                        │
│  • 指令長度可變                                              │
│  • 記憶體直接運算                                            │
│  • 微碼實現                                                  │
│  • 例：x86, x64                                               │
│                                                              │
│  RISC (Reduced Instruction Set Computer)                      │
│  • 指令數量少 (< 100 條)                                      │
│  • 指令長度固定 (32/64 位元)                                  │
│  • 暫存器間運算                                              │
│  • 硬連線實現                                                 │
│  • 例：RISC-V, ARM, MIPS                                      │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 常見指令類型

```python
# 資料傳輸
# LOAD, STORE, MOVE

# 算術
# ADD, SUB, MUL, DIV, MOD, INC, DEC

# 邏輯
# AND, OR, NOT, XOR

# 移位
# SHL, SHR, SAR, ROL, ROR

# 控制轉移
# JMP, CALL, RET, BRANCH, JUMP

# 比較
# CMP, TEST
```

### 指令格式

```python
# RISC-V 指令格式

# R-type (Register)
# 31        25  24  20  19  15  14  12  11  7   6   0
# | funct7    | rs2  | rs1  | funct3 | rd   | opcode |
# 例: add x1, x2, x3

# I-type (Immediate)
# 31         20  19  15  14  12  11  7   6   0
# | imm[11:0] | rs1 | funct3 | rd  | opcode |
# 例: addi x1, x2, 10

# S-type (Store)
# 31    25  24  20  19  15  14  12  11  7   6   0
# | imm[11:5] | rs2 | rs1 | funct3 | imm[4:0] | opcode |

# U-type (Upper immediate)
# 31              12  11  7   6   0
# | imm[31:12]      | rd  | opcode |
```

## CPU 設計技術

### 管線處理

管線將指令執行分為多個階段，實現指令重疊執行：

```python
# 五級管線
# IF -> ID -> EX -> MEM -> WB

# 管線優點
# • 提高指令吞吐量
# • 讓每個時脈週期都有指令完成

# 挑戰
# • 資料 hazard
# • 控制 hazard
# • 結構 hazard
```

### 超純量

超純量架構同時執行多條指令：

```python
# 超純量特點
# • 多個執行單元
# • 指令調度
# • 亂序執行
# • 結果重排序
```

### 多核心

現代 CPU 包含多個核心：

```python
# 多核心設計
# • 每個核心獨立的 ALU、暫存器、快取
# • 共享 L3 快取
# • 核間互連 (Ring, Mesh)
# • 一致性協定 (MESI)
```

### 分支預測

分支預測減少控制 hazard 的影響：

```python
# 分支預測技術
# • 靜態預測：向後跳預測執行，向前跳預測不執行
# • 動態預測：
#   - 1 位元歷史
#   - 2 位元飽和計數器
#   - 局部歷史表
#   - 全域歷史表
#   - 路徑歷史
```

## CPU 效能指標

```python
# 效能指標
# • 時脈頻率 (GHz)

# • IPC (每週期指令數)
# • FLOPS (浮點運算每秒)

# 計算公式
# 效能 = 時脈頻率 × IPC × 核心數
```
