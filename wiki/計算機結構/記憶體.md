# 記憶體

記憶體是計算機用於儲存資料與指令的硬體元件，是電腦系統中最重要的組成部分之一。從 CPU 內部的暫存器到硬碟、光碟等輔助儲存裝置，每個層級的記憶體都有不同的速度和容量特性，共同構成了一個層次化的儲存金字塔。理解記憶體的工作原理對於優化程式效能、開發高效能應用程式至關重要。

## 記憶體層次結構

現代電腦採用層次化的記憶體架構，在速度、容量和成本之間取得平衡：

```
┌─────────────────────────────────────────────────────────────────┐
│                      記憶體金字塔                               │
├─────────────────────────────────────────────────────────────────┤
│                                                              │
│  速度                     容量              成本/位元組        │
│   ↑                       │                     ↑             │
│   │    暫存器 (Registers)  │  ~1 KB              │             │
│   │    CPU 核心內          │  0 周期              │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    L1 快取             │  ~32 KB             │             │
│   │    核心專用            │  1-2 周期            │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    L2 快取             │  ~256 KB            │             │
│   │    核心共享            │  10-20 周期          │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    L3 快取             │  ~8 MB              │             │
│   │    多核共享            │  30-50 周期          │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    主記憶體 (DRAM)      │  GB 級              │             │
│   │    DDR4/DDR5          │  100+ 周期           │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    SSD                │  TB 級              │             │
│   │    NVMe/SATA         │  微秒級              │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    硬碟 (HDD)         │  TB 級              │             │
│   │    機械               │  毫秒級              │             │
│   ├───────────────────────┼─────────────────────┤             │
│   │    磁帶/雲端           │  PB 級              │             │
│   │                       │  分鐘級              │             │
│   └───────────────────────┴─────────────────────┘             │
│                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 局部性原理

程式展現兩種局部性，使得層次化記憶體有效：

```
局部性原理：
• 時間局部性：最近存取的資料很可能再次存取
  - 迴圈中的變數
  - 函式堆疊
  
• 空間局部性：存取某個位置後，很可能存取附近的位置
  - 順序執行的指令
  - 陣列元素順序存取
```

## 記憶體類型

### 揮發性記憶體

揮發性記憶體在電源關閉後會失去資料：

#### DRAM (Dynamic Random Access Memory)

```python
# DRAM 特性：
# • 每個位元需要一個電容 + 一個電晶體
# • 需要定期刷新（通常 64ms）
# • 成本低，密度高
# • 用於主記憶體

# DDR (Double Data Rate)
# DDR4: 2133-3200 MT/s
# DDR5: 4800-8400 MT/s

# 技術參數：
# - 預取 (Prefetch): 8n
# - 通道數: 雙通道/四通道
# - 時序: CL, tRCD, tRP, tRAS
```

#### SRAM (Static Random Access Memory)

```python
# SRAM 特性：
# • 每個位元需要 6 個電晶體
# • 不需要刷新
# • 速度快，用於快取
# • 成本高，密度低

# 應用：
# - CPU L1/L2/L3 快取
# - 暫存器檔案
# - 網路路由器
```

### 非揮發性記憶體

非揮發性記憶體在電源關閉後仍保留資料：

#### ROM (Read-Only Memory)

```
類型：
• Mask ROM: 工廠燒錄，不可更改
• PROM: 可程式一次
• EPROM: 可擦除重寫（需要紫外線）
• EEPROM: 電子擦除，擦除速度慢
```

#### Flash Memory

```python
# Flash 特性：
# • 電子擦除區塊
# • NAND Flash: 高密度，適合儲存
# • NOR Flash: 可隨機存取，適合程式碼

# 應用：
# - USB 隨身碟
# - SSD (固態硬碟)
# - 嵌入式系統儲存
# - 記憶卡
```

#### 新興記憶體

```
技術發展：
• 3D XPoint (Intel Optane)
  - 比 DRAM 慢但便宜
  - 比 NAND 快
  - 已停產

• MRAM (磁阻記憶體)
  - 非揮發
  - 高速
  - 成本高

• ReRAM (電阻式記憶體)
• PCM (相變記憶體)
```

## 記憶體組織

### 位址映射

```python
# 記憶體位址空間
# 32 位元系統: 4 GB 位址空間
# 64 位元系統: 非常大的空間

# 範例：4GB 位址空間
# 0x00000000 - 0x3FFFFFFF: 使用者空間
# 0x40000000 - 0xFFFFFFFF: 核心空間
#   0xFFFF0000 - 0xFFFFFFFF: 核心專屬

# 區段劃分：
# .text: 程式碼
# .data: 已初始化全域變數
# .bss: 未初始化全域變數
# .heap: 動態配置
# .stack: 函式堆疊
```

### 記憶體映射

```python
# 記憶體對應 I/O
# MMIO (Memory-Mapped I/O)
# 某些位址對應到周邊設備

# 範例：
# 0xFFFF0000: UART 控制暫存器
# 0xFFFF1000: GPIO 控制
# 0xFF000000: 顯示緩衝區
```

## 記憶體控制器

### DRAM 控制器的功能

```python
# DRAM 控制器職責：
# • 位址解碼
# • 資料傳輸時序控制
# • 刷新管理
# • 錯誤校正 (ECC)
# • 多通道管理

# 時序參數：
# - tCK: 時脈週期
# - CL: 讀取延遲
# - tRCD: RAS 到 CAS 延遲
# - tRP: 行預充電時間
# - tRAS: 作用時間
```

### 記憶體通道

```python
# 多通道記憶體
# • 雙通道：兩個 DDR 通道並行
# • 四通道：更多頻寬
# • 對稱/非對稱模式

# 頻寬計算：
# DDR4-3200: 25.6 GB/s per channel
# 雙通道: 51.2 GB/s
```

## 錯誤處理

### ECC 記憶體

```python
# ECC (Error Correcting Code)
# • 單一位元錯誤自動修正
# • 雙位元錯誤偵測
# • 使用額外位元 (通常 8 位元/64 位元)

# 應用：
# - 伺服器
# - 工作站
# - 關鍵系統

# 漢明碼：
# 4 位元資料 + 3 位元檢查 = 7 位元
# 可偵測 2 位元錯誤，修正 1 位元錯誤
```
