# LLM Wiki 操作手冊

## 核心原則

1. **LLM 負責維護**: 所有 Wiki 頁面由 LLM 編寫和更新
2. **人類負責方向**: 人類提供來源、提出問題、引導分析
3. **持續累積**: 知識隨時間累積，不是每次重新發現
4. **文章長度**: 每個主題要寫詳細，大約 300 行左右。

## 架構

```
_wiki/
├── AGENTS.md              # Schema: 告知 LLM 如何運作
├── index.md               # 全域索引
├── log.md                 # 全域日誌
├── source/                # 原始來源（不可變）
│   └── 系統程式/
│       └── 教材.pdf
├── 系統程式/
│   ├── index.md           # 分類索引
│   ├── log.md             # 分類日誌
│   ├── sources/           # 來源摘要（自動生成）
│   ├── 概念/              # 概念頁面
│   ├── 主題/              # 主題頁面
│   └── 人物/              # 人物頁面
└── 其他分類...
```

## 操作流程

### 1. 攝取 (Ingest)

當人類提供新來源時：

```
1. 將來源放入 source/ 目錄對應分類
2. 告知 LLM: "請攝取這個來源"
3. LLM 執行:
   a. 閱讀來源，理解內容
   b. 與人類討論要點
   c. 在 sources/ 建立摘要頁
   d. 更新 index.md（添加條目+摘要）
   e. 更新現有相關概念/主題頁面（加入交叉引用）
   f. 追加 log.md
```

**範例流程**:
```markdown
1. 人類放入: source/系統程式/教材.pdf
2. LLM 閱讀後與人類討論要點
3. LLM 創建: 系統程式/sources/教材摘要.md
4. LLM 更新: 系統程式/index.md (添加條目)
5. LLM 更新: 系統程式/概念/編譯器.md (加入引用)
6. LLM 更新: 系統程式/log.md
```

### 2. 查詢 (Query)

當人類提問時：

```
1. 向 LLM 提問
2. LLM 搜索 index.md 找到相關頁面
3. LLM 閱讀相關頁面，綜合答案
4. 引用來源（使用 [來源] 格式）
5. 若答案有價值，考慮存回 Wiki 作為新頁面
```

### 3. 整理 (Lint)

定期檢查 Wiki 健康：

```
- 頁面間的矛盾（聲明是否被新來源推翻）
- 過時資訊（標記為 stale）
- 孤立頁面（無入站鏈接）
- 重要概念但無獨立頁面
- 缺失的交叉引用
- 建議新的攝取來源
```

## 文件規範

### index.md 格式

```markdown
# 分類名稱

分類簡短描述。

## 概念
- [標題](./路徑/標題.md) - 一行摘要（必備）
- [標題](./路徑/標題.md) - 一行摘要 | tags: 標籤1, 標籤2

## 主題
- [標題](./路徑/標題.md) - 一行摘要

## 人物
- [名字](./路徑/名字.md) - 一行摘要 | 日期: YYYY-MM-DD

## 工具
- [名稱](./路徑/名稱.md) - 一行摘要
```

### log.md 格式

```markdown
# 日誌

## [YYYY-MM-DD] ingest | 來源標題
- 攝取了來源
- 新增/更新了 X 個頁面

## [YYYY-MM-DD] query | 問題簡述
- 回答了問題
- 答案存為新頁面

## [YYYY-MM-DD] lint | 健康檢查
- 發現 X 個問題
- 修正了 Y 個問題
```

## 來源管理

### 存放規則
- 原始來源放入 `source/` 目錄
- 來源是不可變的（LLM 只讀不寫）
- 每個來源對應一個摘要頁（放在 `sources/` 子目錄）

### 來源摘要格式
```markdown
# 來源名稱

**來源類型**: PDF/文章/書籍/影片
**來源路徑**: source/路徑/名稱.ext
**攝取日期**: YYYY-MM-DD

## 摘要
（LLM 生成的摘要）

## 關鍵要點
- 要點 1
- 要點 2

## 相關頁面
- [相關概念](./概念/xxx.md)
- [相關主題](./主題/yyy.md)
```

## 交叉引用規範

### 頁內引用
```markdown
詳見 [相關概念](./概念/相關概念.md)
詳見 [章節](#章節名稱)
```

### 頁間引用
```markdown
這與 [人物/名字.md] 的貢獻相關。
參見 [主題/標題.md] 中的詳細說明。
```

### 來源引用
```markdown
根據 [source/教材.pdf] 的描述...
```

## 標籤與元數據

每個頁面可包含：

```markdown
---
tags: [標籤1, 標籤2]
date: YYYY-MM-DD
source_count: 3
last_updated: YYYY-MM-DD
---

# 標題

...
```

## 常見任務

### 新增分類
1. 建立目錄: `mkdir _wiki/新分類`
2. 建立 index.md
3. 建立 log.md
4. 更新全域 index.md

### 新增頁面
1. 撰寫內容（包含 tags 和 date）
2. 在 index.md 添加條目（含摘要）
3. 更新相關頁面的交叉引用

### 合併重複頁面
1. 保留較完整的版本
2. 將較短版本內容整合進去
3. 更新 index.md（移除舊條目，添加新條目）
4. 在較短版本位置添加重定向說明

## 品質檢查清單

每次操作後檢查：

- [ ] index.md 有新條目且有摘要？
- [ ] log.md 有新條目且格式正確？
- [ ] 新頁面有交叉引用？
- [ ] 相關現有頁面有引用新頁面？
- [ ] 內部連結都是有效的？
