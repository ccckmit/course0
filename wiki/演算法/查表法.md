# 查表法 (Lookup Table)

**標籤**: #演算法 #查表法 #空間換時間 #最佳化 #快取
**日期**: 2024-01-15

查表法 (Lookup Table) 是演算法設計中最基本也是最強大的最佳化技術之一。其核心思想是將預先計算好的結果存入表格，在執行時直接透過查表取得答案，從而避免重複的計算。這種方法以空間換取時間，在嵌入式系統效能優化、密碼學、遊戲開發等領域有廣泛應用。

## 查表法的基本原理

### 核心概念

查表法的運作機制非常簡單：預先計算所有可能的輸入對應輸出，儲存於一個表格（通常為陣列、雜湊表或.tree），執行時透過輸入作為索引或鍵直接取得輸出。時間複雜度從 O(n) 或更高降低到 O(1)，代價是額外的記憶體空間。

### 為什麼有效

在電腦系統中，記憶體存取的時間複雜度遠低於複雜計算。現代 CPU 的快取層級也優化了对連續記憶體的存取。因此，當計算成本較高且輸入範圍有限時，查表法往往是最有效的最佳化手段。

## 查表法的類型

### 陣列查表

適用於輸入範圍連續且可映射到整數索引的情況。這是最快速的查表方式，時間複雜度為 O(1)。

```python
# 預先計算 2 的冪次表
POW2 = [1 << i for i in range(32)]

def pow2_lookup(n):
    if 0 <= n < 32:
        return POW2[n]
    raise ValueError("超出範圍")

# 快速判斷是否為 2 的冪次
def is_power_of_two(n):
    return n > 0 and (n & (n - 1)) == 0
```

### 雜湊表查表

適用於輸入範圍不連續或鍵不是整數的情況。平均時間複雜度為 O(1)。

```python
# 費波那契數列查表
FIB_TABLE = {
    0: 0, 1: 1, 2: 1, 3: 2, 4: 3, 5: 5,
    6: 8, 7: 13, 8: 21, 9: 34, 10: 55,
    11: 89, 12: 144, 13: 233, 14: 377, 15: 610
}

def fib_lookup(n):
    if n in FIB_TABLE:
        return FIB_TABLE[n]
    return None

# 超過預設範圍時使用動態規劃
def fib_dynamic(n):
    if n in FIB_TABLE:
        return FIB_TABLE[n]
    
    # 計算並快取
    a, b = FIB_TABLE[14], FIB_TABLE[15]
    for i in range(16, n + 1):
        a, b = b, a + b
        FIB_TABLE[i] = b
    
    return b
```

### 區間查表

適用於連續輸入但只想取得近似值的情況，如數值分析中的查表。

```python
# 三角函數查表（角度到弧度）
import math
import bisect

SIN_TABLE = [math.sin(math.radians(i)) for i in range(361)]
ANGLES = list(range(361))

def sin_lookup(degrees):
    """精確查表"""
    return SIN_TABLE[degrees % 360]

def sin_lookup_interpolated(degrees):
    """線性插值查表（更高精度）"""
    d = degrees % 360
    lower = int(d)
    upper = (lower + 1) % 360
    fraction = d - lower
    return SIN_TABLE[lower] * (1 - fraction) + SIN_TABLE[upper] * fraction
```

## 經典應用場景

### 密碼學中的 S-Box

AES 加密演算法使用 S-Box (替換盒) 進行位元替換，這是一個 256 byte 的查表。直接計算 S-Box 需要多項式運算，查表只需一次記憶體存取。

```python
# 簡化版 S-Box 查表
SBOX = [
    0x63, 0x7c, 0x77, 0x7b, 0xf2, 0x6b, 0x6f, 0xc5,
    0x30, 0x01, 0x67, 0x2b, 0xfe, 0xd7, 0xab, 0x76,
    # ... 完整 S-Box
    0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07
]

def sub_bytes(state):
    """AES SubBytes 操作的查表版本"""
    return [SBOX[b] for b in state]
```

遊戲中大量使用預計算的動畫幀、光影效果、碰撞檢測等。

```python
# 預渲染的光影表（假設）
SHADOW_TABLE = [
    [[0.2, 0.1, 0.0], [0.3, 0.2, 0.1], ...],
    [[0.3, 0.2, 0.1], [0.4, 0.3, 0.2], ...],

def get_shadow_intensity(x, y, z):
    return SHADOW_TABLE[x][y][z]

# 查找表合併
def combine_shadow_and_light(shadow, light):

    for s in range(11):
```

### 訊號處理

數位訊號處理中，正弦波、餘弦波等週期函數常用查表法生成。

```python
import numpy as np

# 生成正弦波查找表
SAMPLE_RATE = 44100
FREQUENCY = 440  # A4
TABLE_SIZE = 1024

SINE_TABLE = [
    np.sin(2 * np.pi * FREQUENCY * i / SAMPLE_RATE)
    for i in range(TABLE_SIZE)
]

def get_sine_sample(time_step):
    """透過查表產生正弦波樣本"""
    index = int(time_step * TABLE_SIZE) % TABLE_SIZE
    return SINE_TABLE[index]
```

### 科學計算

特殊函數（如貝塞爾函數、伽馬函數）通常沒有簡單的解析解，常用查表法。

```python
# 標準常態分佈累積分佈函數 (CDF) 查表
from scipy.stats import norm

Z_TABLE = {i/10: norm.cdf(i/10) for i in range(-40, 41)}

def normal_cdf_lookup(z):
    """快速查表版本"""
    z = max(-4.0, min(4.0, z))
    key = round(z * 10) / 10
    return Z_TABLE[key]

# 對比精確版本
def normal_cdf_exact(z):
    return norm.cdf(z)
```

## 查表法與計算的權衡

### 時間空間權衡

| 特性 | 查表法 | 線上計算 |
|------|--------|----------|
| 時間複雜度 | O(1) | O(n) 或更高 |
| 空間複雜度 | O(k) | O(1) |
| 適用場景 | 離散、小範圍 | 連續、大範圍 |
| 精度 | 取決於表大小 | 可無限精確 |

### 決定何時使用查表法

考慮以下因素：輸入範圍是否有限且可枚舉？計算成本是否高於記憶體存取？表大小是否可接受？是否需要精確結果或近似即可？是否涉及硬體加速（FPGA/ASIC）？

### 混合策略

有時可以結合查表和計算，既享受速度又保持精度。

```python
# 混合方法：三項式近似 + 查表修正
COEFFICIENTS = {
    'sin': [0, 1, 0, -1/6, 0, 1/120],  # sin(x) ≈ x - x³/6 + x⁵/120
}

def sin_hybrid(x):
    """三項式近似 + 查表修正"""
    # 將角度標準化到 [-π, π]
    x = x % (2 * np.pi)
    if x > np.pi:
        x -= 2 * np.pi
    
    # 使用近似公式
    x2 = x * x
    approx = x * (1 - x2/6 * (1 - x2/20))
    
    return approx  # 可添加查表修正項
```

## 查表法的進階技巧

### 壓縮查表

當查表表過大時，可以使用壓縮技術。對稱簡化利用函數的對稱性減少一半表大小。區間劃分將輸入劃分為多個區間，每個區間使用較小的查表。差分編碼儲存差分而非絕對值，減少需要的精度。

```python
# 利用正弦函數對稱性的壓縮表
# sin(x) = sin(π - x) = -sin(-x) = -sin(π + x)
HALF_PI_TABLE = [math.sin(i * 0.001) for i in range(1571)]  # 0 到 π/2

def sin_compressed(x):
    """使用 1/4 表查詢"""
    x = x % (2 * math.pi)
    if x < 0:
        x += 2 * math.pi
    
    if x <= math.pi / 2:
        idx = int(x * 1000)
        return HALF_PI_TABLE[idx]
    elif x <= math.pi:
        idx = int((math.pi - x) * 1000)
        return HALF_PI_TABLE[idx]
    elif x <= 3 * math.pi / 2:
        idx = int((x - math.pi) * 1000)
        return -HALF_PI_TABLE[idx]
    else:
        idx = int((2 * math.pi - x) * 1000)
        return -HALF_PI_TABLE[idx]
```

### 自適應查表

根據運行時的資料特性動態調整查表。

```python
from collections import defaultdict

class AdaptiveLookupTable:
    def __init__(self, compute_func, threshold=100):
        self.compute = compute_func
        self.cache = {}
        self.threshold = threshold
        self.hits = defaultdict(int)
    
    def get(self, key):
        if key in self.cache:
            self.hits[key] += 1
            return self.cache[key]
        
        result = self.compute(key)
        self.cache[key] = result
        
        # 清理低頻訪問的項目
        if len(self.cache) > self.threshold:
            min_hits = min(self.hits.values())
            self.cache = {k: v for k, v in self.cache.items() if self.hits[k] > min_hits}
        
        return result
```

在編譯時或部署前預先計算查表，減少運行時開銷。

## 硬體應用

### FPGA/ASIC

在硬體實現中，查表法特別有價值，因為：邏輯閘數量遠少於計算電路；速度快，無需計算週期；功耗低。

```verilog
// Verilog 中的查表實現
module sine_lut (
    input [7:0] angle,  // 0-255 代表 0-360度
    output [15:0] sine  // Q8.8 固定點輸出
);

    // 預先計算的正弦表
    wire [15:0] sine_table [0:255];
    assign sine = sine_table[angle];

endmodule
```

### 記憶體層級優化

快取友好的查表實現應該：使用連續記憶體避免隨機存取。對齊記憶體以充分利用 CPU 快取。考慮預取策略。

```python
# 快取優化的查表實現
import numpy as np

class CacheOptimizedLUT:
    def __init__(self, size=4096):
        self.size = size
        # 對齊到快取行大小 (64 bytes)
        self.table = np.zeros(size, dtype=np.float32)
    
    def build(self, func):
        for i in range(self.size):
            self.table[i] = func(i * 2 * np.pi / self.size)
    
    def lookup(self, x):
        # 映射輸入到索引
        idx = int(x * self.size / (2 * np.pi)) % self.size
        return self.table[idx]
```

## 實際應用案例

### CRC 校驗

CRC 演算法使用查表法實現快速計算。

```python
# CRC-32 查表實現
CRC_TABLE = []

def init_crc_table():
    global CRC_TABLE
    for i in range(256):
        crc = i
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0xEDB88320
            else:
                crc >>= 1
        CRC_TABLE.append(crc)

def crc32(data):
    crc = 0xFFFFFFFF
    for byte in data:
        crc = CRC_TABLE[(crc ^ byte) & 0xFF] ^ (crc >> 8)
    return crc ^ 0xFFFFFFFF
```

### 圖形變換

影射變換中使用查表法加速計算。

```python
# 旋轉映射表
def create_rotation_table(angle, size):
    table = {}
    for y in range(size):
        for x in range(size):
            # 計算旋轉後的座標
            new_x = x * math.cos(angle) - y * math.sin(angle)
            new_y = x * math.sin(angle) + y * math.cos(angle)
            table[(y, x)] = (new_y, new_x)
    return table
```

### 快速字串匹配

KMP 或 BM 演算法的失敗函數可以預先計算。

```python
# KMP 失敗函數查表
def compute_failure_function(pattern):
    m = len(pattern)
    pi = [0] * m
    for i in range(1, m):
        j = pi[i-1]
        while j > 0 and pattern[i] != pattern[j]:
            j = pi[j-1]
        if pattern[i] == pattern[j]:
            j += 1
        pi[i] = j
    return pi
```

## 與其他演算法的比較

### 動態規劃

查表法與動態規劃密切相關，DP 的遞推表本质上就是查表。當所有子問題都需要解決時，DP 更合適。當只有部分子問題需要解決時，自底向上的查表（ Memoization）更合適。

### 快取

作業系統的 CPU 快取和磁碟快取本質上也是動態的查表法。區別在於快取是自動管理的，而應用層查表需要手動設計。

### 雜湊表

雜湊表是一種動態查表結構，支援任意鍵的 O(1) 查找。當輸入範圍很大或鍵不是整數時，雜湊表是首選。
