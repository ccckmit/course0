# 時間複雜度

時間複雜度是演算法分析中最核心的概念之一，它描述了演算法的執行時間隨輸入規模增長的趨勢。在電腦科學中，我們通常使用大O標記法（Big-O Notation）來表示時間複雜度，這是一種描述函數成長率的數學表示法。理解時間複雜度對於選擇合適的演算法、優化程式效能至關重要，尤其在處理大規模資料時更是如此。

## 為什麼需要時間複雜度？

在實際應用中，同樣的問題可能有多種不同的解決方案。例如，對於排序問題，我們可以選擇氣泡排序、合併排序、快速排序等多種演算法。這些演算法在資料量小時可能差異不明顯，但當資料量達到數百萬甚至數十億時，它們的效能差異將變得非常顯著。

時間複雜度幫助我們在設計演算法時就能預測其效能表現，而不需要在實際執行後才發現問題。這種前瞻性的分析對於建構高效能系統至關重要。

## 大O標記法詳解

大O標記法用於描述函數的上界，即在最壞情況下，演算法執行時間的成長趨勢。它的定義如下：

> 如果存在正數 c 和 n₀，使得當 n ≥ n₀ 時，f(n) ≤ c × g(n)，則 f(n) = O(g(n))

常見的複雜度等級從低到高排列如下：

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ)
```

複雜度等級說明：
- **常數時間 O(1)**：執行時間與輸入規模無關，永遠在固定時間內完成
- **對數時間 O(log n)**：執行時間與輸入規模的對數成正比，如二分搜尋
- **線性時間 O(n)**：執行時間與輸入規模成正比，如線性搜尋
- **線性對數時間 O(n log n)**：如合併排序、快速排序
- **平方時間 O(n²)**：如氣泡排序、選擇排序
- **立方時間 O(n³)**：如矩陣乘法（傳統方法）
- **指數時間 O(2ⁿ)**：如暴力搜尋所有子集
- **階乘時間 O(n!)**：如排列問題的暴力解法

## 常見時間複雜度詳細說明

### 1. 常數時間 O(1)

常數時間複雜度意味著無論輸入規模有多大，演算法都在固定的時間內完成。這是最理想的時間複雜度。

範例：
```python
# 陣列隨機存取 - 永遠只需要一次指標計算
def get_element(arr, index):
    return arr[index]

# 交換兩個變數
def swap(a, b):
    temp = a
    a = b
    b = temp
```

實際應用場景：雜湊表查找（平均情況）、陣列存取、算術運算。

### 2. 對數時間 O(log n)

對數時間複雜度出現在每次操作都將問題規模縮減的演算法中。最典型的例子是二元搜尋。

範例：
```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = (left + right) // 2
        
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1
```

為什麼是 O(log n)？每次比較都將搜尋範圍縮小一半，因此最多需要 log₂(n) 次比較。

實際應用場景：二元搜尋、平衡樹操作（AVL、紅黑樹）、二分搜尋樹。

### 3. 線性時間 O(n)

線性時間複雜度意味著執行時間與輸入規模成正比。這是處理未排序資料時的基礎複雜度。

範例：
```python
# 線性搜尋
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# 求最大值
def find_max(arr):
    max_val = arr[0]
    for val in arr:
        if val > max_val:
            max_val = val
    return max_val

# 計算總和
def sum_array(arr):
    total = 0
    for val in arr:
        total += val
    return total
```

實際應用場景：線性搜尋、遍歷陣列、單次遍歷的所有操作。

### 4. 線性對數時間 O(n log n)

線性對數時間複雜度是高效排序演算法的標準複雜度，如合併排序和快速排序。

範例：
```python
# 合併排序
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

實際應用場景：合併排序、快速排序、堆排序。任何基於比較的高效排序理論下限都是 O(n log n)。

### 5. 平方時間 O(n²)

平方時間複雜度通常出現在嵌套迴圈中。雖然不是最優的，但在小規模資料時仍然可用。

範例：
```python
# 氣泡排序
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# 選擇排序
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

# 檢查是否存在重複元素（暴力方法）
def has_duplicate(arr):
    n = len(arr)
    for i in range(n):
        for j in range(i + 1, n):
            if arr[i] == arr[j]:
                return True
    return False
```

實際應用場景：基本排序演算法（氣泡、選擇、插入）、矩陣運算、暴力搜尋配對。

### 6. 立方時間 O(n³)

立方時間複雜度通常出現在三層嵌套迴圈中，效能通常較差。

範例：
```python
# 矩陣乘法（傳統方法）
def matrix_multiply(A, B):
    n = len(A)
    C = [[0] * n for _ in range(n)]
    
    for i in range(n):
        for j in range(n):
            for k in range(n):
                C[i][j] += A[i][k] * B[k][j]
    
    return C

# 三數之和問題（暴力解）
def three_sum_brute(nums):
    n = len(nums)
    result = []
    
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    result.append([nums[i], nums[j], nums[k]])
    
    return result
```

實際應用場景：傳統矩陣乘法、動態規劃中的三維DP、暴力求解某些最佳化問題。

### 7. 指數時間 O(2ⁿ)

指數時間複雜度通常出現在需要列舉所有可能情況的問題中，隨著輸入規模增加，執行時間會急劇增長。

範例：
```python
# 生成所有子集
def generate_subsets(nums):
    n = len(nums)
    result = []
    
    for mask in range(2 ** n):
        subset = []
        for i in range(n):
            if mask & (1 << i):
                subset.append(nums[i])
        result.append(subset)
    
    return result

# 費波那契數列（暴力遞迴）
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# 漢諾塔
def hanoi(n, source, target, auxiliary):
    if n == 1:
        print(f"Move disk 1 from {source} to {target}")
        return
    
    hanoi(n - 1, source, auxiliary, target)
    print(f"Move disk {n} from {source} to {target}")
    hanoi(n - 1, auxiliary, target, source)
```

實際應用場景：子集生成、某些加密演算法、NP完全問題的暴力解。

## 時間複雜度分析技巧

### 1. 忽略常數係數

在 Big-O 表示法中，我們只關心趨勢，不關心常數。例如：
- O(2n) = O(n)
- O(1000n) = O(n)
- O(0.5n²) = O(n²)

### 2. 忽略低階項

當有多個項時，只保留最高階項：
- O(n² + n) = O(n²)
- O(n³ + n² + n) = O(n³)
- O(log n + n) = O(n)

### 3. 最壞情況 vs 平均情況 vs 最好情況

- **最壞情況（Worst Case）**：輸入最不利時的複雜度
- **平均情況（Average Case）**：所有輸入的平均複雜度
- **最好情況（Best Case）**：輸入最有利時的複雜度

例如，線性搜尋的最壞情況是 O(n)，平均情況也是 O(n)，最好情況是 O(1)（目標在第一個位置）。

### 4. 常見迴圈模式

```python
# O(n) - 單層迴圈
for i in range(n):
    ...

# O(n²) - 雙層迴圈
for i in range(n):
    for j in range(n):
        ...

# O(n³) - 三層迴圈
for i in range(n):
    for j in range(n):
        for k in range(n):
            ...

# O(log n) - 每次迭代減半
while n > 0:
    n = n // 2

# O(n log n) - 分治法常見
def divide_conquer(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = divide_conquer(arr[:mid])
    right = divide_conquer(arr[mid:])
    return merge(left, right)
```

## 複雜度速查表

| 複雜度 | 名稱 | 典型演算法 | n=1000 時的比較次數 |
|--------|------|------------|---------------------|
| O(1) | 常數 | 雜湊表查找 | 1 |
| O(log n) | 對數 | 二分搜尋 | ~10 |
| O(n) | 線性 | 線性搜尋 | 1,000 |
| O(n log n) | 線性對數 | 合併排序 | ~10,000 |
| O(n²) | 平方 | 氣泡排序 | 1,000,000 |
| O(n³) | 立方 | 矩陣乘法 | 1,000,000,000 |
| O(2ⁿ) | 指數 | 子集生成 | 2^1000 ≈ 10^301 |

## 實際應用建議

### 選擇正確的複雜度

- **資料量 < 100**：O(n²) 也可以接受
- **資料量 100-10,000**：選擇 O(n log n)
- **資料量 10,000-1,000,000**：選擇 O(n) 或 O(n log n)
- **資料量 > 1,000,000**：必須選擇 O(n) 或更好

### 常見優化策略

1. **避免嵌套迴圈**：將 O(n²) 化為 O(n)
2. **使用雜湊表**：將 O(n) 化為 O(1)
3. **使用排序**：將 O(n²) 搜尋化為 O(n log n)
4. **分治法**：將 O(n) 化為 O(log n)

## 與空間複雜度的關係

時間複雜度和空間複雜度通常需要權衡。有些演算法用空間換取時間（如動態規劃），有些用時間換取空間（如雜湊表）。理解兩者的關係對於設計高效演算法非常重要。
