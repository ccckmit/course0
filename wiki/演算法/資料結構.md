# 資料結構

資料結構是電腦科學的基石，它定義瞭如何組織、儲存和操作資料。選擇合適的資料結構可以大幅提升演算法的效率，而錯誤的選擇則可能導致效能瓶頸。本質上，資料結構是程式與資料之間的橋樑，好的資料結構設計能夠讓我們以更高效的方式解決問題。

## 為什麼需要資料結構？

在現實世界處理資料時，我們面臨各種不同的需求：

- **快速搜尋**：在大量資料中找到特定項目
- **有效插入/刪除**：頻繁新增或移除資料
- **有序遍歷**：按照特定順序處理所有資料
- **記憶體效率**：在有限的記憶體空間中儲存大量資料

不同的資料結構針對這些需求有不同的優化方向，沒有萬能的資料結構，只有最適合特定場景的選擇。

## 線性資料結構

線性資料結構是最基礎也是最常見的資料結構類型，其元素按照線性順序排列，每個元素（除首尾外）都有唯一的前驅和後繼。

### 1. 陣列（Array）

陣列是最基礎的線性資料結構，元素在記憶體中連續儲存，透過索引直接存取任意位置。

#### 陣列的特點

- **連續記憶體**：所有元素在記憶體中相鄰排列
- **固定大小**：建立時決定大小，通常無法動態擴展
- **O(1) 隨機存取**：透過索引直接訪問，無需遍歷
- **O(n) 插入/刪除**：需要移動後續所有元素

```python
# Python 列表（動態陣列）的使用
arr = [1, 2, 3, 4, 5]

# O(1) 隨機存取
print(arr[2])        # 輸出: 3

# O(n) 插入
arr.insert(2, 99)    # 在索引2插入99

# O(n) 刪除
arr.pop(2)           # 刪除索引2的元素

# 走訪
for i, val in enumerate(arr):
    print(f"Index {i}: {val}")
```

#### 陣列的類型

- **靜態陣列**：大小固定，如 C 語言的 `int arr[10]`
- **動態陣列**：大小可變，如 Python 的 list、C++ 的 vector、Java 的 ArrayList
- **多維陣列**：多個維度的陣列，如二維陣列、三維陣列

```python
# 二維陣列（矩陣）
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# 存取元素
print(matrix[1][2])  # 輸出: 6

# 走訪
for row in matrix:
    for val in row:
        print(val, end=" ")
    print()
```

### 2. 連結串列（Linked List）

連結串列由節點（Node）組成，每個節點包含資料和指向下一個節點的指標（指標）。

#### 單向連結串列

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
    
    def prepend(self, data):
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node
    
    def delete(self, data):
        if not self.head:
            return
        
        if self.head.data == data:
            self.head = self.head.next
            return
        
        current = self.head
        while current.next:
            if current.next.data == data:
                current.next = current.next.next
                return
            current = current.next
    
    def search(self, data):
        current = self.head
        while current:
            if current.data == data:
                return True
            current = current.next
        return False
    
    def display(self):
        current = self.head
        while current:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

# 使用範例
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)
ll.prepend(0)
ll.display()  # 輸出: 0 -> 1 -> 2 -> 3 -> None
```

#### 雙向連結串列

```python
class DNode:
    def __init__(self, data):
        self.data = data
        self.prev = None
        self.next = None

class DoublyLinkedList:
    def __init__(self):
        self.head = None
        self.tail = None
    
    def append(self, data):
        new_node = DNode(data)
        
        if not self.head:
            self.head = self.tail = new_node
        else:
            new_node.prev = self.tail
            self.tail.next = new_node
            self.tail = new_node
    
    def prepend(self, data):
        new_node = DNode(data)
        
        if not self.head:
            self.head = self.tail = new_node
        else:
            new_node.next = self.head
            self.head.prev = new_node
            self.head = new_node
```

#### 連結串列的複雜度

| 操作 | 時間複雜度 | 說明 |
|------|------------|------|
| 隨機存取 | O(n) | 需要遍歷到指定位置 |
| 插入（頭） | O(1) | 直接在頭部添加 |
| 插入（尾） | O(n) | 需要遍歷到尾部（雙向串列可優化為 O(1)） |
| 刪除（頭） | O(1) | 直接刪除頭部 |
| 刪除（尾） | O(n) | 需要遍歷到尾部 |
| 搜尋 | O(n) | 需要遍歷 |

### 3. 堆疊（Stack）

堆疊是遵循 LIFO（Last In, First Out）原則的資料結構，最後加入的元素最先被移除。

```python
class Stack:
    def __init__(self):
        self.items = []
    
    def push(self, item):
        self.items.append(item)
    
    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        return None
    
    def peek(self):
        if not self.is_empty():
            return self.items[-1]
        return None
    
    def is_empty(self):
        return len(self.items) == 0
    
    def size(self):
        return len(self.items)

# 使用範例
stack = Stack()
stack.push(1)
stack.push(2)
stack.push(3)

print(stack.pop())  # 輸出: 3
print(stack.pop())  # 輸出: 2
print(stack.peek()) # 輸出: 1
```

#### 堆疊的應用

- **函數呼叫堆疊**：追蹤程式執行流程
- **運算式求值**：中綴轉後綴、後綴求值
- **深度優先搜尋（DFS）**
- **括號匹配**
- **undo/redo 功能**

```python
# 括號匹配
def is_valid_parentheses(s):
    stack = Stack()
    mapping = {')': '(', '}': '{', ']': '['}
    
    for char in s:
        if char in '({[':
            stack.push(char)
        elif char in ')}]':
            if stack.is_empty() or stack.pop() != mapping[char]:
                return False
    
    return stack.is_empty()

print(is_valid_parentheses("({[]})"))  # True
print(is_valid_parentheses("({)}"))     # False
```

### 4. 佇列（Queue）

佇列是遵循 FIFO（First In, First Out）原則的資料結構，最先加入的元素最先被移除。

```python
class Queue:
    def __init__(self):
        self.items = []
    
    def enqueue(self, item):
        self.items.append(item)
    
    def dequeue(self):
        if not self.is_empty():
            return self.items.pop(0)
        return None
    
    def front(self):
        if not self.is_empty():
            return self.items[0]
        return None
    
    def is_empty(self):
        return len(self.items) == 0
    
    def size(self):
        return len(self.items)

# 使用範例
queue = Queue()
queue.enqueue(1)
queue.enqueue(2)
queue.enqueue(3)

print(queue.dequeue())  # 輸出: 1
print(queue.dequeue())  # 輸出: 2
print(queue.front())    # 輸出: 3
```

#### 優先權佇列

優先權佇列中的元素具有優先權，高優先權的元素先被處理。

```python
import heapq

class PriorityQueue:
    def __init__(self):
        self.heap = []
    
    def enqueue(self, priority, item):
        heapq.heappush(self.heap, (priority, item))
    
    def dequeue(self):
        if not self.is_empty():
            return heapq.heappop(self.heap)[1]
        return None
    
    def is_empty(self):
        return len(self.heap) == 0

# 使用範例
pq = PriorityQueue()
pq.enqueue(3, "低")
pq.enqueue(1, "高")
pq.enqueue(2, "中")

print(pq.dequeue())  # 輸出: 高
print(pq.dequeue())  # 輸出: 中
print(pq.dequeue())  # 輸出: 低
```

## 非線性資料結構

非線性資料結構的元素之間不是簡單的線性順序，而是具有更複雜的層次或網狀關係。

### 1. 樹（Tree）

樹是一種層次結構資料結構，由節點組成，每個節點有零或多個子節點，除根節點外每個節點有唯一父節點。

#### 二元搜尋樹（Binary Search Tree, BST）

```python
class TreeNode:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None

class BST:
    def __init__(self):
        self.root = None
    
    def insert(self, key):
        if not self.root:
            self.root = TreeNode(key)
            return
        
        current = self.root
        while True:
            if key < current.key:
                if not current.left:
                    current.left = TreeNode(key)
                    return
                current = current.left
            else:
                if not current.right:
                    current.right = TreeNode(key)
                    return
                current = current.right
    
    def search(self, key):
        current = self.root
        while current:
            if key == current.key:
                return True
            elif key < current.key:
                current = current.left
            else:
                current = current.right
        return False
    
    def delete(self, key):
        self.root = self._delete_node(self.root, key)
    
    def _delete_node(self, node, key):
        if not node:
            return None
        
        if key < node.key:
            node.left = self._delete_node(node.left, key)
        elif key > node.key:
            node.right = self._delete_node(node.right, key)
        else:
            # 找到要刪除的節點
            if not node.left:
                return node.right
            if not node.right:
                return node.left
            
            # 兩個子樹都存在，找後繼
            min_larger_node = self._find_min(node.right)
            node.key = min_larger_node.key
            node.right = self._delete_node(node.right, min_larger_node.key)
        
        return node
    
    def _find_min(self, node):
        current = node
        while current.left:
            current = current.left
        return current
    
    def inorder(self):
        result = []
        self._inorder_helper(self.root, result)
        return result
    
    def _inorder_helper(self, node, result):
        if node:
            self._inorder_helper(node.left, result)
            result.append(node.key)
            self._inorder_helper(node.right, result)

# 使用範例
bst = BST()
for key in [7, 3, 9, 1, 5, 8, 10]:
    bst.insert(key)

print(bst.inorder())   # 輸出: [1, 3, 5, 7, 8, 9, 10]
print(bst.search(5))   # 輸出: True
print(bst.search(6))  # 輸出: False
```

#### 平衡樹

為瞭解決 BST 在最差情況下退化成鏈表的問題，出現了各種平衡樹：

- **AVL 樹**：高度平衡，任何節點的左右子樹高度差不超過 1
- **紅黑樹**：保證最長路徑不超過最短路徑的兩倍
- **B 樹 / B+ 樹**：適合磁碟存取的平衡樹，常用於資料庫和檔案系統

```python
# 簡化的 AVL 樹節點
class AVLNode:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None
        self.height = 1

class AVLTree:
    def get_height(self, node):
        return node.height if node else 0
    
    def get_balance(self, node):
        return self.get_height(node.left) - self.get_height(node.right) if node else 0
    
    def right_rotate(self, y):
        x = y.left
        T2 = x.right
        
        x.right = y
        y.left = T2
        
        y.height = 1 + max(self.get_height(y.left), self.get_height(y.right))
        x.height = 1 + max(self.get_height(x.left), self.get_height(x.right))
        
        return x
    
    def left_rotate(self, x):
        y = x.right
        T2 = y.left
        
        y.left = x
        x.right = T2
        
        x.height = 1 + max(self.get_height(x.left), self.get_height(x.right))
        y.height = 1 + max(self.get_height(y.left), self.get_height(y.right))
        
        return y
    
    def insert(self, node, key):
        if not node:
            return AVLNode(key)
        
        if key < node.key:
            node.left = self.insert(node.left, key)
        elif key > node.key:
            node.right = self.insert(node.right, key)
        else:
            return node
        
        node.height = 1 + max(self.get_height(node.left), self.get_height(node.right))
        
        balance = self.get_balance(node)
        
        # Left Left Case
        if balance > 1 and key < node.left.key:
            return self.right_rotate(node)
        
        # Right Right Case
        if balance < -1 and key > node.right.key:
            return self.left_rotate(node)
        
        # Left Right Case
        if balance > 1 and key > node.left.key:
            node.left = self.left_rotate(node.left)
            return self.right_rotate(node)
        
        # Right Left Case
        if balance < -1 and key < node.right.key:
            node.right = self.right_rotate(node.right)
            return self.left_rotate(node)
        
        return node
```

### 2. 圖（Graph）

圖由頂點（Vertex）和邊（Edge）組成，用於表示物件之間的關係。

```python
class Graph:
    def __init__(self):
        self.adjacency_list = {}
    
    def add_vertex(self, vertex):
        if vertex not in self.adjacency_list:
            self.adjacency_list[vertex] = []
    
    def add_edge(self, v1, v2, weight=None):
        self.add_vertex(v1)
        self.add_vertex(v2)
        
        self.adjacency_list[v1].append((v2, weight))
        # 對於無向圖，還需要添加：
        # self.adjacency_list[v2].append((v1, weight))
    
    def bfs(self, start):
        visited = set()
        queue = [start]
        result = []
        
        while queue:
            vertex = queue.pop(0)
            if vertex not in visited:
                visited.add(vertex)
                result.append(vertex)
                
                for neighbor, _ in self.adjacency_list[vertex]:
                    if neighbor not in visited:
                        queue.append(neighbor)
        
        return result
    
    def dfs(self, start):
        visited = set()
        result = []
        
        def dfs_helper(vertex):
            visited.add(vertex)
            result.append(vertex)
            
            for neighbor, _ in self.adjacency_list[vertex]:
                if neighbor not in visited:
                    dfs_helper(neighbor)
        
        dfs_helper(start)
        return result

# 使用範例
g = Graph()
g.add_edge('A', 'B')
g.add_edge('A', 'C')
g.add_edge('B', 'D')
g.add_edge('C', 'D')
g.add_edge('D', 'E')

print(g.bfs('A'))  # 輸出: ['A', 'B', 'C', 'D', 'E']
print(g.dfs('A'))  # 輸出: ['A', 'B', 'D', 'C', 'E']
```

#### 圖的表示方法

1. **鄰接矩陣**：使用二維矩陣表示邊，適合稠密圖
2. **鄰接串列**：使用串列表示每個頂點的相鄰頂點，適合稀疏圖
3. **邊列表**：列舉所有邊

### 3. 雜湊表（Hash Table）

雜湊表透過雜湊函數將鍵映射到陣列索引，實現平均 O(1) 時間的查找、插入和刪除。

```python
class HashTable:
    def __init__(self, size=10):
        self.size = size
        self.table = [[] for _ in range(size)]
    
    def _hash(self, key):
        return hash(key) % self.size
    
    def insert(self, key, value):
        index = self._hash(key)
        
        # 更新已存在的鍵
        for i, (k, v) in enumerate(self.table[index]):
            if k == key:
                self.table[index][i] = (key, value)
                return
        
        # 新增鍵值對
        self.table[index].append((key, value))
    
    def get(self, key):
        index = self._hash(key)
        
        for k, v in self.table[index]:
            if k == key:
                return v
        
        return None
    
    def delete(self, key):
        index = self._hash(key)
        
        for i, (k, v) in enumerate(self.table[index]):
            if k == key:
                del self.table[index][i]
                return True
        
        return False

# 使用範例
ht = HashTable()
ht.insert('name', 'Alice')
ht.insert('age', 25)
ht.insert('city', 'Taipei')

print(ht.get('name'))  # 輸出: Alice
print(ht.get('age'))   # 輸出: 25

ht.delete('age')
print(ht.get('age'))   # 輸出: None
```

#### 雜湊衝突處理

1. **連結法（Chaining）**：在相同索引處使用串列存放多個元素
2. **開放定址法（Open Addressing）**：探測其他空位
   - 線性探測（Linear Probing）
   - 二次探測（Quadratic Probing）
   - 雙雜湊（Double Hashing）

## 資料結構複雜度速查表

| 資料結構 | 存取 | 搜尋 | 插入 | 刪除 |
|----------|------|------|------|------|
| 陣列 | O(1) | O(n) | O(n) | O(n) |
| 連結串列 | O(n) | O(n) | O(1)* | O(1)* |
| 堆疊 | - | O(n) | O(1) | O(1) |
| 佇列 | - | O(n) | O(1) | O(1) |
| BST | O(n)** | O(n)** | O(n)** | O(n)** |
| 平衡樹 | O(log n) | O(log n) | O(log n) | O(log n) |
| 雜湊表 | - | O(1) | O(1) | O(1) |
| 圖（鄰接表） | O(V) | O(V+E) | O(1) | O(1) |

* 在已知位置的情況下
** 最差情況，最好情況為 O(log n)

## 選擇資料結構的考量因素

1. **操作類型**：需要進行哪些操作（搜尋、插入、刪除、遍歷）
2. **資料規模**：資料量大小，影響空間和時間效率
3. **記憶體限制**：可用記憶體空間
4. **順序需求**：是否需要維持資料順序
5. **更新頻率**：資料變更的頻率
