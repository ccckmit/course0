# Skill: 程式碼註解生成器

## 描述

自動為 source code 檔案產生深度技術文檔，說明程式的原理（數學、密碼學、演算法等）。

## 觸發條件

當使用者要求：
- 為某個 .c/.cpp/.py/.js/.rs 等程式碼加說明
- 解釋程式的原理
- 為程式加技術文檔

## 輸入

- 程式碼檔案路徑
- （可選）特定領域知識要求（如：密碼學領域請說明底層數學）

## 工作流程

### 1. 閱讀原始碼

先完整閱讀程式碼，理解：
- 資料結構定義
- 主要函數/類別
- 輸入輸出
- 使用的外部函式庫

### 2. 分析領域知識

根據程式碼領域，提取對應的技術原理：

| 領域 | 需要說明的內容 |
|------|---------------|
| 密碼學 | 數學基礎（AES/SHA/RSA 等）、金鑰管理、填充模式 |
| 區塊鏈 | 共識機制、雜湎函數、資料結構 |
| 網路 | 協定設計、socket 编程、加密傳輸 |
| 資料結構 | 時間/空間複雜度、操作流程 |
| 壓縮/編碼 | 演算法原理、編碼方式 |
| 資料庫 | 索引、ACID、事務處理 |

### 3. 撰寫文檔結構

為每個主要功能區塊說明：

```markdown
# [檔案名稱] 深入解析

## 1. 概述
[一句話說明此程式做什麼]

## 2. [主要功能/資料結構]
[說明核心資料結構或主要流程]

## 3. [核心演算法]
[數學/密碼學原理]

## 4. [實作細節]
[程式碼重點解說]

## 5. 安全性分析
[已知問題、攻擊向量、建議]

## 6. 使用方式
[如何編譯、執行]
```

### 4. 具體寫作原則

#### 數學原理
- 說明背后數學公式
- 舉例說明（如：RSA 的加密/解密數學式）

#### 演算法
- 解释核心逻辑
- 說明時間/空間複雜度

#### 安全性
- 分析潛在攻擊
- 给出改進建議

## 輸出

在原始碼同目錄下建立 `[原檔名].md`，例如：
- `aes.c` → `aes.c.md`
- `blockchain.py` → `blockchain.py.md`
- `http_server.js` → `http_server.js.md`

## 範例輸出

參考 `/Users/Shared/ccc/computer0/crpyto/ssl0/src/` 下的 .c.md 檔案：

## 注意事項

- 不要只描述「做什麼」，要說明「為什麼這樣做」
- 包含數學公式或演算法推導
- 分析安全性影響
- 提供實際使用範例
