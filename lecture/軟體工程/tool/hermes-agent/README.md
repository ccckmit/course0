## ccc: hermes-agent 是什麼

**Hermes-Agent** 是由 **Nous Research**（開發 Hermes 系列模型的頂尖團隊）推出的官方開源框架，專門為了將其模型轉化為具有「自主行動能力」的**軟體工程代理**而設計。

如果說 Hermes 是「大腦」，那麼 Hermes-Agent 就是這顆大腦的「神經系統與手腳」。

---

## 1. 核心定位：為什麼需要它？
一般的聊天機器人只能「給建議」，但 Hermes-Agent 的目標是「**執行任務**」。它不僅僅是調用模型，而是建立了一個讓 AI 能在裡面安全運作的環境。

它的核心設計圍繞著一個概念：**工具調用循環（Tool-Use Loop）**。
1.  **思考 (Thought)**：AI 分析任務（例如：修復一個前端 Bug）。
2.  **行動 (Action)**：AI 決定使用工具（例如：讀取 `App.tsx`、執行 `npm test`）。
3.  **觀察 (Observation)**：AI 獲取工具執行的結果（例如：測試報錯了）。
4.  **修正 (Correction)**：AI 根據結果重新思考，進入下一輪循環。

---

## 2. 關鍵特性

### **A. 針對 Hermes 模型優化**
雖然它支援其他模型，但它是為了 Hermes 模型的 **XML 標籤式推理** 優化的。Hermes 擅長輸出類似 `<scratchpad>`（思考區）與 `<tool_call>`（工具調用）的格式，這讓 Hermes-Agent 的穩定性遠高於一般的開源 Agent。

### **B. 本地化與安全性**
Hermes-Agent 強調 **Local-First**（本地優先）。它通常搭配 **Docker** 運作，這意味著 AI 在修改你的程式碼或執行指令時，是在一個隔離的容器裡進行的，不會不小心刪掉你主機上的資料。

### **C. 多步驟複雜任務處理**
它不僅能寫一段 Code，它能處理跨檔案的任務。例如：
> 「幫我把專案裡的狀態管理從 Redux 遷移到 Zustand，並確保所有的單元測試都能通過。」

---

## 3. 與其他工具的關係

* **與 OpenCode 的關係**：
    * **OpenCode** 比較像是一個「現成的產品/外殼」，適合直接拿來用在日常開發。
    * **Hermes-Agent** 則是「底層框架」，如果你想**開發自己的專案機器人**，或是在 CI/CD 流程中嵌入自定義的邏輯，會選用它。
* **與 vLLM / Ollama 的關係**：
    * Hermes-Agent 本身不跑模型，它會透過 API 連接到 **vLLM** 或 **Ollama** 來獲得推論能力。

---

## 4. 典型的工作流範例

如果你運行了 Hermes-Agent，流程通常如下：

1.  **環境初始化**：Agent 啟動並掃描你的專案目錄。
2.  **理解需求**：你輸入「幫我把 API 接口加上錯誤處理機制」。
3.  **自主搜尋**：Agent 調用搜尋工具找出所有定義 API 的檔案。
4.  **撰寫計畫**：它會在終端機告訴你它打算改哪幾個檔案。
5.  **執行修改**：它逐一寫入程式碼。
6.  **自我驗證**：它會主動執行編譯指令，如果編譯報錯，它會**自己讀取錯誤訊息並修復**，直到成功為止。

---

## 總結
**Hermes-Agent** 是 Nous Research 為了證明「開源模型也能達到類似 Devin 或 GitHub Copilot Workspace 的自主性」而交出的答卷。

如果你是一個開發者，想要體驗**完全不受雲端廠商監控**、且具備**高度推理能力**的自動化開發系統，那麼 **Hermes-3 模型 + vLLM (推論) + Hermes-Agent (框架)** 是目前開源界最頂尖的組合之一。
