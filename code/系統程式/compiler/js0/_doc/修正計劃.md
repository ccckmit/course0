# js0i.js 修正計劃

## 問題分析

### 錯誤訊息
```
SyntaxError: [Parser] Line 966: Expected {, got KEYWORD 'return'
```

### 根本原因
當 `node js0i.js js0i.js js/hello.js` 執行時，js0i.js 作為解譯器試圖解析自己的源代碼。在解析 `Env` 類別時遇到問題：

```javascript
get(name) {
  const env=this.lookup(name);
  if(env) return env.vars.get(name).val;
  return undefined;
}
```

**問題在 js0i.js:422-423**：

```javascript
if(this.check(TT.KEYWORD,"get")&&!this.check(TT.LPAREN,undefined,1)) { this.advance(); kind="get"; }
else if(this.check(TT.KEYWORD,"set")&&!this.check(TT.LPAREN,undefined,1)) { this.advance(); kind="set"; }
```

`check()` 函數的定義是：
```javascript
check(type, val) { const t=this.peek(); return t.type===type && (val===undefined||t.value===val); }
```

它只接受 `type` 和 `val` 兩個參數，**第三個參數（offset）被忽略了**！

因此 `!this.check(TT.LPAREN,undefined,1)` 實際上總是檢查當前 token 是否為 `(`，而當前 token 是 `get`，所以這個條件永遠為 `true`。

結果：`get(name) { ... }` 被錯誤地解析為 getter（kind="get"），但 getter 不應該有參數。當解析器嘗試解析參數後期待一個 block `{` 時，卻遇到了 `return` 關鍵字（方法體的第一行）。

### 正常情況 vs 錯誤情況
- 正確：`get name() { }` - getter，無參數
- 錯誤：`get name(arg) { }` - 被錯誤解析為 getter

## 修正方案

修改 `js0i.js` 第 422-423 行：

**原本（錯誤）：**
```javascript
if(this.check(TT.KEYWORD,"get")&&!this.check(TT.LPAREN,undefined,1)) { this.advance(); kind="get"; }
else if(this.check(TT.KEYWORD,"set")&&!this.check(TT.LPAREN,undefined,1)) { this.advance(); kind="set"; }
```

**修正後：**
```javascript
if(this.check(TT.KEYWORD,"get")&&this.peek(1).type!==TT.LPAREN) { this.advance(); kind="get"; }
else if(this.check(TT.KEYWORD,"set")&&this.peek(1).type!==TT.LPAREN) { this.advance(); kind="set"; }
```

這樣可以正確檢查 `get`/`set` 後面的 token 是否為 `(`，以區分 getter/setter 和普通方法。

## 驗證方法

修正後執行：
```bash
./test.sh
```

應該全部通過：
1. `node js/fact.js` - 正常執行
2. `node js0i.js js/fact.js` - 正常執行
3. `node js0i.js js0i.js js/hello.js` - 正常執行（之前失敗）
4. `node js0i.js js0i.js js/fact.js` - 正常執行
