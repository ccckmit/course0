# SHA-1 密碼學深入解析

## 1. 演算法概述

SHA-1（Secure Hash Algorithm 1）曾是主要的雜湎演算法，輸出 160 位元（20 位元組）的雜湎值。1995 年由 NSA 設計，FIPS 180-1 標準。

自 2005 年起發現理論碰撞攻擊，2017 年 SHA-1 碰撞實際產生。現已被 SHA-256 取代。

## 2. 與 SHA-256 的主要差異

| 特性 | SHA-1 | SHA-256 |
|------|-------|--------|
| 輸出長度 | 160 位元 | 256 位元 |
| 區塊大小 | 512 位元 | 512 位元 |
| 回合數 | 80 | 64 |
| 初始化 | 5 個 32 位元 | 8 個 32 位元 |
| 安全性 | 有已知碰撞 | 目前安全 |

## 3. 數學基礎

### 位元運算

#### ROTLEFT(a, b)

左旋轉：

```c
#define ROTLEFT(a, b) (((a) << (b)) | ((a) >> (32 - (b))))
```

### 邏輯函數

#### 0 ≤ i < 20（第一回合）

```
f = (B AND C) OR (NOT B AND D)
K = 0x5A827999
```

這是 CH 函數的變體。

#### 20 ≤ i < 40（第二回合）

```
f = B XOR C XOR D
K = 0x6ED9EBA1
```

XOR（Parity）。

#### 40 ≤ i < 60（第三回合）

```
f = (B AND C) OR (B AND D) OR (C AND D)
K = 0x8F1BBCDC
```

MAJ 函數的變體。

#### 60 ≤ i < 80（第四回合）

```
f = B XOR C XOR D
K = 0xCA62C1D6
```

與第二回合相同。

### 常數設計

每 20 回合更換函數的設計：
1. 引入非線性混淆
2. 增加差分分析的複雜度
3. 防止特定攻擊模式

## 4. 初始化

SHA-1 使用 5 個 32 位元初始化值：

```c
uint32_t state[5] = {
    0x67452301,  // A
    0xEFCDAB89,  // B
    0x98BADCFE,  // C
    0x10325476,  // D
    0xC3D2E1F0  // E
};
```

## 5. 訊息排程

### W[0..15]

從 512 位元區塊讀入 16 個 32 位元字：

```c
for (int i = 0; i < 16; i++) {
    W[i] = (data[i * 4] << 24) | (data[i * 4 + 1] << 16) 
         | (data[i * 4 + 2] << 8) | data[i * 4 + 3];
}
```

### W[16..79]

使用XOR 反饋：

```c
for (int i = 16; i < 80; i++) {
    W[i] = ROTLEFT(W[i - 3] ^ W[i - 8] ^ W[i - 14] ^ W[i - 16], 1);
}
```

注意：
- SHA-256 使用 SIG0/SIG1 函數
- SHA-1 使用簡單的 XOR + ROTATE
- SHA-1 需要 80 個字（後面是重複使用的）

## 6. Compression 函數

### 回合運算

```c
for (int i = 0; i < 80; i++) {
    uint32_t f, k;
    if (i < 20) {
        f = (b & c) | (~b & d);
        k = 0x5A827999;
    } else if (i < 40) {
        f = b ^ c ^ d;
        k = 0x6ED9EBA1;
    } else if (i < 60) {
        f = (b & c) | (b & d) | (c & d);
        k = 0x8F1BBCDC;
    } else {
        f = b ^ c ^ d;
        k = 0xCA62C1D6;
    }
    
    uint32_t temp = ROTLEFT(a, 5) + f + e + k + W[i];
    e = d;
    d = c;
    c = ROTLEFT(b, 30);
    b = a;
    a = temp;
}
```

### 設計原理

1. **ROTLEFT(a, 5)**：將 A 的影響快速傳播到其他位置
2. **c = ROTLEFT(b, 30)**：讓 B 的資訊延遲傳遞
3. **f 的變化**：每 20 回合更換，增加複雜度

### 狀態更新

每個區塊處理後：

```c
state[0] += a;
state[1] += b;
state[2] += c;
state[3] += d;
state[4] += e;
```

## 7. 填充

與 SHA-256 類似：

```
訊息長度（位元）附加到最後
```

但位元順序使用小端序（不同於 SHA-256 的大端序）：

```c
uint64_t bits = (uint64_t)len * 8;
for (int i = 0; i < 8; i++) {
    msg[padded_len - 1 - i] = (bits >> (i * 8)) & 0xff;
}
```

## 9. 安全性分析

### 歷史

- 2005 年：王小雲等人發現攻擊可在 2^69 步內找到碰撞
- 2017 年：Google 實作 SHAttered 攻擊，產生 PDF 碰撞
- 建議：遷移至 SHA-256

### 攻擊類型

1. 差分攻擊：利用特定差分路径
2. 模運算差分攻擊（Modifferential Cryptanalysis）
3. 垃圾邮件攻擊（垃圾邮件碰撞）

### 現狀

- 不應用於新系統
- 現有系統若僅使用 SHA-1，建議升級
- TLS 1.3 已移除 SHA-1

## 10. 使用場景

仍可使用但應避免的情況：
- 陳舊相容性
- 校驗和非安全性關鍵的場景
- Git（使用 SHA-1 作為內容定址）

### Git 使用 SHA-1

Git 使用 SHA-1 作為物件的定址：
- 內容雜湎作為物件名稱
- 利用內容雜湎的完整性

> 注意：Git 社群已計畫遷移至 SHA-256。

## 12. 密碼學原理總結

### 雪崩效應

每 1 位元變化導致約 50% 輸出位元變化。

### 单向性

從輸出無法推出輸入。

### 抗碰撞性

實際使用已不安全（理論上 2^80 步，實務上已少於）。

### 設計美學

SHA-1 和 SHA-2 系列代表了：
- 資料雜湊函數設計的演進
- 非線性構建的重要性
- 回合結構的設計原則
