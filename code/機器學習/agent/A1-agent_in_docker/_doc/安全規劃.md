# Agent0 安全規劃

## 目標

確保 AI Agent 在隔離的沙盒環境中運行，防止對宿主系統造成破壞。

## 安全策略

### 1. 工作目錄隔離

- Agent 只能在指定的工作目錄（沙盒）內操作
- 預設沙盒：`~/.agent0_sandbox/`
- 禁止存取 `/` 根目錄、`/home`、`/Users` 等系統目錄

### 2. 危險命令黑名單

以下命令直接拒絕執行：
- `rm -rf /` 或任何刪除系統目錄的命令
- `dd` / `mkfs` / `fdisk` 等磁碟操作
- `shutdown` / `reboot` / `halt`
- `chmod 777`（過度開放權限）
- `chown`（變更檔案所有者）
- `sudo` / `su`
- `curl` / `wget` 下載並執行未知腳本

### 3. 路徑檢查

- 所有檔案路徑必須在沙盒目錄內
- 禁止使用 `..` 穿越目錄（如 `/sandbox/../etc/passwd`）
- 禁止絕對路徑指向沙盒外部

### 4. 網路限制

- 禁止對外部網路的請求（除了必要的 Ollama API）
- 禁止綁定到 0.0.0.0 或允許外部存取

### 5. 資源限制

- 執行超時：30 秒
- 記憶體限制：避免 fork 炸彈

## 實作方式

```python
# 1. 路徑驗證
def is_safe_path(path):
    sandbox = os.path.abspath(SANDBOX_DIR)
    target = os.path.abspath(path)
    return target.startswith(sandbox)

# 2. 命令黑名單
DANGEROUS_PATTERNS = [
    r'rm\s+-rf\s+/',
    r'sudo\s+',
    r'dd\s+.*of=',
    # ...
]

# 3. 執行前檢查
def validate_command(cmd):
    if any(re.search(p, cmd) for p in DANGEROUS_PATTERNS):
        return False, "危險命令"
    return True, None
```
